# 用户认证：测试与部署指南

本文档对应 `TODOList/用户认证相关/10_测试与部署.txt` 的落地资产。

## 1. 测试体系

### 1.1 单元/集成测试（pytest + coverage）
- 认证核心与数据库测试：
  - `services/backend/tests/test_auth_core_module.py`
  - `services/backend/tests/test_auth_db_design.py`
- 新增安全与缓存集成测试：
  - `services/backend/tests/test_auth_security_and_cache_integration.py`

执行：

```bash
cd services/backend
../../venv/bin/pytest tests/test_auth_core_module.py tests/test_auth_db_design.py tests/test_auth_security_and_cache_integration.py \
  --cov=app/auth --cov-report=term-missing
```

### 1.2 E2E 测试（Playwright）
- 新增认证流程 E2E：
  - `tests/e2e/auth-workflow.admin.test.ts`
- 新增配置：
  - `configs/playwright.auth.config.ts`

执行：

```bash
npm run test:e2e:auth -- --project=chromium
```

覆盖流程：
- 注册（发送验证码 -> 完成注册）
- 登录
- 找回密码（发送验证码 -> 重置）

### 1.3 安全测试（OWASP ZAP）
- 脚本：`deployment/security/zap-auth-baseline.sh`

执行：

```bash
./deployment/security/zap-auth-baseline.sh https://your-domain.com
```

### 1.4 性能测试（Locust）
- 场景：`services/backend/tests/performance/locust_auth.py`
- 一键运行：`scripts/run_auth_locust.sh`

执行：

```bash
AUTH_PERF_HOST=http://127.0.0.1:8000 ./scripts/run_auth_locust.sh
```

报告输出：`reports/auth_performance/`

## 2. 生产部署

### 2.1 Docker + Nginx
- 后端镜像：`services/backend/Dockerfile`
- 编排文件：`deployment/docker-compose.auth.yml`
- HTTPS 反向代理：`deployment/nginx/auth_https.conf`
- 环境变量模板：`deployment/env/auth.production.env.example`

### 2.2 HTTPS
- 证书续签脚本：`scripts/renew_ssl_cert.sh`
- 示例：

```bash
./scripts/renew_ssl_cert.sh auth.example.com ops@example.com
```

### 2.3 数据库备份/恢复
- 备份：`scripts/backup_auth_db.sh`
- 恢复：`scripts/restore_auth_db.sh`
- 定时任务模板：`deployment/auth-maintenance.cron`

### 2.4 日志收集与轮转
- logrotate 配置：`deployment/logrotate/udake-auth`

### 2.5 一键部署
- 脚本：`scripts/deploy_auth_production.sh`

执行：

```bash
AUTH_DEPLOY_ENV_FILE=deployment/env/auth.production.env ./scripts/deploy_auth_production.sh
```

## 3. 部署后核验清单

```bash
curl -fsSk https://127.0.0.1/health
curl -fsSk https://127.0.0.1/api/auth/csrf-token
docker compose --env-file deployment/env/auth.production.env -f deployment/docker-compose.auth.yml ps
```

建议在上线后持续监控：
- 接口 P95 响应时间 < 500ms
- 错误率 < 1%
- 备份文件每日/每小时正常生成
