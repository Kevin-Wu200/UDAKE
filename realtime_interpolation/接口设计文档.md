# 实时插值系统接口设计文档

## 文档信息
- **文档版本**: v1.0
- **创建日期**: 2026-03-16
- **文档状态**: 草稿
- **作者**: iFlow CLI

## 1. 接口概述

实时插值系统提供以下主要接口：
1. **数据订阅接口**：管理用户的订阅请求
2. **增量更新接口**：处理新数据的增量更新
3. **查询接口**：查询插值结果
4. **缓存管理接口**：管理缓存系统
5. **状态监控接口**：监控系统状态

## 2. 数据订阅接口

### 2.1 创建订阅

**接口路径**: `POST /api/subscriptions`

**请求体**:
```json
{
  "data_type": "environmental_monitoring",
  "spatial_extent": {
    "min_x": 0,
    "max_x": 1000,
    "min_y": 0,
    "max_y": 800
  },
  "update_frequency": 5,
  "interpolation_params": {
    "method": "ordinary_kriging",
    "variogram_model": {
      "model_type": "spherical",
      "sill": 1.0,
      "range": 100.0,
      "nugget": 0.1
    },
    "grid_resolution": 100
  },
  "notification_config": {
    "websocket": true,
    "sse": false,
    "push_threshold": 0.1
  }
}
```

**响应体**:
```json
{
  "subscription_id": "sub_001",
  "status": "created",
  "created_at": "2026-03-16T10:00:00Z"
}
```

### 2.2 获取订阅信息

**接口路径**: `GET /api/subscriptions/{subscription_id}`

**响应体**:
```json
{
  "subscription_id": "sub_001",
  "data_type": "environmental_monitoring",
  "spatial_extent": {
    "min_x": 0,
    "max_x": 1000,
    "min_y": 0,
    "max_y": 800
  },
  "update_frequency": 5,
  "interpolation_params": {
    "method": "ordinary_kriging",
    "variogram_model": {
      "model_type": "spherical",
      "sill": 1.0,
      "range": 100.0,
      "nugget": 0.1
    },
    "grid_resolution": 100
  },
  "notification_config": {
    "websocket": true,
    "sse": false,
    "push_threshold": 0.1
  },
  "created_at": "2026-03-16T10:00:00Z",
  "active": true
}
```

### 2.3 列出所有订阅

**接口路径**: `GET /api/subscriptions`

**查询参数**:
- `active`: boolean（可选）- 是否只返回激活的订阅

**响应体**:
```json
{
  "subscriptions": [
    {
      "subscription_id": "sub_001",
      "data_type": "environmental_monitoring",
      "active": true
    }
  ],
  "total": 1
}
```

### 2.4 暂停/恢复订阅

**接口路径**: `PUT /api/subscriptions/{subscription_id}/status`

**请求体**:
```json
{
  "active": false
}
```

**响应体**:
```json
{
  "subscription_id": "sub_001",
  "status": "paused",
  "timestamp": "2026-03-16T10:30:00Z"
}
```

### 2.5 删除订阅

**接口路径**: `DELETE /api/subscriptions/{subscription_id}`

**响应体**:
```json
{
  "subscription_id": "sub_001",
  "status": "deleted",
  "timestamp": "2026-03-16T10:35:00Z"
}
```

## 3. 增量更新接口

### 3.1 单点更新

**接口路径**: `POST /api/updates`

**请求体**:
```json
{
  "subscription_id": "sub_001",
  "point": {
    "id": "pt_001",
    "x": 500.5,
    "y": 400.3,
    "value": 25.6,
    "timestamp": "2026-03-16T10:30:00Z",
    "metadata": {
      "sensor_id": "sensor_123",
      "quality": "high"
    }
  }
}
```

**响应体**:
```json
{
  "update_id": "update_001",
  "subscription_id": "sub_001",
  "timestamp": "2026-03-16T10:30:05Z",
  "update_type": "incremental",
  "affected_region": {
    "min_x": 450,
    "max_x": 550,
    "min_y": 350,
    "max_y": 450
  },
  "prediction_grid": {
    "resolution": 100,
    "data": [[25.1, 25.3, ...], ...]
  },
  "variance_grid": {
    "resolution": 100,
    "data": [[0.5, 0.4, ...], ...]
  },
  "version": 123,
  "statistics": {
    "update_time_ms": 150,
    "affected_points": 2500,
    "cache_hit_rate": 0.85
  }
}
```

### 3.2 批量更新

**接口路径**: `POST /api/updates/batch`

**请求体**:

```json
  "update_type": "batch",

      "update_id": "update_002",
      "point_id": "pt_002",
      "status": "success"
    },
    {
      "update_id": "update_003",
      "point_id": "pt_003",
      "status": "success"
    }
  ],
  "statistics": {
    "total_points": 2,
    "successful_updates": 2,
    "failed_updates": 0,
    "total_time_ms": 280
  }
}
```

### 3.3 获取更新状态

## 4. 查询接口

### 4.1 查询插值结果

**接口路径**: `GET /api/subscriptions/{subscription_id}/results`

**查询参数**:
- `version`: int（可选）- 指定版本号
- `region`: string（可选）- 指定查询区域（JSON格式）

**响应体**:
```json
{
  "subscription_id": "sub_001",
  "version": 123,
  "timestamp": "2026-03-16T10:30:05Z",
  "prediction_grid": {
    "resolution": 100,
    "data": [[25.1, 25.3, ...], ...]
  },
  "variance_grid": {
    "resolution": 100,
    "data": [[0.5, 0.4, ...], ...]
  },
  "spatial_extent": {
    "min_x": 0,
    "max_x": 1000,
    "min_y": 0,
    "max_y": 800
  }
}
```

### 4.2 查询指定点

**接口路径**: `GET /api/subscriptions/{subscription_id}/query`

**查询参数**:
- `x`: float - X坐标
- `y`: float - Y坐标
- `version`: int（可选）- 指定版本号

**响应体**:
```json
{
  "x": 500.5,
  "y": 400.3,
  "prediction": 25.6,
  "variance": 0.5,
  "version": 123,
  "timestamp": "2026-03-16T10:30:05Z"
}
```

### 4.3 查询区域统计

**接口路径**: `GET /api/subscriptions/{subscription_id}/statistics`

**查询参数**:
- `region`: string（可选）- 指定查询区域（JSON格式）
- `version`: int（可选）- 指定版本号

**响应体**:
```json
{
  "subscription_id": "sub_001",
  "version": 123,
  "statistics": {
    "mean": 25.3,
    "std": 2.5,
    "min": 20.1,
    "max": 30.5,
    "variance_mean": 0.5,
    "total_points": 10000
  }
}
```

```json
{

    "miss_rate": 0.40

    "hit_rate": 0.80,
    "miss_rate": 0.20
  }

```

### 5.2 清除缓存

**接口路径**: `DELETE /api/cache`

**查询参数**:
- `level`: string（可选）- 缓存级别（l1, l2, l3, all）

```json

  "cleared_entries": 850,
  "timestamp": "2026-03-16T10:35:00Z"
}
```

**接口路径**: `POST /api/cache/preheat`

**请求体**:
```json
{
  "subscription_id": "sub_001",
  "strategy": "static"
}
```

**响应体**:
```json
{
  "status": "success",
  "preheated_entries": 1000,
  "timestamp": "2026-03-16T10:36:00Z"
}
```

## 9. 接口认证

所有API接口（除了系统状态接口）都需要认证：

**认证方式**: Bearer Token

**请求头**:
```
Authorization: Bearer {token}
```

## 10. 速率限制

为了保护系统稳定性，所有接口都有速率限制：

| 接口类型 | 限制 |
|----------|------|
| 数据订阅 | 10次/分钟 |
| 增量更新 | 100次/分钟 |
| 查询接口 | 200次/分钟 |
| 缓存管理 | 20次/分钟 |
| 状态监控 | 60次/分钟 |

超过限制时返回：
```json
{
  "error": {
    "code": "ERR_RATE_LIMIT_EXCEEDED",
    "message": "超过速率限制",
    "retry_after": 60
  }
}
```

## 11. 版本控制

所有响应数据都包含版本号，支持缓存一致性检查：

```json
{
  "version": 123,
  "timestamp": "2026-03-16T10:30:05Z",
  "data": {...}
}
```

客户端可以使用 `If-None-Match` 请求头来利用缓存：

```
If-None-Match: "123"
```

如果数据未变更，服务器返回 `304 Not Modified`。
