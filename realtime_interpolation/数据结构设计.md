## 2. 核心数据结构

### 2.1 空间数据结构

#### 2.1.1 BoundingBox（边界框）

```python
@dataclass
class BoundingBox:
    min_x: float  # 最小X坐标
    max_x: float  # 最大X坐标
    min_y: float  # 最小Y坐标
    max_y: float  # 最大Y坐标
```

**用途**：
- 定义空间范围
- 快速判断点是否在范围内
- 计算空间交集

**关键方法**：
- `contains(x, y)`: 判断点是否在边界框内
- `intersects(other)`: 判断两个边界框是否相交
- `area()`: 计算面积

**优化考虑**：
- 使用float类型存储，精度足够且内存占用小
- 内联方法以提高性能
- 支持快速序列化

#### 2.1.2 DataPoint（数据点）

```python
@dataclass
class DataPoint:
    x: float              # X坐标
    y: float              # Y坐标
    value: float          # 观测值
    id: str               # 唯一标识符
    timestamp: datetime   # 时间戳（可选）
    metadata: dict        # 元数据（可选）
```

**用途**：
- 表示单个观测数据点
- 支持时间序列数据
- 存储额外的元数据信息

**设计考虑**：
- 使用str类型的ID而非int，支持分布式场景
- timestamp支持可选，兼容静态数据
- metadata使用dict，灵活存储各种属性

### 2.2 插值相关数据结构

#### 2.2.1 VariogramModel（变异函数模型）

```python
@dataclass
class VariogramModel:
    model_type: str  # spherical, exponential, gaussian, linear
    sill: float      # 基台值
    range: float     # 变程
    nugget: float    # 块金值
```

**用途**：
- 描述空间相关性
- 计算协方差矩阵
- 支持多种变异函数模型

**支持的模型**：
1. **球状模型（spherical）**：
   - 最常用的模型
   - 适合大多数空间数据

2. **指数模型（exponential）**：
   - 适合长距离相关性
   - 渐近趋于基台值

3. **高斯模型（gaussian）**：
   - 适合非常平滑的空间变化
   - 在原点处可导

4. **线性模型（linear）**：
   - 最简单的模型
   - 适合线性相关

**关键方法**：
- `covariance(point1, point2)`: 计算两点间的协方差
- `_covariance_by_distance(distance)`: 根据距离计算协方差

### 2.3 订阅管理数据结构

#### 2.3.1 Subscription（订阅）

```python
@dataclass
class Subscription:
    subscription_id: str           # 订阅ID
    data_type: str                 # 数据类型
    spatial_extent: BoundingBox    # 空间范围
    update_frequency: int          # 更新频率（秒）
    interpolation_params: dict     # 插值参数
    notification_config: dict      # 通知配置
    created_at: datetime           # 创建时间
    active: bool                   # 是否激活
```

**用途**：
- 管理用户的订阅请求
- 控制更新频率
- 配置插值和通知参数

**设计考虑**：
- 使用dict存储参数，灵活性高
- active标志支持暂停/恢复订阅
- created_at用于审计和统计

### 2.4 更新结果数据结构

#### 2.4.1 UpdateResult（更新结果）

```python
@dataclass
class UpdateResult:
    update_id: str                 # 更新ID
    subscription_id: str           # 订阅ID
    timestamp: datetime            # 时间戳
    update_type: str               # 更新类型
    affected_region: BoundingBox   # 影响域
    prediction_grid: np.ndarray    # 预测值栅格
    variance_grid: np.ndarray      # 方差栅格
    version: int                   # 版本号
    statistics: dict               # 统计信息
```

**用途**：
- 封装增量更新的结果
- 包含预测值和方差
- 提供性能统计信息

**设计考虑**：
- 使用numpy数组存储栅格数据，效率高
- version支持缓存一致性检查
- statistics用于性能监控

### 2.5 缓存数据结构

#### 2.5.1 CacheEntry（缓存条目）

```python
@dataclass
class CacheEntry:
    key: str                    # 缓存键
    value: Any                  # 缓存值
    version: int                # 版本号
    expire_time: float          # 过期时间
    access_count: int           # 访问计数
    last_access: float          # 最后访问时间
```

**用途**：
- 存储缓存条目
- 支持多种替换策略
- 跟踪访问模式

**设计考虑**：
- access_count支持LFU策略
- last_access支持LRU策略
- version支持一致性检查

**关键方法**：
- `is_expired()`: 检查是否过期
- `touch()`: 更新访问时间和计数

### 2.6 事件数据结构

#### 2.6.1 Event（事件）

```python
@dataclass
class Event:
    event_type: str         # 事件类型
    subscription_id: str    # 订阅ID
    timestamp: datetime     # 时间戳
    data: dict              # 事件数据
```

**用途**：
- 表示系统事件
- 支持事件发布订阅
- 便于序列化和传输

**事件类型**：
- `interpolation_update`: 插值更新
- `cache_hit`: 缓存命中
- `cache_miss`: 缓存未命中
- `error`: 错误事件
- `system_status`: 系统状态

### 2.7 系统监控数据结构

#### 2.7.1 SystemStatus（系统状态）

```python
@dataclass
class SystemStatus:
    timestamp: datetime                # 时间戳
    system_status: str                 # 系统状态
    performance: dict                  # 性能指标
    subscriptions: dict                # 订阅统计
    alerts: list                       # 告警列表
```

**用途**：
- 监控系统健康状态
- 收集性能指标
- 管理告警信息

**性能指标**：
- `avg_update_time_ms`: 平均更新时间
- `max_update_time_ms`: 最大更新时间
- `concurrent_updates`: 并发更新数
- `cache_hit_rate`: 缓存命中率
- `memory_usage_gb`: 内存使用量
- `cpu_usage_percent`: CPU使用率

## 4. 数据结构使用场景

### 4.1 数据订阅流程

```
用户请求
  ↓
创建 Subscription
  ↓
设置 spatial_extent（BoundingBox）
  ↓
配置 interpolation_params（包含 VariogramModel）
  ↓
返回 Subscription
```

### 4.2 增量更新流程

```
新数据到达（DataPoint）
  ↓
计算影响域（BoundingBox）
  ↓
更新协方差矩阵（使用 VariogramModel）
  ↓
生成 UpdateResult
  ↓
缓存（CacheEntry）
  ↓
发布事件（Event）
```

### 4.3 缓存查询流程

```
查询请求
  ↓
生成缓存键
  ↓
检查 CacheEntry
  ↓
验证版本号
  ↓
检查是否过期
  ↓
返回结果或触发计算
```

## 5. 数据结构优化策略

### 5.1 内存优化

1. **使用numpy数组**：栅格数据使用numpy数组，内存占用小
2. **共享引用**：相同对象使用引用而非拷贝
3. **延迟加载**：大数据按需加载
4. **对象池**：频繁创建的对象使用对象池

### 5.2 性能优化

1. **内联方法**：简单方法直接实现，减少函数调用开销
2. **缓存计算结果**：避免重复计算
3. **批量操作**：支持批量处理数据
4. **异步IO**：IO操作异步化

### 5.3 可维护性优化

1. **类型注解**：使用类型注解提高代码可读性
2. **文档字符串**：详细的文档说明
3. **单元测试**：每个数据结构都有对应的测试
4. **版本控制**：数据结构版本化，支持升级

## 7. 数据结构验证

### 7.1 输入验证

```python
def validate_datapoint(point: DataPoint) -> bool:
    """验证数据点"""
    if not isinstance(point.x, (int, float)):
        raise ValueError("x坐标必须是数字")
    if not isinstance(point.y, (int, float)):
        raise ValueError("y坐标必须是数字")
    if not isinstance(point.value, (int, float)):
        raise ValueError("value必须是数字")
    if not isinstance(point.id, str):
        raise ValueError("id必须是字符串")
    return True
```

### 7.2 一致性检查

```python
def check_consistency(subscription: Subscription) -> bool:
    """检查订阅一致性"""
    if subscription.update_frequency <= 0:
        raise ValueError("更新频率必须大于0")
    if not subscription.spatial_extent.min_x < subscription.spatial_extent.max_x:
        raise ValueError("空间范围无效")
    return True
```
