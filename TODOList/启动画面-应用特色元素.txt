启动画面应用特色元素

任务描述：
在启动画面中添加与应用功能相关的视觉元素，如地图网格、数据点、不确定性波纹等，增强品牌识别度和功能暗示。

具体修改：
1. 添加地图网格背景
   - 在背景层添加抽象的地图网格
   - 使用 SVG 或 CSS 绘制网格线
   - 网格线条使用半透明白色
   - 添加缓慢的平移或缩放动画
   - 原因：暗示应用的地图功能

2. 添加采样点动画
   - 在背景中随机分布几个采样点图标
   - 采样点使用应用中的图标样式
   - 添加脉冲效果（从小到大淡出）
   - 不同采样点有不同的动画延迟
   - 原因：展示应用的采样功能

3. 添加不确定性波纹效果
   - 在采样点周围添加波纹效果
   - 波纹使用渐变透明度
   - 波纹扩散速度和大小随机
   - 暗示不确定性分析和克里金插值
   - 原因：突出应用的核心算法

4. 添加数据流动画
   - 创建数据点之间的连线
   - 数据流使用发光效果
   - 数据流动画从采样点到中心
   - 暗示数据处理流程
   - 原因：可视化数据处理过程

5. 添加3D效果暗示
   - 在背景中添加等高线纹理
   - 使用渐变色模拟地形高度
   - 轻微的3D旋转效果
   - 暗示3D克里金功能
   - 原因：展示高级功能

6. 性能优化
   - 使用 CSS 动画而非 JavaScript
   - 使用 transform 和 opacity 而非其他属性
   - 限制动画元素数量（不超过10个）
   - 使用 will-change 属性优化性能
   - 原因：确保启动速度不受影响

7. 响应式适配
   - 移动端减少元素数量
   - 小屏幕隐藏部分装饰元素
   - 确保在所有设备上流畅运行

影响范围：
- apps/frontend/css/splash-screen.css
- 可能需要创建新的 SVG 文件或使用 CSS 绘制元素
- apps/frontend/js/components/SplashScreen.ts（如果需要动态生成元素）

测试要点：
- 不同屏幕尺寸下的显示效果
- 动画性能测试（CPU和GPU使用率）
- 启动时间是否受到影响
- 视觉元素是否干扰主要内容
- 深色模式下的显示效果
