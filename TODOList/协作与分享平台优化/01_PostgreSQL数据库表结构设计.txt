# 任务1.1：PostgreSQL数据库表结构设计

优先级：最高（数据持久化基础）

## 任务描述
设计并创建PostgreSQL数据库表结构，支持协作与分享平台的所有数据存储需求。

## 子任务清单

### 核心表设计
- [x] 创建workflows表（工作流定义和元数据）
  - 字段：id, name, description, definition, created_at, updated_at, owner_id, is_public
  - 索引：owner_id, is_public, created_at
- [x] 创建workflow_versions表（版本历史）
  - 字段：id, workflow_id, version_number, definition, created_at, created_by_id
  - 索引：workflow_id, version_number
- [x] 创建teams表（团队信息）
  - 字段：id, name, description, created_at, owner_id
  - 索引：owner_id
- [x] 创建team_members表（团队成员）
  - 字段：id, team_id, user_id, role, joined_at
  - 索引：team_id, user_id

### 协作相关表
- [x] 创建invitations表（邀请记录）
  - 字段：id, team_id, workflow_id, invited_by_id, invitee_email, token, status, expires_at
  - 索引：team_id, workflow_id, token, status
- [x] 创建delegations表（权限委托）
  - 字段：id, workflow_id, delegator_id, delegate_id, permissions, granted_at, expires_at
  - 索引：workflow_id, delegator_id, delegate_id
- [x] 创建share_links表（分享链接）
  - 字段：id, workflow_id, created_by_id, token, access_mode, password, expires_at, access_count
  - 索引：workflow_id, token
- [x] 创建comments表（评论）
  - 字段：id, workflow_id, parent_id, user_id, content, created_at, updated_at
  - 索引：workflow_id, parent_id, user_id

### 通知和日志表
- [x] 创建notifications表（通知）
  - 字段：id, user_id, type, title, content, is_read, created_at, reference_id
  - 索引：user_id, is_read, type
- [x] 创建collaboration_operations表（协作操作日志）
  - 字段：id, workflow_id, user_id, operation_type, operation_data, created_at
  - 索引：workflow_id, user_id, created_at
- [x] 创建collaboration_cursors表（协作光标）
  - 字段：id, workflow_id, user_id, cursor_position, color, last_updated
  - 索引：workflow_id, user_id
- [x] 创建collaboration_conflicts表（冲突记录）
  - 字段：id, workflow_id, user_id, conflict_type, conflict_data, resolved, created_at
  - 索引：workflow_id, user_id, resolved

## 依赖项
- 数据库：PostgreSQL 14+
- ORM：SQLAlchemy或asyncpg

## 验证标准
- [x] 所有表创建成功
- [x] 外键关系正确建立
- [x] 索引创建正确
- [x] 支持事务操作
- [x] 符合数据库范式

## 实施顺序
1. 设计核心表结构（workflows, workflow_versions, teams, team_members）
2. 设计协作相关表（invitations, delegations, share_links, comments）
3. 设计通知和日志表（notifications, collaboration_operations, collaboration_cursors, collaboration_conflicts）
4. 编写SQL迁移脚本
5. 执行表创建和验证
