## 概述

UDAKE（智能不确定性驱动空间决策平台）是一个基于克里金插值的空间分析系统，包含以下组件：

- **后端服务**：FastAPI + PyKrige + NumPy/SciPy
- **前端应用**：Vite + 原生JavaScript
- **地图引擎**：支持高德地图、ArcGIS、天地图
- **移动应用**：Android（Capacitor）
- **桌面应用**：Electron

## 部署方案选择

### 方案对比

| 方案 | 优点 | 缺点 | 适用场景 |
|------|------|------|----------|
| **Docker部署** | 环境隔离、部署快速、易于维护 | 需要Docker知识 | 推荐使用 |
| **传统部署** | 完全控制、资源占用少 | 配置复杂、依赖管理麻烦 | 无Docker环境 |
| **混合部署** | 灵活性高 | 运维复杂 | 特殊需求 |

### 推荐选择

**生产环境推荐：Docker部署**
- 环境一致性保证
- 快速部署和回滚
- 便于扩展和迁移

## 方案一：Docker部署（推荐）

### 1. 准备服务器

```bash
# 更新系统
sudo apt update && sudo apt upgrade -y

# 安装基础工具
sudo apt install -y git curl wget vim

# 克隆项目
git clone <your-repo-url> /opt/udake
cd /opt/udake
```

### 2. 安装Docker

```bash
# 安装Docker
curl -fsSL https://get.docker.com -o get-docker.sh
sudo sh get-docker.sh

# 安装Docker Compose
sudo curl -L "https://github.com/docker/compose/releases/latest/download/docker-compose-$(uname -s)-$(uname -m)" -o /usr/local/bin/docker-compose
sudo chmod +x /usr/local/bin/docker-compose

# 验证安装
docker --version
docker-compose --version

# 将当前用户添加到docker组
sudo usermod -aG docker $USER
newgrp docker
```

### 3. 创建后端Dockerfile

在项目根目录创建 `backend/Dockerfile`：

```dockerfile
FROM python:3.9-slim

# 设置工作目录
WORKDIR /app

# 安装系统依赖
RUN apt-get update && apt-get install -y \
    gdal-bin \
    libgdal-dev \
    gcc \
    g++ \
    && rm -rf /var/lib/apt/lists/*

# 复制依赖文件
COPY requirements.txt .

# 安装Python依赖
RUN pip install --no-cache-dir -r requirements.txt

# 复制应用代码
COPY . .

# 暴露端口
EXPOSE 8000

# 启动命令
CMD ["python", "run.py"]
```

### 4. 创建前端Dockerfile

在项目根目录创建 `frontend/Dockerfile`：

```dockerfile
FROM node:18-alpine AS builder

WORKDIR /app

# 复制package文件
COPY package*.json ./

# 安装依赖
RUN npm ci

# 复制源代码
COPY . .

# 构建生产版本
RUN npm run build:prod

# 生产镜像
FROM nginx:alpine

# 复制构建产物
COPY --from=builder /app/dist /usr/share/nginx/html

# 复制nginx配置
COPY deployment/nginx.conf /etc/nginx/nginx.conf

EXPOSE 80

CMD ["nginx", "-g", "daemon off;"]
```

### 5. 配置环境变量

```bash
# 复制环境变量模板
cp deployment/生产环境变量.env .env

# 编辑环境变量
vim .env
```

关键配置项：

```env
# 应用配置
APP_NAME=智能不确定性驱动空间决策平台
VERSION=1.0.0
DEBUG=False

# 服务器配置
HOST=0.0.0.0
PORT=8000

# CORS配置
CORS_ORIGINS=["https://your-domain.com"]

# 地图API密钥（必须配置）
VITE_AMAP_API_KEY=your_amap_api_key
VITE_AMAP_SECURITY_CODE=your_amap_security_code

# 任务配置
MAX_CONCURRENT_TASKS=10
TASK_TIMEOUT=7200
```

### 6. 启动服务

```bash
# 进入部署目录
cd deployment

# 构建并启动服务
docker-compose up -d

# 查看服务状态
docker-compose ps

# 查看日志
docker-compose logs -f
```

### 7. 验证部署

```bash
# 检查后端健康
curl http://localhost:8000/health

# 检查前端访问
curl http://localhost

# 访问API文档
curl http://localhost:8000/docs
```

### 8. 常用Docker命令

```bash
# 停止服务
docker-compose down

# 重启服务
docker-compose restart

# 查看日志
docker-compose logs -f backend
docker-compose logs -f nginx

# 进入容器
docker-compose exec backend bash

# 更新服务
git pull
docker-compose build --no-cache
docker-compose up -d

# 清理旧镜像
docker image prune -a
```

## 方案二：传统部署

### 1. 安装Python环境

```bash
# 安装Python 3.9
sudo apt install -y python3.9 python3.9-venv python3-pip

# 创建虚拟环境
cd /opt/udake
python3.9 -m venv venv
source venv/bin/activate

# 安装依赖
pip install --upgrade pip
pip install -r requirements.txt
```

### 2. 安装GDAL依赖

```bash
# 安装GDAL系统库
sudo apt install -y gdal-bin libgdal-dev

# 验证GDAL安装
gdalinfo --version
python -c "from osgeo import gdal; print(gdal.__version__)"
```

### 3. 配置后端服务

```bash
# 创建systemd服务文件
sudo vim /etc/systemd/system/udake-backend.service
```

内容如下：

```ini
[Unit]
Description=UDAKE Backend Service
After=network.target

[Service]
Type=simple
User=www-data
WorkingDirectory=/opt/udake/backend
Environment="PATH=/opt/udake/venv/bin"
ExecStart=/opt/udake/venv/bin/python run.py
Restart=always
RestartSec=10

[Install]
WantedBy=multi-user.target
```

启动服务：

```bash
# 重载systemd配置
sudo systemctl daemon-reload

# 启动服务
sudo systemctl start udake-backend

# 设置开机自启
sudo systemctl enable udake-backend

# 查看状态
sudo systemctl status udake-backend
```

### 4. 安装Node.js和构建前端

```bash
# 安装Node.js 18
curl -fsSL https://deb.nodesource.com/setup_18.x | sudo -E bash -
sudo apt install -y nodejs

# 验证安装
node --version
npm --version

# 安装依赖
npm install

# 构建生产版本
npm run build:prod
```

### 5. 配置Nginx

创建Nginx配置：

```bash
sudo vim /etc/nginx/sites-available/udake
```

内容见[Nginx配置](#nginx配置)章节

启用配置：

```bash
# 创建软链接
sudo ln -s /etc/nginx/sites-available/udake /etc/nginx/sites-enabled/

# 测试配置
sudo nginx -t

# 重启Nginx
sudo systemctl restart nginx
```

### 6. 配置环境变量

```bash
# 复制环境变量文件
cp .env.example .env

# 编辑配置
vim .env
```

## Nginx配置

### 完整配置文件

```nginx
# /etc/nginx/sites-available/udake

# 后端API代理
upstream backend {
    server 127.0.0.1:8000;
    keepalive 32;
}

# HTTP服务器
server {
    listen 80;
    server_name your-domain.com www.your-domain.com;

    # 重定向到HTTPS
    return 301 https://$server_name$request_uri;
}

# HTTPS服务器
server {
    listen 443 ssl http2;
    server_name your-domain.com www.your-domain.com;

    # SSL证书配置
    ssl_certificate /etc/letsencrypt/live/your-domain.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/your-domain.com/privkey.pem;
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers HIGH:!aNULL:!MD5;
    ssl_prefer_server_ciphers on;

    # 安全头部
    add_header X-Frame-Options "SAMEORIGIN" always;
    add_header X-Content-Type-Options "nosniff" always;
    add_header X-XSS-Protection "1; mode=block" always;
    add_header Strict-Transport-Security "max-age=31536000; includeSubDomains" always;

    # 日志配置
    access_log /var/log/nginx/udake_access.log;
    error_log /var/log/nginx/udake_error.log;

    # 客户端上传大小限制
    client_max_body_size 100M;

    # 前端静态文件
    location / {
        root /opt/udake/frontend/dist;
        index index.html;
        try_files $uri $uri/ /index.html;

        # 静态资源缓存
        location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2|ttf|eot)$ {
            expires 1y;
            add_header Cache-Control "public, immutable";
        }
    }

    # API代理
    location /api/ {
        proxy_pass http://backend/api/;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_cache_bypass $http_upgrade;
        proxy_read_timeout 300s;
        proxy_connect_timeout 75s;
    }

    # WebSocket代理
    location /ws/ {
        proxy_pass http://backend/ws/;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_read_timeout 86400;
    }

    # 结果文件
    location /results/ {
        alias /opt/udake/backend/app/结果文件/;
        autoindex off;
        add_header Content-Type application/octet-stream;
        add_header Content-Disposition "attachment";
    }

    # Gzip压缩
    gzip on;
    gzip_vary on;
    gzip_min_length 1000;
    gzip_comp_level 6;
    gzip_types text/plain text/css application/json application/javascript text/xml application/xml application/xml+rss text/javascript;
}
```

## SSL证书配置

### 使用Let's Encrypt免费证书

```bash
# 安装Certbot
sudo apt install -y certbot python3-certbot-nginx

# 获取证书（自动配置Nginx）
sudo certbot --nginx -d your-domain.com -d www.your-domain.com

# 测试自动续期
sudo certbot renew --dry-run

# 查看证书信息
sudo certbot certificates
```

### 证书自动续期

Certbot会自动配置cron任务。如需手动配置：

```bash
# 添加cron任务
sudo crontab -e

# 添加以下行（每天凌晨3点检查续期）
0 3 * * * certbot renew --quiet --post-hook "systemctl reload nginx"
```

## 备份策略

### 1. 数据备份

```bash
# 创建备份脚本
cat > /opt/scripts/backup.sh << 'EOF'
#!/bin/bash
# 数据备份脚本

BACKUP_DIR="/opt/backups/udake"
DATE=$(date +%Y%m%d_%H%M%S)

# 创建备份目录
mkdir -p $BACKUP_DIR

# 备份结果文件
tar -czf $BACKUP_DIR/results_$DATE.tar.gz /opt/udake/backend/app/结果文件/

# 备份环境配置
cp /opt/udake/.env $BACKUP_DIR/.env_$DATE

# 备份Nginx配置
cp /etc/nginx/sites-available/udake $BACKUP_DIR/nginx_$DATE.conf

# 删除30天前的备份
find $BACKUP_DIR -name "*.tar.gz" -mtime +30 -delete

echo "Backup completed: $DATE"
EOF

chmod +x /opt/scripts/backup.sh

# 添加到cron（每天凌晨2点备份）
0 2 * * * /opt/scripts/backup.sh
```

### 2. 数据库备份（如使用）

```bash
# PostgreSQL备份
pg_dump -U username -d udake > $BACKUP_DIR/udake_$DATE.sql

# MySQL备份
mysqldump -u username -p udake > $BACKUP_DIR/udake_$DATE.sql
```

### 3. 恢复数据

```bash
# 恢复结果文件
tar -xzf $BACKUP_DIR/results_20250116_020000.tar.gz -C /

# 恢复环境配置
cp $BACKUP_DIR/.env_20250116_020000 /opt/udake/.env
```

## 性能优化

### 1. 后端优化

#### Uvicorn配置

修改 `backend/run.py`：

```python
uvicorn.run(
    "app.main:app",
    host=settings.HOST,
    port=settings.PORT,
    workers=4,           # 工作进程数（CPU核心数）
    limit_concurrency=100,  # 最大并发连接数
    timeout_keep_alive=30,  # 保持连接超时
    log_level="info"
)
```

#### 添加Gunicorn + Uvicorn

```bash
pip install gunicorn

# 使用Gunicorn启动
gunicorn app.main:app \
    --workers 4 \
    --worker-class uvicorn.workers.UvicornWorker \
    --bind 0.0.0.0:8000 \
    --timeout 300
```

### 2. 前端优化

#### CDN加速

```nginx
# 静态资源使用CDN
location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2|ttf|eot)$ {
    proxy_pass https://cdn.your-domain.com;
    expires 1y;
    add_header Cache-Control "public, immutable";
}
```

#### 启用HTTP/2

Nginx配置已包含 `http2` 支持。

### 3. 系统优化

#### 调整文件描述符限制

```bash
# 临时调整
ulimit -n 65536

# 永久调整
sudo vim /etc/security/limits.conf

# 添加以下内容
* soft nofile 65536
* hard nofile 65536
```

#### 优化TCP参数

```bash
sudo vim /etc/sysctl.conf

# 添加以下内容
net.core.somaxconn = 1024
net.ipv4.tcp_max_syn_backlog = 4096
net.ipv4.tcp_tw_reuse = 1
net.ipv4.tcp_fin_timeout = 30

# 应用配置
sudo sysctl -p
```

## 安全加固

### 1. 防火墙配置

```bash
# 安装ufw
sudo apt install -y ufw

# 配置防火墙规则
sudo ufw default deny incoming
sudo ufw default allow outgoing

# 允许SSH
sudo ufw allow 22/tcp

# 允许HTTP/HTTPS
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp

# 启用防火墙
sudo ufw enable

# 查看状态
sudo ufw status
```

### 2. 禁用root远程登录

```bash
sudo vim /etc/ssh/sshd_config

# 修改以下配置
PermitRootLogin no
PasswordAuthentication no

# 重启SSH服务
sudo systemctl restart sshd
```

### 3. 安装fail2ban

```bash
# 安装fail2ban
sudo apt install -y fail2ban

# 创建配置文件
sudo cp /etc/fail2ban/jail.conf /etc/fail2ban/jail.local

# 启动服务
sudo systemctl start fail2ban
sudo systemctl enable fail2ban
```

### 4. 定期更新

```bash
# 自动安全更新
sudo apt install -y unattended-upgrades

# 配置自动更新
sudo dpkg-reconfigure -plow unattended-upgrades
```

