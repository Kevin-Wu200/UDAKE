# 地图引擎切换功能测试指南

## 快速开始

### 1. 配置天地图 Token（推荐）

如果你想使用天地图模式，需要先配置 Token：

1. 访问 https://console.tianditu.gov.cn/ 注册并申请 Token
2. 编辑 `frontend/js/config/tianditu.config.js`
3. 将 `TOKEN: 'YOUR_TIANDITU_TOKEN_PLACEHOLDER'` 替换为你的真实 Token

### 2. 选择地图引擎

编辑 `frontend/js/config/map.config.js`：

```javascript
export const MapConfig = {
    MAP_PROVIDER: 'tianditu',  // 使用天地图
    // MAP_PROVIDER: 'arcgis',  // 或使用 ArcGIS
};
```

### 3. 启动应用

```bash
cd dist
npm start
```

## 常见问题

### 问题 1：地图无法加载

**可能原因：**
- Leaflet 库未正确加载
- 天地图 Token 无效
- 网络连接问题

**解决方法：**
1. 检查浏览器控制台错误信息
2. 确认 Leaflet CSS 和 JS 已加载
3. 检查天地图 Token 是否正确
4. 尝试切换到 ArcGIS 模式

### 问题 2：采样点无法显示

**可能原因：**
- 坐标系统不匹配
- 适配器实现问题

**解决方法：**
1. 检查控制台错误信息
2. 确认坐标格式正确（经度、纬度）
3. 查看 `addMarker` 方法是否正常执行

### 问题 3：栅格图层无法显示（天地图模式）

**这是预期行为：**
- 天地图模式下加载 GeoTIFF 需要额外插件
- 当前实现提供了接口，但未集成插件
- 建议使用 ArcGIS 模式测试栅格功能

**解决方法：**
- 切换到 ArcGIS 模式测试栅格功能
- 或集成 georaster-layer-for-leaflet 插件

### 问题 4：切换地图引擎后功能异常

**可能原因：**
- 缓存问题
- 适配器状态未清理

**解决方法：**
1. 完全关闭应用
2. 清除浏览器缓存
3. 重新启动应用

## 性能对比

### 天地图模式
- ✅ 加载速度快
- ✅ 资源占用少
- ⚠️ 栅格图层需要插件

### ArcGIS 模式
- ✅ 功能完整
- ✅ 栅格图层原生支持
- ⚠️ 加载速度较慢
- ⚠️ 资源占用较多
