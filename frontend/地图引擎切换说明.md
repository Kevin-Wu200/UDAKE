# 地图引擎切换说明

## 概述

UDAKE 平台现已支持两种地图引擎：
- **ArcGIS API for JavaScript** - 原有地图引擎
- **天地图 + Leaflet** - 新增轻量级地图引擎

## 切换方式

编辑 `frontend/js/config/map.config.js` 文件，修改 `MAP_PROVIDER` 的值：

```javascript
export const MapConfig = {
    // 地图引擎选择：'arcgis' 或 'tianditu'
    MAP_PROVIDER: 'tianditu',  // 改为 'arcgis' 可切换回 ArcGIS
    ...
};
```

## 配置天地图 Token

1. 访问天地图开发者平台：https://console.tianditu.gov.cn/
2. 注册账号并申请 Token
3. 编辑 `frontend/js/config/tianditu.config.js`，替换 Token：

```javascript
export const TiandituConfig = {
    // 天地图 Token（需要替换为真实 Token）
    TOKEN: 'YOUR_TIANDITU_TOKEN_PLACEHOLDER',  // 替换为真实 Token
    ...
};
```

## 配置 ArcGIS API Key

编辑 `frontend/js/config/arcgis.config.js`，替换 API Key：

```javascript
export const ArcGISConfig = {
    // API Key配置
    API_KEY: "YOUR_ARCGIS_API_KEY_PLACEHOLDER",  // 替换为真实 API Key
    ...
};
```

## 架构说明

### 适配器模式

系统采用适配器模式，实现了地图引擎的解耦：

```
地图初始化.js
    ↓
MapConfig (选择引擎)
    ↓
MapAdapter (统一接口)
    ↓
├── ArcGISAdapter (ArcGIS 实现)
└── TiandituAdapter (天地图 + Leaflet 实现)
```

### 核心文件

- `frontend/js/config/map.config.js` - 地图引擎配置
- `frontend/js/config/arcgis.config.js` - ArcGIS 配置
- `frontend/js/config/tianditu.config.js` - 天地图配置
- `frontend/js/adapters/MapAdapter.js` - 地图适配器接口
- `frontend/js/adapters/ArcGISAdapter.js` - ArcGIS 适配器实现
- `frontend/js/adapters/TiandituAdapter.js` - 天地图适配器实现
- `frontend/js/地图初始化.js` - 地图初始化逻辑
- `frontend/js/图层管理.js` - 图层管理器

### 统一接口

所有地图操作通过 `MapAdapter` 接口进行：

- `initMap(containerId)` - 初始化地图
- `addPointsLayer(geojson, layerName)` - 添加点图层
- `addRasterLayer(type, url)` - 添加栅格图层
- `addMarker(pointData)` - 添加单个标记
- `addPolygon(coordinates, options)` - 添加多边形
- `toggleLayer(layerName, visible)` - 切换图层可见性
- `setLayerOpacity(layerName, opacity)` - 设置图层透明度
- `removeLayer(layerName)` - 移除图层
- `clearAllLayers()` - 清除所有图层
- `zoomToLayer(layerName)` - 缩放到图层
- `setClickHandler(handler)` - 设置点击事件
- `getSamplingPoints()` - 获取采样点

## 坐标系统

- **统一使用 WGS-84 (EPSG:4326)** 作为数据坐标系
- 天地图使用 WGS-84 坐标系，无需转换
- ArcGIS 使用 Web Mercator，内部自动处理转换
- 插值算法接收 WGS-84 坐标

## 功能支持

### 天地图模式

✅ 支持的功能：
- 地图基础展示
- 采样点显示
- GeoJSON 图层加载
- 点击查询
- 图层控制
- 坐标系统信息

⚠️ 部分支持：
- 栅格图层（需要额外插件支持）

### ArcGIS 模式

✅ 完整支持所有功能

## 注意事项

1. **不删除 ArcGIS 代码** - 所有 ArcGIS 相关代码保留，可随时切换
2. **Mock 模式** - 未配置 Token/API Key 时自动进入 Mock 模式，地图仍可加载
3. **栅格图层** - 天地图模式下加载 GeoTIFF 需要额外插件（如 georaster-layer-for-leaflet）
4. **深色模式** - ArcGIS 支持自动切换深色底图，天地图暂不支持
5. **性能** - 天地图 + Leaflet 更轻量，加载速度更快

## 未来扩展

可以轻松添加其他地图引擎：
1. 创建新的适配器类（如 `MapboxAdapter`）
2. 实现 `MapAdapter` 接口
3. 在 `map.config.js` 中添加新选项
4. 在 `地图初始化.js` 中添加分支逻辑

## 问题排查

### 地图无法加载

1. 检查 `map.config.js` 中的 `MAP_PROVIDER` 配置
2. 检查浏览器控制台错误信息
3. 确认 Leaflet 库已正确加载（天地图模式）
4. 确认 ArcGIS API 已正确加载（ArcGIS 模式）

### Token/API Key 错误

1. 确认 Token/API Key 已正确配置
2. 检查 Token/API Key 是否有效
3. 查看控制台警告信息

### 图层无法显示

1. 检查图层控制面板中的开关状态
2. 确认数据格式正确
3. 查看控制台错误信息
