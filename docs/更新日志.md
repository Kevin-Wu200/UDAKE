# 更新日志

## 2026-04-02

### 域名环境变量配置与文档同步

- 完成 `OFFICIAL_WEB`、`ADMIN_WEB` 域名变量的部署文档补充，明确不同环境的配置方式。
- 补充构建时域名替换机制说明：`npm run build:dev/test/prod` 会在 Vite 构建后执行 `scripts/replace-domains.js`。
- 更新开发文档中的本地环境配置，新增 `.env` 配置示例、生效验证步骤与调试技巧。
- 更新 API 文档中的基础 URL 说明，改为使用环境变量管理（`VITE_API_BASE_URL` / `API_BASE_URL`）。

### 变更原因与影响

- 原因：此前文档中存在固定域名与旧启动方式示例，易导致多环境部署不一致。
- 影响：开发、测试、生产环境的 API 与官网域名配置流程统一，减少手动替换域名导致的错误。
- 兼容性：旧的 `VITE_API_URL` 仍可使用，但推荐迁移到 `VITE_API_BASE_URL`。

### 修改文件清单

- `docs/构建部署指南.md`
- `docs/系统文档.md`
- `docs/分布式计算API与测试指南.md`
- `docs/更新日志.md`

### 迁移注意事项

- `OFFICIAL_WEB` 与 `ADMIN_WEB` 必须成对配置，否则构建后替换步骤会失败。
- 修改 `configs/env/.env.*` 后需重启 `npm run dev`，避免读取到旧缓存变量。
- 若新增外链域名模式，需同步更新 `scripts/replace-domains.js` 的 `REPLACERS` 并执行 `npm run test:replace-domains`。

### 升级指南

1. 在 `configs/env/.env.development`、`.env.testing`、`.env.production` 中补齐 `OFFICIAL_WEB` 与 `ADMIN_WEB`。
2. 使用 `npm run build:dev` 或 `npm run build:test` 先验证构建输出与替换报告。
3. 检查 `dist/domain-replaced/` 目录确认链接已按目标域名替换。
4. 发布前执行 `npm run build:prod` 与 `npm run test:replace-domains`。
