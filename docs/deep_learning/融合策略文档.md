# 融合策略文档

## 概述

融合策略（Fusion Strategy）是融合模型的核心组件，定义了如何组合多个子模型的预测结果。UDAKE 提供了8种预定义策略，每种策略都有其特定的应用场景和优缺点。本文档详细介绍了每种策略的原理、适用场景和实现细节。

## 策略列表

| 策略名称 | 代码标识 | 需要真值 | 自适应能力 | 不确定性估计 |
|---------|---------|---------|-----------|------------|
| 简单平均 | `simple_average` | 否 | 无 | 支持 |
| 加权平均 | `weighted_average` | 否 | 无 | 支持 |
| 中位数 | `median` | 否 | 无 | 支持 |
| 最大最小 | `max_min` | 否 | 无 | 支持 |
| 堆叠融合 | `stacking` | 是 | 有 | 支持 |
| 贝叶斯模型平均 | `bayesian_model_average` | 是 | 无 | 支持 |
| 方差加权 | `variance_weighted` | 否 | 无 | 支持 |
| 动态融合 | `dynamic` | 否 | 有 | 支持 |

## 策略详解

### 1. 简单平均（Simple Average）

#### 1.1 原理
简单平均是最基础的融合策略，对所有模型的预测结果取等权重平均值：

```
fused(x) = (1/N) * Σ model_i(x)
```

其中 N 是模型数量，model_i(x) 是第 i 个模型的预测。

#### 1.2 不确定性估计
启用不确定性估计时，使用样本方差估计预测不确定性：

```
variance(x) = (1/(N-1)) * Σ (model_i(x) - fused(x))²
```

#### 1.3 优点
- **简单高效**：计算复杂度低，易于实现
- **无需训练**：不需要标注数据或训练过程
- **鲁棒性好**：对单一模型错误不敏感
- **无偏估计**：在模型误差无偏时保持无偏性

#### 1.4 缺点
- **无法区分性能**：对所有模型一视同仁
- **性能受限**：无法充分利用高性能模型优势
- **信息浪费**：忽略模型间的性能差异

#### 1.5 适用场景
- 模型性能相近且无明显差异
- 快速原型验证
- 缺乏标注数据
- 对计算效率要求高的场景

#### 1.6 使用示例

```python
from deep_learning.fusion import FusionConfig, FusionStrategy

config = FusionConfig(
    strategy=FusionStrategy.SIMPLE_AVERAGE,
    enable_uncertainty=True
)

result = engine.fuse(models, config=config)
```

### 2. 加权平均（Weighted Average）

#### 2.1 原理
加权平均根据模型性能分配不同权重，加权求和得到融合结果：

```
fused(x) = Σ w_i * model_i(x)
```

其中 w_i 是第 i 个模型的权重，满足 Σ w_i = 1。

#### 2.2 权重计算
权重通过权重计算方法确定，常见方法包括：

- **RMSE Based**：w_i ∝ 1/RMSE_i
- **MAE Based**：w_i ∝ 1/MAE_i
- **R² Based**：w_i ∝ R²_i

权重会进行归一化处理，确保和为1。

#### 2.3 不确定性估计
使用加权方差估计不确定性：

```
variance(x) = Σ w_i * (model_i(x) - fused(x))²
```

#### 2.4 优点
- **性能导向**：突出高性能模型贡献
- **灵活配置**：支持多种权重计算方法
- **理论成熟**：有坚实的理论基础
- **可解释性强**：权重直观反映模型重要性

#### 2.5 缺点
- **依赖评估**：需要准确评估模型性能
- **权重敏感**：权重计算方法影响结果
- **静态权重**：权重不随输入变化

#### 2.6 适用场景
- 模型性能差异明显
- 有可靠的性能评估数据
- 需要明确模型贡献度
- 对融合精度要求较高

#### 2.7 使用示例

```python
from deep_learning.fusion import FusionConfig, FusionStrategy, WeightMethod

config = FusionConfig(
    strategy=FusionStrategy.WEIGHTED_AVERAGE,
    weight_method=WeightMethod.RMSE_BASED,
    normalize=True,
    smoothing=True,
    smoothing_factor=0.1
)

result = engine.fuse(models, config=config, true_values=true_values)
```

### 3. 中位数（Median）

#### 3.1 原理
中位数策略取所有模型预测的中位数作为融合结果：

```
fused(x) = median(model_1(x), model_2(x), ..., model_N(x))
```

#### 3.2 不确定性估计
使用四分位距（IQR）估计标准差：

```
IQR = Q75 - Q25
sigma = IQR / 1.35
variance = sigma²
```

#### 3.3 优点
- **抗异常值**：对极端预测不敏感
- **鲁棒性强**：不受个别模型错误影响
- **统计稳定**：对数据分布变化不敏感
- **无需参数**：无需额外参数配置

#### 3.4 缺点
- **信息损失**：只保留中间值，丢失极值信息
- **效率较低**：需要排序操作，计算复杂度较高
- **不适用于小样本**：模型数量少时效果有限

#### 3.5 适用场景
- 存在异常预测或离群值
- 模型数量较多（N ≥ 5）
- 需要保守估计
- 数据质量不稳定

#### 3.6 使用示例

```python
from deep_learning.fusion import FusionConfig, FusionStrategy

config = FusionConfig(
    strategy=FusionStrategy.MEDIAN,
    enable_uncertainty=True
)

result = engine.fuse(models, config=config)
```

### 4. 最大最小（Max-Min）

#### 4.1 原理
最大最小策略取预测范围的中点：

```
max_pred = max(model_i(x))
min_pred = min(model_i(x))
fused(x) = (max_pred + min_pred) / 2
```

#### 4.2 不确定性估计
使用预测范围估计标准差：

```
range = max_pred - min_pred
sigma = range / 4
variance = sigma²
```

#### 4.3 优点
- **捕捉范围**：反映预测的不确定性范围
- **保守估计**：避免过度自信
- **简单直观**：易于理解和解释
- **无需真值**：不依赖标注数据

#### 4.4 缺点
- **受极端值影响**：最大最小值对异常值敏感
- **信息利用不足**：只利用了极值信息
- **可能不准确**：在模型预测集中时效果差

#### 4.5 适用场景
- 需要保守估计的风险评估
- 模型预测较为分散
- 对预测范围感兴趣
- 缺乏真值标注

#### 4.6 使用示例

```python
from deep_learning.fusion import FusionConfig, FusionStrategy

config = FusionConfig(
    strategy=FusionStrategy.MAX_MIN,
    enable_uncertainty=True
)

result = engine.fuse(models, config=config)
```

### 5. 堆叠融合（Stacking）

#### 5.1 原理
堆叠融合通过交叉验证学习元权重，训练线性组合器：

```
fused(x) = Σ w_i * model_i(x)
```

权重 w_i 通过K折交叉验证学习，最小化预测误差。

#### 5.2 学习过程
1. 将训练数据分为K折
2. 对于每一折，使用其他K-1折训练权重
3. 在验证集上评估权重效果
4. 平均所有折的权重作为最终权重
5. 约束权重非负且和为1

#### 5.3 数学形式
使用最小二乘法求解权重：

```
min ||Xw - y||²
s.t. w_i ≥ 0, Σ w_i = 1
```

其中 X 是预测矩阵，y 是真值，w 是待学习权重。

#### 5.4 不确定性估计
使用加权方差：

```
variance(x) = Σ w_i * (model_i(x) - fused(x))²
```

#### 5.5 优点
- **自动学习**：自动学习最优权重
- **数据驱动**：充分利用标注数据
- **避免过拟合**：交叉验证防止过拟合
- **性能优秀**：通常能达到较好效果

#### 5.6 缺点
- **计算开销大**：需要多次训练
- **依赖真值**：必须有标注数据
- **样本要求高**：需要足够训练样本
- **时间成本高**：训练过程耗时

#### 5.7 适用场景
- 有充足的标注数据
- 对融合精度要求极高
- 模型性能差异明显
- 离线训练场景

#### 5.8 使用示例

```python
from deep_learning.fusion import FusionConfig, FusionStrategy, WeightMethod

config = FusionConfig(
    strategy=FusionStrategy.STACKING,
    weight_method=WeightMethod.RMSE_BASED,
    n_folds=5,
    enable_uncertainty=True
)

result = engine.fuse(
    models,
    config=config,
    true_values=true_values
)
```

### 6. 贝叶斯模型平均（Bayesian Model Average）

#### 6.1 原理
基于贝叶斯框架，根据模型证据分配后验概率：

```
P(M|D) = P(D|M) * P(M) / P(D)
fused(x) = Σ P(M_i|D) * model_i(x)
```

其中 P(M|D) 是模型后验概率，P(D|M) 是模型证据。

#### 6.2 权重计算
使用BIC（贝叶斯信息准则）近似模型证据：

```
BIC = k * ln(n) - 2 * ln(L)
P(M|D) ∝ exp(-BIC/2)
```

其中 k 是参数数量，n 是样本数，L 是似然。

#### 6.3 不确定性估计
考虑模型间的不确定性：

```
variance(x) = Σ P(M_i|D) * (model_i(x) - fused(x))² + Σ P(M_i|D) * σ_i²
```

其中 σ_i² 是模型 i 的预测方差。

#### 6.4 优点
- **理论严谨**：基于概率论和贝叶斯理论
- **考虑模型不确定性**：量化模型选择的不确定性
- **概率解释**：提供概率解释的融合结果
- **自动权重**：无需人工指定权重

#### 6.5 缺点
- **计算复杂**：需要计算模型证据
- **假设依赖**：依赖先验假设
- **近似误差**：实际应用中常使用近似方法
- **数据要求**：需要足够数据估计参数

#### 6.6 适用场景
- 需要概率解释的场景
- 模型不确定性重要
- 有充足数据
- 对理论严谨性要求高

#### 6.7 使用示例

```python
from deep_learning.fusion import FusionConfig, FusionStrategy, WeightMethod

config = FusionConfig(
    strategy=FusionStrategy.BAYESIAN_MODEL_AVERAGE,
    weight_method=WeightMethod.BMA,
    enable_uncertainty=True
)

result = engine.fuse(
    models,
    config=config,
    true_values=true_values
)
```

### 7. 方差加权（Variance Weighted）

#### 7.1 原理
根据预测方差的倒数分配权重：

```
w_i(x) = (1/σ_i²(x)) / Σ (1/σ_j²(x))
fused(x) = Σ w_i(x) * model_i(x)
```

其中 σ_i²(x) 是模型 i 在输入 x 处的预测方差。

#### 7.2 逻辑
- 方差小的模型更可靠，给予更大权重
- 方差大的模型不确定性高，给予较小权重
- 权重随输入动态变化

#### 7.3 不确定性估计
融合后的方差为：

```
1/variance(x) = Σ (1/σ_i²(x))
```

#### 7.4 优点
- **不确定性导向**：优先信任低不确定性模型
- **动态权重**：权重随输入变化
- **理论合理**：基于不确定性理论
- **无需真值**：不依赖标注数据

#### 7.5 缺点
- **依赖方差估计**：需要准确的方差估计
- **方差不准确**：模型方差估计误差会影响效果
- **回退机制**：没有方差时需要回退策略

#### 7.6 适用场景
- 模型提供可靠的方差估计
- 不确定性量化重要
- 需要动态权重
- 无标注数据

#### 7.7 使用示例

```python
from deep_learning.fusion import FusionConfig, FusionStrategy, WeightMethod

config = FusionConfig(
    strategy=FusionStrategy.VARIANCE_WEIGHTED,
    weight_method=WeightMethod.RMSE_BASED,  # 回退权重方法
    enable_uncertainty=True
)

result = engine.fuse(models, config=config)
```

### 8. 动态融合（Dynamic）

#### 8.1 原理
根据输入特征和上下文动态调整权重：

```
w_i(x) = softmax(log(w_i^base) + log(r_i(x)) / diff_scale(x))
fused(x) = Σ w_i(x) * model_i(x)
```

其中：
- w_i^base 是基础权重
- r_i(x) 是可靠性评分
- diff_scale(x) 是难度缩放因子

#### 8.2 权重调整因素

##### 8.2.1 模型分歧
```
disagreement_i(x) = |model_i(x) - mean(model_j(x))|
```

##### 8.2.2 可靠性评分
```
reliability_i(x) = 1 / (variance_i(x) + disagreement_i(x))
```

##### 8.2.3 难度缩放
```
diff_scale(x) = clip(difficulty(x), 0.2, 5.0)
```

#### 8.3 不确定性估计
使用动态加权方差：

```
variance(x) = Σ w_i(x) * (model_i(x) - fused(x))²
```

#### 8.4 诊断信息
返回每个样本点的动态权重：

```
diagnostics = {
    "dynamic_weights": {
        "model_1": [w_1(x_1), w_1(x_2), ...],
        "model_2": [w_2(x_1), w_2(x_2), ...],
        ...
    }
}
```

#### 8.5 优点
- **自适应能力强**：根据输入动态调整
- **个性化融合**：每个样本点有不同权重
- **利用上下文**：考虑输入特征和难度
- **灵活性好**：适应数据分布变化

#### 8.6 缺点
- **计算复杂**：需要逐点计算权重
- **需要上下文**：依赖难度等上下文信息
- **可解释性**：权重变化规律不如静态策略清晰
- **不稳定**：权重可能波动较大

#### 8.7 适用场景
- 数据分布多变
- 不同区域需要不同融合策略
- 有上下文信息可用
- 对自适应能力要求高

#### 8.8 使用示例

```python
from deep_learning.fusion import FusionConfig, FusionStrategy, WeightMethod, AdaptiveLearningMode

config = FusionConfig(
    strategy=FusionStrategy.DYNAMIC,
    weight_method=WeightMethod.ADAPTIVE,
    adaptive_mode=AdaptiveLearningMode.NEURAL,
    enable_uncertainty=True
)

context = {
    "difficulty": difficulty_scores  # 难度评分
}

result = engine.fuse(
    models,
    config=config,
    context=context
)
```

## 策略对比

### 性能对比

| 策略 | 平均性能 | 最佳情况 | 最差情况 | 稳定性 |
|-----|---------|---------|---------|--------|
| 简单平均 | 中 | 模型性能相近时 | 性能差异大时 | 高 |
| 加权平均 | 高 | 性能评估准确时 | 评估不准确时 | 中 |
| 中位数 | 中 | 存在异常值时 | 模型数量少时 | 高 |
| 最大最小 | 低 | 需要保守估计时 | 预测集中时 | 中 |
| 堆叠融合 | 最高 | 数据充足时 | 数据不足时 | 低 |
| 贝叶斯模型平均 | 高 | 理论假设成立时 | 假设不成立时 | 中 |
| 方差加权 | 高 | 方差估计准确时 | 方差估计不准时 | 中 |
| 动态融合 | 高 | 上下文信息丰富时 | 上下文信息不足时 | 低 |

### 计算开销对比

| 策略 | 时间复杂度 | 空间复杂度 | 训练需求 |
|-----|-----------|-----------|---------|
| 简单平均 | O(N) | O(N) | 无 |
| 加权平均 | O(N) | O(N) | 可选 |
| 中位数 | O(N log N) | O(N) | 无 |
| 最大最小 | O(N) | O(N) | 无 |
| 堆叠融合 | O(K*N*M) | O(N*M) | 必需 |
| 贝叶斯模型平均 | O(N) | O(N) | 可选 |
| 方差加权 | O(N) | O(N) | 无 |
| 动态融合 | O(N*M) | O(N*M) | 可选 |

其中 N 是模型数量，M 是样本数量，K 是折数。

## 策略选择指南

### 快速选择

**选择简单平均**，如果：
- 模型性能相近
- 需要快速实现
- 缺乏标注数据

**选择加权平均**，如果：
- 模型性能差异明显
- 有可靠的性能评估
- 需要明确模型贡献

**选择中位数**，如果：
- 存在异常预测
- 模型数量较多
- 需要鲁棒性

**选择堆叠融合**，如果：
- 有充足标注数据
- 对精度要求极高
- 离线训练场景

**选择动态融合**，如果：
- 数据分布多变
- 有上下文信息
- 需要自适应能力

### 决策树

```
是否有真值标注？
├─ 是 → 数据量充足？
│   ├─ 是 → 精度要求高？→ 堆叠融合
│   └─ 否 → 理论严谨？→ 贝叶斯模型平均
│             └─ 否 → 加权平均
└─ 否 → 模型性能相近？
    ├─ 是 → 需要抗异常？→ 中位数
    │         └─ 否 → 简单平均
    └─ 否 → 有方差估计？→ 方差加权
              └─ 有上下文？→ 动态融合
                └─ 加权平均
```

## 策略优化

### 1. 参数调优
- **归一化**：启用 `normalize=True` 确保权重和为1
- **平滑**：启用 `smoothing=True` 减少权重波动
- **约束**：设置 `min_weight` 和 `max_weight` 限制权重范围

### 2. 组合策略
- **主备策略**：主策略失效时自动切换到备策略
- **层次融合**：先按区域分组融合，再整体融合
- **多策略投票**：多个策略结果加权平均

### 3. 在线学习
- **权重更新**：根据新数据在线更新权重
- **策略切换**：根据性能自动选择最优策略
- **持续优化**：持续监控和调整参数

## 常见问题

### Q1: 如何评估不同策略的效果？
使用 `compare_strategies` 方法比较所有策略的评估指标：

```python
results = engine.compare_strategies(models, true_values=true_values)
for strategy, result in results.items():
    print(f"{strategy}: RMSE={result.metrics['rmse']}")
```

### Q2: 堆叠融合的折数如何选择？
- 数据量大（>1000）：5-10折
- 数据量中等（100-1000）：5折
- 数据量小（<100）：留一法

### Q3: 动态融合的上下文如何获取？
上下文可以包括：
- 难度评分：基于特征距离或密度
- 时序信息：时间窗口内的历史表现
- 空间信息：邻近点的预测一致性
- 置信度：模型输出的置信度分数

### Q4: 如何处理权重计算失败？
系统会自动回退到简单平均：

```python
config = FusionConfig(
    strategy=FusionStrategy.VARIANCE_WEIGHTED,
    weight_method=WeightMethod.RMSE_BASED  # 回退权重方法
)
```

### Q5: 不确定性估计何时使用？
- **风险管理**：需要量化预测风险
- **决策支持**：需要不确定性信息辅助决策
- **模型评估**：评估模型预测可靠性
- **主动学习**：基于不确定性选择样本
