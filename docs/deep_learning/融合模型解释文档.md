# 融合模型解释文档

## 概述

融合模型（Fusion Model）是 UDAKE 系统的核心组件，负责整合多个子模型的预测结果，通过智能权重分配和策略选择，生成更准确、更鲁棒的融合预测。融合模型支持多种融合策略、自适应权重计算和不确定性量化，适用于空间插值、时空预测、异常检测等多种场景。

## 核心概念

### 1. 模型融合

模型融合（Model Fusion）是指将多个独立模型的预测结果整合为一个统一预测的过程。通过结合不同模型的优势，融合模型能够：

- **降低误差**：不同模型的误差相互抵消，降低整体预测误差
- **提高鲁棒性**：单一模型失效时，其他模型提供支撑
- **增强泛化能力**：融合多角度信息，适应更多数据分布
- **量化不确定性**：通过模型间分歧估计预测不确定性

### 2. 融合策略

融合策略定义了如何组合多个模型的预测结果。UDAKE 支持以下策略：

#### 2.1 简单平均（Simple Average）
- **原理**：对所有模型的预测结果取等权重平均值
- **优点**：简单易实现，无需训练
- **缺点**：无法区分模型性能差异
- **适用场景**：模型性能相近时

#### 2.2 加权平均（Weighted Average）
- **原理**：根据模型性能分配不同权重，加权求和
- **优点**：突出高性能模型贡献
- **缺点**：依赖权重计算方法
- **适用场景**：模型性能差异明显时

#### 2.3 中位数（Median）
- **原理**：取所有模型预测的中位数
- **优点**：对异常值不敏感
- **缺点**：丢失极端信息
- **适用场景**：存在异常预测时

#### 2.4 最大最小（Max-Min）
- **原理**：取最大和最小预测的平均值
- **优点**：捕捉预测范围
- **缺点**：极端值影响大
- **适用场景**：需要保守估计时

#### 2.5 堆叠融合（Stacking）
- **原理**：使用交叉验证学习元权重，训练线性组合器
- **优点**：自动学习最优权重
- **缺点**：计算开销大，需真值标注
- **适用场景**：有充足训练数据时

#### 2.6 贝叶斯模型平均（Bayesian Model Average）
- **原理**：基于贝叶斯框架，根据模型证据分配后验概率
- **优点**：理论严谨，考虑模型不确定性
- **缺点**：计算复杂
- **适用场景**：需要概率解释时

#### 2.7 方差加权（Variance Weighted）
- **原理**：根据预测方差的倒数分配权重
- **优点**：优先信任低方差模型
- **缺点**：依赖方差估计准确性
- **适用场景**：模型提供方差估计时

#### 2.8 动态融合（Dynamic）
- **原理**：根据输入特征和上下文动态调整权重
- **优点**：适应性强，个性化融合
- **缺点**：需要上下文信息
- **适用场景**：数据分布多变时

### 3. 权重计算方法

权重计算方法决定了如何评估模型性能并分配权重。UDAKE 支持以下方法：

#### 3.1 基于性能（Performance Based）
- **RMSE Based**：根据均方根误差倒数分配权重
- **MAE Based**：根据平均绝对误差倒数分配权重
- **R² Based**：根据决定系数分配权重

#### 3.2 基于不确定性（Uncertainty Based）
- **Variance Based**：根据预测方差倒数分配权重
- **Uncertainty Quantification**：结合认知不确定性和偶然不确定性

#### 3.3 自适应学习（Adaptive）
- **Neural Based**：使用神经网络学习权重映射
- **Online Learning**：在线更新权重，适应数据漂移

### 4. 自适应模式

自适应模式控制权重更新和策略选择的学习机制：

- **STATIC**：静态权重，不进行学习
- **ONLINE**：在线学习，根据新数据更新权重
- **NEURAL**：基于神经网络的自适应学习
- **HYBRID**：混合模式，结合多种学习策略

## 系统架构

### 1. 核心组件

```
融合模型系统
├── ModelFusionEngine（融合引擎）
│   ├── FusionWeightCalculator（权重计算器）
│   ├── FusionStrategies（策略执行器）
│   ├── FusionEvaluator（评估器）
│   └── 缓存系统（LRU Cache）
├── AdaptiveFusionSystem（自适应系统）
│   ├── 在线权重更新
│   ├── 策略自动选择
│   └── 性能监控
├── HybridFusionBridge（混合融合）
│   ├── 克里金与深度学习融合
│   ├── 多模态融合
│   └── 自适应模型选择
├── FusionFeatureAnalyzer（特征分析器）
│   ├── 特征重要性分析
│   ├── 模型贡献度分析
│   └── 预测诊断
└── FusionModelManager（模型管理器）
    ├── 模型注册
    ├── 版本管理
    └── 持久化存储
```

### 2. 工作流程

#### 2.1 预测融合流程

```python
# 1. 准备输入
models = [
    ModelPrediction(
        model_id="model_1",
        predictions=[...],
        variances=[...],
        ...
    ),
    # ... 其他模型
]

# 2. 配置融合参数
config = FusionConfig(
    strategy=FusionStrategy.DYNAMIC,
    weight_method=WeightMethod.ADAPTIVE,
    adaptive_mode=AdaptiveLearningMode.NEURAL,
    enable_uncertainty=True
)

# 3. 执行融合
result = engine.fuse(
    models=models,
    config=config,
    true_values=true_values,  # 可选，用于评估
    context=context  # 可选，用于动态融合
)

# 4. 获取结果
fused_predictions = result.fused_predictions
uncertainty = result.fused_variances
weights = result.weights
metrics = result.metrics
```

#### 2.2 自适应学习流程

```python
# 1. 在线预测
result = adaptive.online_fuse(
    models=models,
    base_config=config,
    true_values=true_values,
    context=context
)

# 2. 权重更新
online_weights = result["online_weights"]

# 3. 策略调整
selected_strategy = result["selected_strategy"]

# 4. 持续学习
# 系统会根据新数据自动调整权重和策略
```

### 3. 缓存机制

为了提高性能，融合引擎实现了多层缓存：

#### 3.1 权重缓存
- 缓存模型评估结果和计算出的权重
- 避免重复计算，加速融合过程
- LRU 策略管理缓存大小

#### 3.2 融合结果缓存
- 缓存融合计算结果
- 支持相同输入的快速响应
- 自动失效和更新

#### 3.3 缓存监控
- 命中率统计
- 性能指标追踪
- 缓存大小管理

## 评估指标

### 1. 预测性能指标

- **RMSE**：均方根误差
- **MAE**：平均绝对误差
- **MAPE**：平均绝对百分比误差
- **R²**：决定系数
- **sMAPE**：对称平均绝对百分比误差

### 2. 不确定性指标

- **Mean Sigma**：平均标准差
- **Calibration Error**：校准误差
- **CRPS**：连续分级概率评分
- **Coverage**：覆盖率

### 3. 多样性指标

- **Ensemble Diversity**：集成多样性
- **Pairwise Correlation**：成对相关性
- **Disagreement Measure**：分歧度量

### 4. 改进指标

- **Improvement vs Baseline**：相对于基线的改进
- **Performance Gain**：性能提升
- **Reduction in Uncertainty**：不确定性降低

## 高级特性

### 1. 混合融合

#### 1.1 克里金与深度学习融合
- **Residual Mode**：深度学习预测克里金残差

- **Confidence Mode**：基于置信度加权

#### 1.2 多模态融合

### 2. 自适应模型选择
- 根据输入特征动态选择模型
- 性能和不确定性平衡
- 在线学习和更新

### 3. 特征分析
