## 不确定性量化模型概述

### 什么是不确定性量化？

不确定性量化是估计模型预测的不确定性程度，为决策提供可靠性评估。不确定性可分为两种类型：

1. **认知不确定性**：源于模型参数的不确定性，可以通过更多数据减少
2. **偶然不确定性**：源于数据本身的不确定性，无法通过更多数据减少

### UDAKE不确定性量化模型体系

UDAKE平台提供四种深度学习不确定性量化模型，提供完整的不确定性估计和分解能力：

#### 1. 贝叶斯神经网络（BNN）

**核心原理**：
BNN将神经网络权重视为随机变量，通过变分推断近似后验分布，自然地量化认知不确定性。

**数学模型**：
```
p(w) = 𝒩(w; 0, σ²)  # 先验分布
qθ(w) ≈ p(w|𝒟)      # 后验分布近似
```

```
ℒ = 𝔼qθ(w)[log p(y|x,w)] - DKL(qθ(w) || p(w))  # ELBO损失
```

**优势**：
- 理论基础扎实，基于贝叶斯框架
- 自然地分解认知和偶然不确定性
- 权重信噪比分析
- 后验分布可视化

**适用场景**：
- 需要概率解释
- 数据稀缺
- 需要主动学习

**代码位置**：`/deep_learning/models/uncertainty/bnn.py`

#### 2. MC Dropout

**核心原理**：
MC Dropout在推理时保持Dropout激活，通过多次前向传播获得预测分布，是一种简单而有效的不确定性量化方法。

**数学模型**：
```
ĝ = Dropout(σ(Wx + b))  # Dropout激活
ŷ = (1/S)∑s=1ˢ f(x; θs)  # 蒙特卡洛平均
```

```
σ²(x) = τ²(1/S)∑s=1ˢ f(x; θs)² - ŷ² + (1/L)||ŷ - ȳ||²  # 不确定性估计
```

**优势**：
- 实现简单，易于部署
- 计算效率高
- 支持自适应性采样
- 稳定性评分

**适用场景**：
- 快速不确定性估计
- 在线应用
- 计算资源有限

**代码位置**：`/deep_learning/models/uncertainty/mc_dropout.py`

#### 3. 深度集成（Deep Ensemble）

**核心原理**：
深度集成通过训练多个具有不同初始化的模型并聚合其预测，提供稳健的不确定性估计。

**数学模型**：
```
θk ~ Uniform(θmin, θmax), k = 1, ..., K  # 不同初始化
ŷ = (1/K)∑k=1ᴷ f(x; θk)                 # 集成预测
```

```
σ²(x) = (1/K)∑k=1ᴷ σk²(x) + (1/K)∑k=1ᴷ f(x; θk)² - ŷ²  # 不确定性估计
```

**优势**：
- 不确定性估计稳健
- 支持模型多样性分析
- 成员贡献评估
- 并行计算友好

**适用场景**：
- 追求高质量不确定性估计
- 计算资源充足
- 需要模型多样性分析

**代码位置**：`/deep_learning/models/uncertainty/deep_ensemble.py`

#### 4. 证据深度学习（EDL）

**核心原理**：
EDL通过证据理论将模型的置信度与证据相结合，为分类任务提供可靠的不确定性估计，能够区分知识和数据不确定性。

**数学模型**：
```
ei = softplus(zi) = log(1 + exp(zi))  # 证据激活
p(π|e) = Dir(π; e + 1)                # Dirichlet分布
```

```
𝒰 = K / (∑i=1ᴷ ei + K)  # 不确定性估计
```

**优势**：
- 区分知识和数据不确定性
- 理论基础扎实
- 置信度分布分析
- 可靠性图表

**适用场景**：
- 分类任务
- 需要区分不确定性类型
- 需要置信度校准

**代码位置**：`/deep_learning/models/uncertainty/edl.py`

### 不确定性类型分解

```
总不确定性 = 认知不确定性 + 偶然不确定性

认知不确定性（Epistemic）：
- 源于模型参数的不确定性
- 可以通过更多数据减少
- 反映模型的置信度
- 指导主动学习

偶然不确定性（Aleatoric）：
- 源于数据本身的不确定性
- 无法通过更多数据减少
- 反映数据的噪声
- 指导风险决策
```

## LIME解释方法说明

### LIME在不确定性量化中的应用

#### 1. 解释不确定性来源

LIME可以解释为什么模型对某个预测给出特定程度的不确定性：

```python
from lime.lime_tabular import LimeTabularExplainer

# 创建LIME解释器
explainer = LimeTabularExplainer(
    training_data=training_samples,
    feature_names=['feature1', 'feature2', 'feature3'],
    mode='regression'
)

# 解释不确定性
explanation = explainer.explain_instance(
    data_point=query_point,
    predict_fn=model.predict_with_uncertainty,
    num_features=5
)

# 显示解释
explanation.show_in_notebook()
```

#### 2. 识别影响不确定性的关键特征

```python
# 获取特征重要性
feature_importance = explanation.as_list()

# 输出特征重要性
for feature, importance in feature_importance:
    print(f"{feature}: {importance:.4f}")
```

#### 3. 局部不确定性分析

```python
# 局部近似解释
local_approx = explanation.local_pred

# 与全局模型对比
global_pred, global_uncertainty = model.predict_with_uncertainty(query_point)
print(f"局部近似: {local_approx:.4f}")
print(f"全局预测: {global_pred:.4f}")
print(f"全局不确定性: {global_uncertainty:.4f}")
```

## 不确定性参数说明

### BNN参数配置

```python
class BayesianNeuralRegressor:
    def __init__(self,
                 in_dim: int,
                 hidden_dim: int = 32,
                 prior: PriorType | None = None,
                 seed: int = 42):
        """
        参数说明:
        - in_dim: 输入维度
        - hidden_dim: 隐藏层维度
          * 较小值 (16-32): 模型能力有限
          * 中等值 (32-64): 平衡性能
          * 较大值 (64-128): 表达能力强
          * 推荐值: 32-64

        - prior: 先验分布类型
          * None: 默认高斯先验
          * "gaussian": 高斯先验
          * "laplace": 拉普拉斯先验
          * "horseshoe": 马蹄先验
        """
```

### MC Dropout参数配置

```python
@dataclass
class MCDropoutConfig:
    in_dim: int
    hidden_dim: int = 32
    dropout_rate: float = 0.2  # Dropout率
    dropout_type: DropoutType = "standard"
    n_samples: int = 100       # 蒙特卡洛采样次数
```

**关键参数说明**：

- `dropout_rate`: Dropout率
  - 较小值 (0.1-0.2): 不确定性估计不足
  - 中等值 (0.2-0.5): 平衡不确定性估计
  - 较大值 (0.5-0.7): 不确定性估计过强
  - 推荐值: 0.2-0.5

- `n_samples`: 蒙特卡洛采样次数
  - 较小值 (10-50): 估计不稳定
  - 中等值 (50-100): 平衡估计和速度
  - 较大值 (100-500): 估计稳定，速度慢
  - 推荐值: 100

## 解释结果解读指南

### 不确定性解读

#### 1. 预测不确定性

```python
# 获取预测和不确定性
prediction, uncertainty = model.predict_with_uncertainty(query_point)

print(f"预测值: {prediction:.4f}")
print(f"不确定性: {uncertainty:.4f}")
print(f"95%置信区间: [{prediction - 1.96*uncertainty:.4f}, {prediction + 1.96*uncertainty:.4f}]")
```

**解读**：
- 不确定性越小，预测越可靠
- 置信区间越窄，预测越精确
- 高不确定性区域需要更多数据

#### 2. 不确定性类型分解

```python
# 获取认知和偶然不确定性
epistemic, aleatoric = model.decompose_uncertainty(query_point)

print(f"认知不确定性: {epistemic:.4f}")
print(f"偶然不确定性: {aleatoric:.4f}")
print(f"总不确定性: {epistemic + aleatoric:.4f}")
```

**解读**：
- 认知不确定性高：需要更多数据
- 偶然不确定性高：数据本身噪声大
- 两种不确定性都高：需要更多数据且数据质量差

#### 3. 不确定性空间分布

```python
# 不确定性空间分布
import matplotlib.pyplot as plt

fig, ax = plt.subplots(figsize=(10, 8))
sc = ax.scatter(query_points[:, 0], query_points[:, 1],
               c=uncertainties, s=100, alpha=0.6, cmap='Reds')
ax.set_xlabel('X Coordinate')
ax.set_ylabel('Y Coordinate')
ax.set_title('Uncertainty Spatial Distribution')
plt.colorbar(sc, label='Uncertainty')
plt.show()
```

**解读**：
- 高不确定性区域：数据稀缺或噪声大
- 低不确定性区域：数据充足且质量好
- 不确定性分布指导采样策略

## 使用示例

### 示例1：BNN不确定性估计

y_train = np.random.rand(1000)

# 2. 配置参数

    in_dim=5,
    hidden_dim=32,

posterior_samples = model.sample_posterior(X_test, n_samples=100)

## 常见问题解答

### Q1: 如何选择合适的不确定性量化方法？

**A**: 根据需求选择：

- **BNN**: 需要概率解释，理论基础扎实
- **MC Dropout**: 快速不确定性估计，易于部署
- **Deep Ensemble**: 高质量不确定性估计
- **EDL**: 分类任务，区分不确定性类型

### Q2: 如何评估不确定性量化质量？

**A**: 使用以下指标：

```python
# 校准误差
calibration_error = compute_calibration_error(predictions, uncertainties, y_true)

# CRPS
crps = compute_crps(predictions, uncertainties, y_true)

# 负对数似然
nll = compute_nll(predictions, uncertainties, y_true)
```

### Q3: 如何降低认知不确定性？

**A**: 使用主动学习：

```python
# 选择高认知不确定性区域
high_epistemic_indices = np.argsort(epistemic_uncertainty)[-10:]

# 在这些区域采集新数据
new_samples = query_points[high_epistemic_indices]
new_labels = collect_labels(new_samples)

# 重新训练模型
model.fit(np.vstack([X_train, new_samples]),
          np.hstack([y_train, new_labels]))
```

### Q4: 如何处理偶然不确定性？

**A**: 偶然不确定性无法通过更多数据减少，但可以：

1. **改进数据质量**：减少测量噪声
2. **使用不确定性感知损失**：
```python
def uncertainty_aware_loss(y_pred, y_true, uncertainty):
    return np.mean((y_pred - y_true)² / uncertainty²)
```

### Q5: 不确定性估计如何影响决策？

**A**: 基于不确定性进行风险决策：

```python
# 高不确定性区域：保守决策
if uncertainty > high_threshold:
    decision = conservative_decision()
# 低不确定性区域：激进决策
else:
    decision = aggressive_decision()
```
