# 不确定性量化模型参数说明

## 概述

本文档详细说明了UDAKE项目中所有不确定性量化模型的参数配置，包括模型初始化参数、训练参数和预测参数。每个模型都有其特定的参数集，用于控制模型的行为和性能。

---

## 1. 贝叶斯神经网络（BNN）

### 1.1 模型初始化参数

#### BayesianNeuralRegressor

| 参数名 | 类型 | 默认值 | 范围 | 说明 |
|--------|------|--------|------|------|
| `in_dim` | int | 必需 | ≥ 1 | 输入特征维度 |
| `hidden_dim` | int | 32 | ≥ 4 | 隐藏层维度 |
| `prior` | PriorType | GaussianPrior(sigma=1.0) | - | 先验分布类型（GaussianPrior或GaussianMixturePrior） |
| `seed` | int | 42 | 任意 | 随机种子，用于结果复现 |

#### GaussianPrior

| 参数名 | 类型 | 默认值 | 范围 | 说明 |
|--------|------|--------|------|------|
| `sigma` | float | 1.0 | > 0 | 高斯先验的标准差 |

#### GaussianMixturePrior

| 参数名 | 类型 | 默认值 | 范围 | 说明 |
|--------|------|--------|------|------|
| `sigmas` | tuple | (1.0, 0.1) | > 0 | 混合高斯的标准差元组 |
| `weights` | tuple | (0.5, 0.5) | [0, 1] | 混合高斯的权重（需归一化） |

### 1.2 训练参数

#### fit() 方法

| 参数名 | 类型 | 默认值 | 范围 | 说明 |
|--------|------|--------|------|------|
| `x` | np.ndarray | 必需 | - | 训练特征数据，形状为 (n_samples, in_dim) |
| `y` | np.ndarray | 必需 | - | 训练目标数据，形状为 (n_samples,) |
| `epochs` | int | 220 | ≥ 1 | 训练轮数 |
| `lr` | float | 8e-3 | > 0 | 学习率 |
| `kl_weight` | float | 0.05 | [0, 1] | KL散度权重，控制正则化强度 |
| `kl_anneal_epochs` | int | 20 | ≥ 0 | KL退火周期 |
| `nll_weight` | float | 1.0 | [0, 1] | 负对数似然权重 |

### 1.3 预测参数

#### predict() 方法

| 参数名 | 类型 | 默认值 | 范围 | 说明 |
|--------|------|--------|------|------|
| `x` | np.ndarray | 必需 | - | 预测特征数据 |
| `num_samples` | int | 80 | ≥ 2 | 蒙特卡洛采样次数 |
| `confidence` | float | 0.95 | (0, 1) | 置信水平 |
| `temperature` | float | 1.0 | ≥ 1e-4 | 温度参数，调节预测分布分散程度 |

#### predict_bnn() 方法（高级接口）

| 参数名 | 类型 | 默认值 | 范围 | 说明 |
|--------|------|--------|------|------|
| `features` | np.ndarray 或 list | 必需 | - | 原始特征数据 |
| `feature_names` | list[str] | None | - | 特征名称列表 |
| `num_samples` | int | 80 | ≥ 2 | 采样次数 |
| `confidence` | float | 0.95 | (0, 1) | 置信水平 |
| `temperature` | float | 1.0 | ≥ 1e-4 | 温度参数 |
| `use_training_stats` | bool | True | - | 是否使用训练时的统计量进行标准化 |

### 1.4 参数调优建议

#### 通用调优

- **hidden_dim**: 根据数据复杂度选择，建议范围 [16, 128]
- **lr**: 学习率过大导致不稳定，过小导致收敛慢，建议范围 [1e-4, 1e-2]
- **kl_weight**: 控制过拟合，数据量小时增大，建议范围 [0.01, 0.2]

#### 先验选择

- **GaussianPrior**: 适用于大多数场景，sigma建议范围 [0.5, 2.0]
- **GaussianMixturePrior**: 适用于复杂参数分布，建议使用默认配置

#### 采样参数

- **num_samples**: 增加次数提高估计精度，但增加计算成本，建议范围 [50, 200]
- **temperature**: T>1增加不确定性，T<1减少不确定性

## 2. MC Dropout

### 2.1 模型初始化参数

#### MCDropoutRegressor

| 参数名 | 类型 | 默认值 | 范围 | 说明 |
|--------|------|--------|------|------|
| `in_dim` | int | 必需 | ≥ 1 | 输入特征维度 |
| `hidden_dim` | int | 32 | ≥ 4 | 隐藏层维度 |
| `dropout_rate` | float | 0.2 | [0, 0.8] | Dropout比率 |
| `dropout_type` | DropoutType | "standard" | - | Dropout类型（"standard", "spatial", "variational"） |
| `seed` | int | 42 | 任意 | 随机种子 |

#### DropoutType 枚举值

| 值 | 说明 |
|----|------|
| "standard" | 标准Dropout，独立地随机丢弃神经元 |
| "spatial" | 空间Dropout，整个特征通道一起丢弃 |
| "variational" | 变分Dropout，使用固定的dropout mask |

### 2.2 训练参数

#### fit() 方法

| 参数名 | 类型 | 默认值 | 范围 | 说明 |
|--------|------|--------|------|------|
| `x` | np.ndarray | 必需 | - | 训练特征数据 |
| `y` | np.ndarray | 必需 | - | 训练目标数据 |
| `epochs` | int | 180 | ≥ 1 | 训练轮数 |
| `lr` | float | 8e-3 | > 0 | 学习率 |
| `nll_weight` | float | 0.4 | [0, 1] | 负对数似然权重（相对于MSE） |

### 2.3 预测参数

#### predict() 方法

| 参数名 | 类型 | 默认值 | 范围 | 说明 |
|--------|------|--------|------|------|
| `x` | np.ndarray | 必需 | - | 预测特征数据 |
| `num_samples` | int | 80 | ≥ 2 | Dropout采样次数 |
| `confidence` | float | 0.95 | (0, 1) | 置信水平 |

### 2.4 参数调优建议

#### Dropout配置

- **dropout_rate**: 控制正则化强度，建议范围 [0.1, 0.5]
  - 数据量小或容易过拟合时，增大dropout_rate
  - 数据量大或欠拟合时，减小dropout_rate

- **dropout_type**: 根据数据特性选择
  - "standard": 适用于大多数场景
  - "spatial": 适用于图像或序列数据
  - "variational": 适用于需要一致性的场景

#### 训练参数

- **nll_weight**: 平衡MSE和NLL损失，建议范围 [0.3, 0.7]
  - 增大nll_weight强调不确定性估计
  - 减小nll_weight强调预测精度

#### 采样参数

- **num_samples**: 建议范围 [50, 150]，过大会增加计算成本

## 3. Deep Ensemble（深度集成）

### 3.1 模型初始化参数

#### DeepEnsembleRegressor

| 参数名 | 类型 | 默认值 | 范围 | 说明 |
|--------|------|--------|------|------|
| `in_dim` | int | 必需 | ≥ 1 | 输入特征维度 |
| `n_members` | int | 5 | ≥ 2 | 集成成员数量 |
| `seed` | int | 42 | 任意 | 随机种子 |

### 3.2 训练参数

#### fit() 方法

| 参数名 | 类型 | 默认值 | 范围 | 说明 |
|--------|------|--------|------|------|
| `x` | np.ndarray | 必需 | - | 训练特征数据 |
| `y` | np.ndarray | 必需 | - | 训练目标数据 |
| `epochs` | int | 180 | ≥ 1 | 每个成员的训练轮数 |
| `data_mode` | str | "bootstrap" | - | 数据采样模式（"bootstrap"或"subsample"） |
| `hidden_dims` | list[int] | [24, 32, 40] | ≥ 4 | 成员隐藏层维度列表（循环使用） |
| `learning_rates` | list[float] | [8e-3, 6e-3, 1e-2] | > 0 | 成员学习率列表（循环使用） |

#### 数据采样模式

| 模式 | 说明 |
|------|------|
| "bootstrap" | 有放回采样，适合小数据集 |
| "subsample" | 无放回采样，使用80%的数据，适合大数据集 |

### 3.3 预测参数

#### predict() 方法

| 参数名 | 类型 | 默认值 | 范围 | 说明 |
|--------|------|--------|------|------|
| `x` | np.ndarray | 必需 | - | 预测特征数据 |
| `aggregation` | EnsembleAgg | "mean" | - | 聚合方式（"mean", "weighted", "median"） |
| `member_weights` | dict[str, float] | None | - | 成员权重字典（仅用于"weighted"聚合） |
| `confidence` | float | 0.95 | (0, 1) | 置信水平 |

#### select_members() 方法

| 参数名 | 类型 | 默认值 | 范围 | 说明 |
|--------|------|--------|------|------|
| `x_val` | np.ndarray | 必需 | - | 验证集特征数据 |
| `y_val` | np.ndarray | 必需 | - | 验证集目标数据 |
| `method` | SelectMethod | "validation" | - | 选择方法（"validation", "diversity", "adaptive"） |
| `top_k` | int | None | ≥ 1 | 选择的成员数量（None表示全部） |

#### 选择方法

| 方法 | 说明 |
|------|------|
| "validation" | 基于验证集NLL选择最佳成员 |
| "diversity" | 基于预测多样性选择成员 |
| "adaptive" | 综合验证分数和多样性 |

### 3.4 参数调优建议

#### 成员数量

- **n_members**: 根据计算资源和精度需求选择
  - 快速原型: 3-5
  - 标准应用: 5-10
  - 高精度需求: 10-20

#### 成员多样性

- **hidden_dims**: 使用不同维度增加多样性，建议 [16, 24, 32, 40, 48]
- **learning_rates**: 使用不同学习率增加多样性，建议 [4e-3, 6e-3, 8e-3, 1e-2]
- **data_mode**: 根据数据量选择，小数据集用"bootstrap"，大数据集用"subsample"

#### 聚合策略

- **mean**: 默认选择，简单有效
- **weighted**: 当成员性能差异较大时使用，需提供member_weights
- **median**: 对异常值更鲁棒

#### 成员选择

- **method**: 根据目标选择
  - 追求最高精度: "validation"
  - 追求最大多样性: "diversity"
  - 平衡精度和多样性: "adaptive"

## 4. 证据深度学习（EDL）

### 4.1 模型初始化参数

#### EDLClassifier

| 参数名 | 类型 | 默认值 | 范围 | 说明 |
|--------|------|--------|------|------|
| `in_dim` | int | 必需 | ≥ 1 | 输入特征维度 |
| `num_classes` | int | 必需 | ≥ 2 | 类别数量 |
| `hidden_dim` | int | 32 | ≥ 4 | 隐藏层维度 |
| `evidence_activation` | ActivationType | "softplus" | - | 证据激活函数（"softplus"或"relu"） |
| `seed` | int | 42 | 任意 | 随机种子 |

#### ActivationType 枚举值

| 值 | 说明 |
|----|------|
| "softplus" | 平滑激活，保证证据非负，推荐使用 |
| "relu" | 稀疏激活，可能产生更多不确定性 |

### 4.2 训练参数

#### fit() 方法

| 参数名 | 类型 | 默认值 | 范围 | 说明 |
|--------|------|--------|------|------|
| `x` | np.ndarray | 必需 | - | 训练特征数据 |
| `y` | np.ndarray | 必需 | - | 训练标签数据（整数类别） |
| `epochs` | int | 220 | ≥ 1 | 训练轮数 |
| `lr` | float | 8e-3 | > 0 | 学习率 |
| `evidence_weight` | float | 0.4 | [0, 1] | EDL损失权重 |
| `kl_weight` | float | 0.05 | [0, 1] | KL正则化权重 |

### 4.3 预测参数

#### predict() 方法

| 参数名 | 类型 | 默认值 | 范围 | 说明 |
|--------|------|--------|------|------|
| `x` | np.ndarray | 必需 | - | 预测特征数据 |
| `confidence` | float | 0.95 | (0, 1) | 置信水平 |

### 4.4 参数调优建议

#### 激活函数选择

- **evidence_activation**: 
  - "softplus": 平滑，推荐用于大多数场景
  - "relu": 可能产生更高的不确定性估计

#### 损失权重

- **evidence_weight**: 控制EDL损失的影响，建议范围 [0.2, 0.6]
  - 增大以强调不确定性估计
  - 减小以强调分类精度

- **kl_weight**: 控制正则化强度，建议范围 [0.01, 0.1]
  - 数据量小时增大
  - 数据量大时减小

#### 网络结构

- **hidden_dim**: 根据数据复杂度选择，建议范围 [16, 128]
- **num_classes**: 必须等于实际类别数量

## 5. 通用参数

### 5.1 数据预处理

所有模型都支持自动的数据标准化：

| 参数 | 说明 |
|------|------|
| 特征标准化 | (x - mean) / std |
| 最小标准差保护 | std ≥ 1e-8 |
| 统计量来源 | 训练集统计量或当前批次统计量 |

### 5.2 数值稳定性

| 参数 | 默认值 | 说明 |
|------|--------|------|
| logvar下界 | -8.0 | 对数方差的最小值 |
| logvar上界 | 5.0 | 对数方差的最大值 |
| 最小方差 | 1e-6 | 方差的最小值 |
| 最小温度 | 1e-4 | 温度参数的最小值 |

### 5.3 缓存机制

| 参数 | 默认值 | 说明 |
|------|--------|------|
| 预测缓存大小 | 24 | 缓存预测结果的数量 |
| 批量缓存大小 | 16 | 缓存批量预测结果的数量 |

### 5.4 置信区间

置信区间使用正态分布近似：

```
[mean - z * sqrt(variance), mean + z * sqrt(variance)]
```

其中z是标准正态分布的分位数，由confidence参数决定。

## 6. 参数选择流程图

### 6.1 模型选择

```
开始
  ↓
需要分类吗？
  ├─ 是 → EDL
  └─ 否 → 回归任务
          ↓
        数据量如何？
        ├─ 小（<1000） → BNN
        ├─ 中（1000-10000） → MC Dropout
        └─ 大（>10000） → Deep Ensemble
```

### 6.2 参数调优流程

1. **初始配置**: 使用默认参数
2. **训练验证**: 在验证集上评估性能
3. **性能分析**:
   - 预测精度不足 → 调整网络结构（hidden_dim）
   - 不确定性估计不准 → 调整不确定性相关参数
   - 过拟合 → 增加正则化（kl_weight, dropout_rate）
   - 欠拟合 → 减少正则化，增加模型容量
4. **迭代优化**: 重复步骤2-3直到满意

## 7. 常见问题

### 7.1 参数设置问题

**Q: 如何选择合适的num_samples？**

A: num_samples影响不确定性估计的精度和计算成本：
- 快速原型: 30-50
- 标准应用: 50-100
- 高精度需求: 100-200

**Q: temperature参数应该如何设置？**

A: temperature调节预测分布的分散程度：
- T = 1.0: 默认行为
- T > 1.0: 增加不确定性估计
- T < 1.0: 减少不确定性估计

**Q: KL权重如何调整？**

A: KL权重控制正则化强度：
- 数据量小或容易过拟合: 增大KL权重
- 数据量大或欠拟合: 减小KL权重

### 7.2 性能优化问题

**Q: 如何提高预测速度？**

A: 
- 减少num_samples
- 使用缓存机制
- 对于Deep Ensemble，减少n_members
- 使用批量预测

**Q: 如何提高不确定性估计质量？**

A:
- 增加num_samples
- 使用更合适的先验分布
- 调整KL权重
- 对于Deep Ensemble，增加成员数量

### 7.3 模型选择问题

**Q: 哪种模型最适合我的场景？**

A: 根据以下因素选择：
- 数据量: 小数据集用BNN，大数据集用Deep Ensemble
- 计算资源: 资源有限用MC Dropout，资源充足用Deep Ensemble
- 不确定性类型: 需要同时量化认知和偶然不确定性用BNN或MC Dropout
- 任务类型: 分类用EDL，回归用其他模型
