# 不确定性量化模型特征分析

## 1. 目标

统一 BNN、MC Dropout、Deep Ensemble、EDL 四类不确定性模型的特征定义、命名映射和提取/分解方案，便于 LIME/SHAP 与可视化面板复用。

## 2. 模型特征列表

### 2.1 BNN（Bayesian Neural Network）

- `mean`：预测均值
- `variance`：总预测方差
- `aleatoric`：偶然不确定性
- `epistemic`：认知不确定性
- `lower` / `upper`：置信区间
- `confidence`：置信度
- `num_samples`：采样次数

### 2.2 MC Dropout

- `mean`：预测均值
- `variance`：总预测方差
- `aleatoric`：偶然不确定性
- `epistemic`：认知不确定性
- `lower` / `upper`：置信区间
- `confidence`：置信度
- `t`：蒙特卡洛采样次数

### 2.3 Deep Ensemble

- `mean`：集成均值预测
- `variance`：总预测方差
- `aleatoric`：成员内不确定性
- `epistemic`：成员间不确定性
- `lower` / `upper`：置信区间
- `quantiles.q10` / `quantiles.q50` / `quantiles.q90`：分位数特征
- `member_count`：集成成员数量
- `aggregation`：聚合方式（`mean`/`weighted`/`median`）

### 2.4 EDL（Evidential Deep Learning）

- `logits`：分类 logits
- `evidence`：证据强度
- `alpha`：Dirichlet 参数
- `probabilities`：类别概率
- `prediction`：预测类别
- `confidence`：置信度
- `uncertainty.total`：总不确定性
- `uncertainty.data`：数据不确定性
- `uncertainty.knowledge`：知识不确定性
- `uncertainty.threshold`：阈值

## 3. 特征名称映射表

统一在 `deep_learning/models/uncertainty/features.py` 的 `feature_name_mapping()` 中维护，覆盖核心键：

- `mean` -> 预测均值
- `variance` -> 总预测方差
- `aleatoric` -> 偶然不确定性
- `epistemic` -> 认知不确定性
- `quantiles.q10` / `q50` / `q90` -> 分位数预测
- `uncertainty.total` / `data` / `knowledge` -> EDL不确定性分量

## 4. 不确定性特征提取方案

采用统一接口 `extract_model_features(model_type, prediction)`：

1. 按模型类型读取标准特征键集合  
2. 自动处理嵌套字段（如 `quantiles.q10`、`uncertainty.total`）  
3. 数值特征统一转为 `np.ndarray`  
4. 补齐跨模型别名（如 EDL 的 `knowledge` -> `epistemic`）  
5. 输出可直接用于解释与可视化的扁平特征字典

## 5. 认知/偶然不确定性区分方案

采用统一接口 `decompose_uncertainty_sources(model_type, prediction)`：

- BNN/MC Dropout/Deep Ensemble：直接使用模型输出 `epistemic` 与 `aleatoric`
- EDL：`knowledge` 映射为 `epistemic`，`data` 映射为 `aleatoric`
- 统一计算：
  - `total`
  - `epistemic_ratio`
  - `aleatoric_ratio`
- 保障异常场景下的稳健性（缺失项自动补齐，值域裁剪为非负）

运行示例：

```bash

```

示例输出包含四类模型（`bnn`、`mc_dropout`、`deep_ensemble`、`edl`）的：

- 标准化特征键集合（`feature_keys`）
- 特征中文名称映射（`feature_labels`）
- 认知/偶然不确定性分解结果（`decomposition`）
