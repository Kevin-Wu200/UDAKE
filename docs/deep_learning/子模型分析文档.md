# 子模型分析文档

## 概述

子模型分析是融合模型系统的重要组成部分，用于评估、监控和优化参与融合的各个子模型。通过系统的分析，我们可以理解每个子模型的贡献度、稳定性和重要性，从而优化融合策略，提高整体预测性能。

## 分析目标

1. **性能评估**：量化每个子模型的预测性能
2. **贡献度分析**：评估每个子模型对融合结果的贡献
3. **稳定性分析**：评估子模型预测的一致性和稳定性
4. **多样性分析**：评估子模型间的差异和互补性
5. **优化建议**：提供子模型选择和权重优化建议

## 核心分析指标

### 1. 基础特征

#### 1.1 模型数量（model_count）
- **定义**：参与融合的子模型总数
- **意义**：反映融合系统的规模和复杂度
- **范围**：≥ 2
- **理想值**：根据场景，通常 3-10 个模型

#### 1.2 预测步长（prediction_horizon）
- **定义**：每个模型的预测序列长度
- **意义**：反映预测任务的规模
- **范围**：≥ 1
- **影响**：影响计算开销和内存使用

#### 1.3 模型分歧均值（disagreement_mean）
- **定义**：各子模型预测的标准差平均值
- **计算**：`mean(std(model_predictions))`
- **意义**：反映模型间预测的一致性
- **范围**：≥ 0
- **理想值**：适中（过高说明模型差异大，过低说明冗余）

#### 1.4 模型分歧标准差（disagreement_std）
- **定义**：各子模型预测标准差的标准差
- **计算**：`std(std(model_predictions))`
- **意义**：反映分歧程度的变化
- **范围**：≥ 0
- **理想值**：较低（说明分歧程度稳定）

### 2. 权重特征

#### 2.1 权重熵（weight_entropy）
- **定义**：权重分布的熵值
- **计算**：
  ```
  H(w) = -Σ w_i * log(w_i) / log(N)
  ```
  其中 N 是模型数量
- **意义**：反映权重分布的均匀程度
- **范围**：[0, 1]
- **解读**：
  - 0：所有权重集中在一个模型
  - 1：所有模型权重相等
- **理想值**：根据策略，通常 0.3-0.7

#### 2.2 权重基尼系数（weight_gini）
- **定义**：权重分布的不平等程度
- **计算**：
  ```
  G = (2 * Σ i * w_(i)) / (N * Σ w_i) - (N + 1) / N
  ```
  其中 w_(i) 是排序后的权重
- **意义**：反映权重分配的不均匀性
- **范围**：[0, 1]
- **解读**：
  - 0：完全均匀分配
  - 1：完全不均匀分配
- **理想值**：根据策略，通常 0.2-0.5

#### 2.3 最大权重占比（weight_top1_ratio）
- **定义**：最大权重占总权重的比例
- **计算**：`max(weights)`
- **意义**：反映主模型的主导地位
- **范围**：[1/N, 1]
- **解读**：
  - 高值（>0.5）：依赖单个模型
  - 中值（0.3-0.5）：平衡融合
  - 低值（<0.3）：均匀融合
- **理想值**：根据策略，通常 0.2-0.4

#### 2.4 有效模型数（weight_effective_models）
- **定义**：权重分布的有效模型数量
- **计算**：`1 / Σ w_i²`
- **意义**：反映实际参与融合的模型数量
- **范围**：[1, N]
- **解读**：
  - 接近 1：依赖单个模型
  - 接近 N：所有模型均衡贡献
- **理想值**：≥ 2

### 3. 模型贡献指标

#### 3.1 平均绝对贡献（mean_abs_contribution）
- **定义**：子模型对融合输出的平均绝对贡献
- **计算**：`mean(|w_i * (pred_i - fused)|)`
- **意义**：反映子模型对融合结果的影响幅度
- **范围**：≥ 0
- **解读**：
  - 高值：对融合结果影响大
  - 低值：对融合结果影响小
- **理想值**：根据权重，通常与权重成正比

#### 3.2 平均符号贡献（mean_signed_contribution）
- **定义**：子模型对融合输出的平均符号贡献
- **计算**：`mean(w_i * (pred_i - fused))`
- **意义**：反映子模型对融合结果的系统性偏差
- **范围**：(-∞, +∞)
- **解读**：
  - 正值：倾向于拉高融合结果
  - 负值：倾向于拉低融合结果
  - 接近 0：无系统性偏差
- **理想值**：接近 0

#### 3.3 贡献稳定性（stability）
- **定义**：子模型贡献的时序稳定性
- **计算**：`1 / (1 + std(|w_i * (pred_i - fused)|))`
- **意义**：反映子模型贡献的一致性
- **范围**：(0, 1]
- **解读**：
  - 接近 1：贡献稳定
  - 接近 0：贡献波动大
- **理想值**：> 0.7

#### 3.4 与融合结果的一致性（agreement_with_fusion）
- **定义**：子模型预测与融合结果的相关性
- **计算**：`corr(pred_i, fused)`
- **意义**：反映子模型与整体融合的一致程度
- **范围**：[-1, 1]
- **解读**：
  - 接近 1：高度一致
  - 接近 0：无明显相关
  - 接近 -1：相反趋势
- **理想值**：> 0.5

#### 3.5 边际收益（marginal_gain）
- **定义**：去掉该模型后误差的变化
- **计算**：`mean(|fused_without_i - true|) - mean(|fused - true|)`
- **意义**：反映子模型的实际价值
- **范围**：(-∞, +∞)
- **解读**：
  - 正值：该模型降低误差（有用）
  - 负值：该模型增加误差（有害）
  - 接近 0：无显著影响
- **理想值**：> 0

### 4. 策略特征

#### 4.1 融合层级（fusion_level）
- **决策层融合**（decision）：简单平均、加权平均等
- **元学习融合**（meta_learning）：堆叠融合
- **概率融合**（probabilistic）：贝叶斯模型平均
- **不确定性感知**（uncertainty_aware）：方差加权
- **自适应融合**（adaptive）：动态融合

#### 4.2 策略复杂度（complexity）
- **低**（low）：简单平均、加权平均、中位数、最大最小
- **中**（medium）：堆叠融合、贝叶斯模型平均、方差加权
- **高**（high）：动态融合

#### 4.3 动态权重波动性（dynamic_weight_volatility）
- **定义**：动态融合中权重的时序波动
- **计算**：每个模型权重序列的标准差
- **意义**：反映权重调整的稳定性
- **范围**：≥ 0
- **解读**：
  - 低值：权重稳定
  - 高值：权重频繁调整
- **理想值**：适中（根据数据分布变化）

## 分析流程

### 1. 数据准备

```python
from deep_learning.fusion import FusionFeatureAnalyzer

# 准备模型预测
models = [
    ModelPrediction(
        model_id="model_1",
        predictions=[...],
        variances=[...]
    ),
    ModelPrediction(
        model_id="model_2",
        predictions=[...],
        variances=[...]
    ),
    # ... 更多模型
]

# 准备权重和融合结果
weights = {
    "model_1": 0.4,
    "model_2": 0.3,
    # ... 更多权重
}

fused_predictions = [...]
true_values = [...]  # 可选

# 创建分析器
analyzer = FusionFeatureAnalyzer()
```

### 2. 执行分析

```python
# 执行分析
analysis = analyzer.analyze(
    models=models,
    weights=weights,
    strategy="weighted_average",
    weight_method="rmse_based",
    fused_predictions=fused_predictions,
    true_values=true_values
)
```

### 3. 结果解读

```python
# 基础特征
basic = analysis["basic_features"]
print(f"模型数量: {basic['model_count']}")
print(f"模型分歧均值: {basic['disagreement_mean']:.4f}")

# 权重特征
weight_features = analysis["weight_features"]
print(f"权重熵: {weight_features['weight_entropy']:.4f}")
print(f"有效模型数: {weight_features['weight_effective_models']:.2f}")

# 模型贡献
for contrib in analysis["model_contributions"]:
    model_id = contrib["model_id"]
    marginal_gain = contrib["marginal_gain"]
    stability = contrib["stability"]
    print(f"{model_id}: 边际收益={marginal_gain:.4f}, 稳定性={stability:.4f}")
```

### 1. 模型选择建议

#### 保留高价值模型

```python
    m for m in analysis["model_contributions"]
    if m["marginal_gain"] > 0
]
```

#### 剔除低贡献模型
```python
# 剔除边际收益为负的模型

    m for m in analysis["model_contributions"]

]
```

#### 保留稳定模型
```python
# 保留高稳定性模型
stable_models = [
    m for m in analysis["model_contributions"]
    if m["stability"] > 0.7
]
```

### 2. 权重优化建议

#### 基于边际收益调整权重
```python
# 提高高边际收益模型的权重
def optimize_weights_by_marginal_gain(contributions):
    total_gain = sum(max(0, c["marginal_gain"]) for c in contributions)
    if total_gain == 0:
        return {c["model_id"]: 1.0/len(contributions) for c in contributions}

    weights = {}
    for c in contributions:
        gain = max(0, c["marginal_gain"])
        weights[c["model_id"]] = gain / total_gain
    return weights
```

#### 基于稳定性调整权重
```python
# 提高高稳定性模型的权重
def optimize_weights_by_stability(contributions):
    total_stability = sum(c["stability"] for c in contributions)
    weights = {}
    for c in contributions:
        weights[c["model_id"]] = c["stability"] / total_stability
    return weights
```

### 3. 多样性优化

#### 检查模型冗余
```python
# 检查高相关性模型
high_corr_pairs = []
contribs = analysis["model_contributions"]
for i in range(len(contribs)):
    for j in range(i+1, len(contribs)):
        corr = np.corrcoef(
            models[i].predictions,
            models[j].predictions
        )[0, 1]
        if abs(corr) > 0.9:
            high_corr_pairs.append(
                (contribs[i]["model_id"], contribs[j]["model_id"], corr)
            )
```

#### 保留多样性模型
```python
# 保留低相关性模型组合
def select_diverse_models(models, threshold=0.8):
    selected = []
    for model in models:
        is_diverse = True
        for selected_model in selected:
            corr = np.corrcoef(
                model.predictions,
                selected_model.predictions
            )[0, 1]
            if abs(corr) > threshold:
                is_diverse = False
                break
        if is_diverse:
            selected.append(model)
    return selected
```

## 常见问题

### Q1: 如何判断一个模型是否需要保留？
**答案**：综合以下指标：
- 边际收益 > 0
- 稳定性 > 0.7
- 与融合结果一致性 > 0.5
- 与其他模型相关性 < 0.9

### Q2: 权重熵过高或过低怎么办？
**答案**：
- 过高（>0.8）：考虑增加权重约束，突出高性能模型
- 过低（<0.2）：考虑引入更多模型或降低权重约束

### Q3: 如何处理模型分歧过大的问题？
**答案**：
- 检查模型训练数据是否一致
- 检查模型超参数设置
- 考虑使用中位数或最大最小策略
- 剔除异常模型

### Q4: 如何评估融合效果？
**答案**：
- 比较融合前后性能指标（RMSE、MAE、R²）
- 计算改进百分比
- 分析不确定性估计的准确性
- 评估融合结果的稳定性

### Q5: 如何选择最优模型组合？
**答案**：
- 使用特征选择算法（如LASSO、RFE）
- 基于边际收益和稳定性筛选
- 考虑模型多样性和互补性
- 通过交叉验证验证组合效果
