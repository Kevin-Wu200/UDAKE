## 1. 简介

### 1.1 概述

UDAKE 深度学习模块提供了四种异常检测模型，用于检测空间插值数据中的异常样本：

- **VAE (变分自编码器)**：基于重建误差和潜在距离的异常检测
- **GCAE (图卷积自编码器)**：基于图结构的异常检测
- **GAN (生成对抗网络)**：基于生成器和判别器的异常检测
- **对比学习 (Contrastive Learning)**：基于特征表示的异常检测

### 1.2 模型特点

| 模型 | 核心思想 | 优势 | 劣势 |
|-----|---------|------|------|
| VAE | 重建误差 + 潜在距离 | 简单高效，可解释性强 | 对复杂模式可能欠拟合 |
| GCAE | 图结构分析 | 考虑空间上下文，适合图数据 | 计算开销大，需要图构建 |
| GAN | 生成对抗 | 判别力强，适合分布外检测 | 训练不稳定，可能模式崩溃 |
| 对比学习 | 特征表示学习 | 支持在线学习，适应性强 | 需要足够的正常样本 |

### 1.3 适用场景

- **VAE**：快速原型、简单模式、需要可视化
- **GCAE**：空间数据、网络数据、需要考虑拓扑
- **GAN**：高判别需求、分布外检测、生成需求
- **对比学习**：在线学习、概念漂移、孤立异常

## 2. 模型对比

### 2.1 技术对比

#### 2.1.1 架构对比

| 特性 | VAE | GCAE | GAN | 对比学习 |
|-----|-----|------|-----|---------|
| **输入** | 坐标 + 值 | 坐标 + 值 + 图 | 坐标 + 值 | 坐标 + 值 + 增强 |
| **核心组件** | 编码器-解码器 | GCN + GAT + EdgeConv | 生成器 + 判别器 | 编码器 + 投影器 + 特征库 |
| **潜在空间** | 是 (4维) | 是 (4维) | 否 (生成器) | 是 (8维) |
| **训练方式** | 自监督 | 自监督 | 对抗训练 | 自监督 |
| **损失函数** | 重建 + KL | 特征重建 + 结构 | 生成器 + 判别器 + GP | SimCLR + MoCo + BYOL |

#### 2.1.2 异常分数对比

| 模型 | 主要分数 | 次要分数 | 综合方法 |
|-----|---------|---------|---------|
| VAE | 重建误差 | 潜在距离 | 加权组合 (0.7 + 0.3) |
| GCAE | 节点分数 | 边分数、子图分数 | 主要使用节点分数 |
| GAN | 判别器分数 | 重建误差、梯度分数 | 加权组合 (0.5 + 0.35 + 0.15) |
| 对比学习 | 特征距离 | 密度、最近邻、特征库相似度 | 加权组合 (0.4 + 0.3 + 0.2 + 0.1) |

#### 2.1.3 性能对比

| 指标 | VAE | GCAE | GAN | 对比学习 |
|-----|-----|------|-----|---------|
| **训练速度** | 快 | 中等 | 慢 | 中等 |
| **预测速度** | 快 | 中等 | 快 | 快 |
| **内存占用** | 低 | 中等 | 中等 | 中等 |
| **准确率** | 中等 | 高 | 高 | 高 |
| **稳定性** | 高 | 中等 | 低 | 高 |
| **可解释性** | 高 | 中等 | 中等 | 中等 |

### 2.2 使用场景对比

#### 2.2.1 数据类型

| 数据类型 | 推荐模型 | 原因 |
|---------|---------|------|
| 简单空间数据 | VAE | 快速高效，足够准确 |
| 复杂空间数据 | GCAE | 考虑空间上下文 |
| 需要生成 | GAN | 可以生成正常样本 |
| 流数据 | 对比学习 | 支持在线更新 |
| 网络数据 | GCAE | 天然适合图结构 |
| 传感器网络 | GCAE 或 对比学习 | 考虑空间关系和在线学习 |

#### 2.2.2 异常类型

| 异常类型 | 推荐模型 | 原因 |
|---------|---------|------|
| 点异常 | 所有模型 | 都能检测孤立异常 |
| 上下文异常 | GCAE | 考虑邻居信息 |
| 集群异常 | GCAE 或 对比学习 | 考虑局部密度 |
| 分布外异常 | GAN 或 对比学习 | 对新分布敏感 |
| 模式变化 | 对比学习 | 在线学习适应 |

### 2.3 参数复杂度对比

| 模型 | 关键参数数 | 调参难度 | 推荐配置 |
|-----|-----------|---------|---------|
| VAE | 5 | 低 | latent_dim=4, beta=0.1, max_epochs=40 |
| GCAE | 5 | 中 | latent_dim=4, knn_k=8, structure_weight=0.4 |
| GAN | 4 | 高 | latent_dim=6, max_epochs=60, gp_weight=10 |
| 对比学习 | 5 | 中 | feature_dim=12, projection_dim=8, temperature=0.2 |

```python
# 实现早停
def train_with_early_stopping(detector, coords, values, patience=5):
    best_loss = float('inf')
    patience_counter = 0
    
    for epoch in range(100):  # 最大轮数
        detector.fit(coords, values)
        
        current_loss = detector.history[-1]['total_loss']
        
        if current_loss < best_loss:
            best_loss = current_loss
            patience_counter = 0
        else:
```

```python
class CachedDetector:
    def __init__(self, detector):
        self.detector = detector
        self.cache = {}
    
    def predict(self, coords, values):
        # 创建缓存键
        key = tuple(coords.tobytes()) + tuple(values.tobytes())
        
        if key in self.cache:
            return self.cache[key]
        
        # 预测

        # 缓存结果
        self.cache[key] = result
        
        return result
```

```python
# 减少模型大小
def compress_model(detector):
    """减少模型内存占用"""
    # 降低精度
    for attr in dir(detector):
        if isinstance(getattr(detector, attr), np.ndarray):
            arr = getattr(detector, attr)
            setattr(detector, attr, arr.astype(np.float32))
    
    return detector
```

## 7. 常见问题

### 7.1 模型选择

**Q: 如何选择最适合的模型？**

A: 参考 [模型选择指南](#4-模型选择指南)，考虑：
1. 数据特征（大小、分布、空间关系）
2. 需求（速度、准确率、可解释性）
3. 应用场景（在线学习、流数据、静态数据）

**Q: 可以同时使用多个模型吗？**

A: 可以，推荐使用集成方法：
1. 简单集成：投票或平均
2. 加权集成：根据性能加权
3. 堆叠集成：学习最优组合
4. 动态集成：根据数据选择模型

**Q: 哪个模型最准确？**

A: 没有绝对的最准确，取决于数据：
- 简单数据：VAE 足够
- 复杂数据：GCAE 或 GAN
- 流数据：对比学习
- 建议在验证集上比较

### 7.2 训练问题

**Q: 训练很慢怎么办？**

A: 优化方法：
1. 减少数据量（采样）
2. 降低模型复杂度（维度）
3. 减少训练轮数
4. 使用更快的硬件

**Q: 训练不收敛怎么办？**

A: 检查：
1. 数据质量（NaN/Inf）
2. 数据量是否足够
3. 参数设置是否合理
4. 尝试不同的随机种子

**Q: 如何判断训练是否完成？**

A: 检查：
1. 损失是否稳定
2. 验证集性能是否提升
3. 早停策略
4. 固定训练轮数

### 7.3 预测问题

**Q: 预测结果不稳定？**

A: 解决方法：
1. 设置固定随机种子
2. 使用更大的数据集
3. 使用集成方法
4. 平滑预测结果

**Q: 检测到的异常太多？**

A: 降低异常数：
1. 提高阈值（percentile）
2. 调整分数权重
3. 检查数据质量
4. 使用后处理规则

**Q: 检测不到异常？**

A: 增加检测：
1. 降低阈值
2. 使用更敏感的模型
3. 检查模型是否训练充分
4. 确认数据确实有异常

### 7.4 性能问题

**Q: 如何提高准确率？**

A: 改进方法：
1. 使用更好的模型
2. 调整参数
3. 增加训练数据
4. 使用集成方法

**Q: 如何提高速度？**

A: 加速方法：
1. 使用更简单的模型
2. 减少数据量
3. 使用批量处理
4. 使用并行计算

**Q: 如何降低内存占用？**

A: 减少内存：
1. 降低模型维度
2. 使用数据采样
3. 压缩模型
4. 使用增量训练

### B. 性能基准测试

```python
# 性能基准测试脚本
import time
import numpy as np
from deep_learning.models.anomaly_detection import *

def benchmark_model(model_class, coords, values, **kwargs):
    """基准测试单个模型"""
    # 训练
    start = time.time()
    model = model_class(**kwargs)
    model.fit(coords, values)
    train_time = time.time() - start
    
    # 预测
    start = time.time()
    result = model.predict(coords, values)
    predict_time = time.time() - start
    
    return {
        'train_time': train_time,
        'predict_time': predict_time,
        'anomaly_count': result['anomaly_count']
    }

# 运行基准测试
models = {
    'VAE': (VAEAnomalyDetector, {}),
    'GCAE': (GCAEAnomalyDetector, {}),
    'GAN': (GANAnomalyDetector, {}),
    'Contrastive': (ContrastiveAnomalyDetector, {'epochs': 30})
}

for name, (model_class, kwargs) in models.items():
    result = benchmark_model(model_class, coords, values, **kwargs)
    print(f"{name:12s}: 训练={result['train_time']:.2f}s, "
          f"预测={result['predict_time']:.2f}s, "
          f"异常={result['anomaly_count']}")
```
