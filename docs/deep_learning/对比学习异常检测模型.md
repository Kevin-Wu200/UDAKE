# 对比学习异常检测模型文档

## 1. 模型概述

### 1.1 简介

对比学习 (Contrastive Learning) 异常检测器是一种基于自监督学习的异常检测方法。该模型通过学习数据表示，使正常样本在特征空间中更相似，异常样本与正常样本更不同。支持 SimCLR、MoCo、BYOL 等对比学习风格的近似实现。

### 1.2 工作原理

对比学习异常检测器的核心思想：

1. **数据增强**：
   - 空间增强：坐标旋转、缩放
   - 值增强：添加噪声
   - 掩码策略：部分值替换为中位数

2. **特征编码**：
   - 编码器：将输入映射到特征空间
   - 投影器：将特征映射到对比空间
   - 使用 L2 归一化确保稳定性

3. **对比学习目标**：
   - **SimCLR**：最大化正样本对相似度，最小化负样本对相似度
   - **MoCo**：使用队列存储负样本
   - **BYOL**：通过自蒸馏学习表示

4. **特征库更新**：
   - 在线更新特征库
   - 保持最近 N 个样本的表示

5. **异常检测**：
   - 特征空间距离：样本与特征库中心的距离
   - 密度估计：基于 kNN 的密度
   - 最近邻分析：与最近样本的相似度
   - 特征库相似度：与特征库的最大相似度
   - 综合分数：四者的加权组合

### 1.3 网络架构

```
训练阶段:
输入: 坐标 + 值
  ↓
[数据增强]:
  - 空间增强: 旋转、缩放
  - 值增强: 添加噪声
  - 掩码策略: 部分替换
  ↓
[特征构建]:
  - coord_x, coord_y
  - radius, angle
  - value, value_centered
  ↓
[编码器]: 特征 → 隐藏特征 (feature_dim=12)
  ↓
[投影器]: 隐藏特征 → 投影特征 (projection_dim=8)
  ↓
[归一化]: L2 归一化
  ↓
[对比损失]:
  - SimCLR 损失
  - MoCo 损失
  - BYOL 损失
  ↓
[特征库更新]: 队列管理 (queue_size=256)

推理阶段:
输入: 新坐标 + 新值
  ↓
[特征构建]
  ↓
[编码器 + 投影器]
  ↓
[计算]:
  - 特征距离 = ||z - 特征库中心||
  - 密度 = 1 - mean(top-k 相似度)
  - 最近邻 = 1 - max(相似度)
  - 特征库相似度 = max(相似度)
  ↓
综合分数 = 0.4×特征距离 + 0.3×密度 + 0.2×最近邻 + 0.1×(1-特征库相似度)
```

### 1.4 损失函数

**SimCLR 损失**：
```
L_SimCLR = -E[log(exp(sim(z_i, z_j)/τ) / Σ_k exp(sim(z_i, z_k)/τ))]
```
- z_i, z_j：正样本对
- z_k：所有样本（包括负样本）
- τ：温度参数 (0.2)

**MoCo 损失**：
```
L_MoCo = -E[log(exp(sim(z_i, z_pos)/τ) / Σ_k exp(sim(z_i, z_k)/τ))]
```
- z_pos：正样本
- 队列提供负样本

**BYOL 损失**：
```
L_BYOL = E[||z_1 - z_2||²]
```
- z_1, z_2：在线和目标网络的输出

**总损失**：
```
L_total = 0.4 × L_SimCLR + 0.35 × L_MoCo + 0.25 × L_BYOL
```

### 1.5 适用场景

- 适合检测偏离正常簇的异常
- 适合检测孤立异常
- 适合需要在线学习的场景
- 适合数据分布随时间变化的情况
- 不适合异常比例很高的情况

## 2. LIME 解释指南

### 2.1 概述

对于对比学习模型，LIME 解释关注特征对综合异常分数的贡献。

### 2.2 关键特征

对比学习模型使用以下特征：

- **空间特征**：coord_x, coord_y, radius, angle
- **值特征**：value, value_centered
- **特征距离**：feature_distance
- **密度分数**：density_score
- **最近邻分数**：nearest_score
- **特征库相似度**：bank_similarity

### 2.3 实现步骤

```python
import numpy as np

# 2. 预处理数据

    coords, values,
    batch_size=64,
    use_training_stats=True,
```

### 2.4 解释结果解读

## 4. 异常分数解释说明

### 4.1 分数组成

对比学习模型提供四层异常分数：

```
综合分数 = 0.4 × 特征距离 + 0.3 × 密度 + 0.2 × 最近邻 + 0.1 × (1 - 特征库相似度)
```

### 4.2 分数详解

#### 4.2.1 特征距离

**定义**：样本嵌入与特征库中心的欧氏距离

**计算**：
```python
center = feature_bank.mean(axis=0)
feat_dist = ||z - center||
```

**意义**：
- 高值：样本在特征空间远离正常分布中心
- 低值：样本位于特征空间中心附近

**典型范围**：0 ~ ∞（通常 0-3）

**特点**：
- 基于特征库的分布
- 捕捉全局异常

#### 4.2.2 密度估计

**定义**：基于 kNN 的密度估计

**计算**：
```python
sim = z @ feature_bank.T  # 余弦相似度
order = sort(sim, axis=1)
k = min(5, order.shape[1])
density = 1 - mean(order[:, -k:])  # 1 - 平均 top-k 相似度
```

**意义**：
- 高值：样本周围相似样本少，密度低
- 低值：样本周围相似样本多，密度高

**典型范围**：0 ~ 1

**特点**：
- 基于局部邻域
- 捕捉孤立异常

#### 4.2.3 最近邻分数

**定义**：1 - 与最近样本的相似度

**计算**：
```python
nearest = 1 - max(sim, axis=1)
```

**意义**：
- 高值：与最近的样本也不相似
- 低值：有非常相似的样本

**典型范围**：0 ~ 1

**特点**：
- 捕捉最近邻异常
- 对孤立点敏感

#### 4.2.4 特征库相似度

**定义**：与特征库的最大相似度

**计算**：
```python
bank_similarity = max(sim, axis=1)
```

**意义**：
- 高值：与特征库中某些样本很相似
- 低值：与特征库中所有样本都不相似

**典型范围**：0 ~ 1

**特点**：
- 反映样本是否见过类似模式
- 在异常分数中取反使用

#### 4.2.5 综合分数

**定义**：四个分数的加权组合

**计算**：
```python
combined = 0.4 × 归一化(feat_dist) 
         + 0.3 × density 
         + 0.2 × nearest 
         + 0.1 × (1 - bank_similarity)
```

**意义**：
- 结合全局和局部信息
- 考虑距离、密度、相似度
- 全面的异常评估

**典型范围**：0 ~ 1

**权重说明**：
- 特征距离最大 (0.4)：全局分布最重要
- 密度次之 (0.3)：局部密度很重要
- 最近邻 (0.2)：补充信息
- 特征库相似度 (0.1)：辅助判断

### 4.3 阈值确定

支持多种阈值方法（与其他模型相同）：

1. **百分位数法**：默认 percentile=95.0
2. **标准差法**：k=2.5
3. **IQR 法**

### 4.4 分数解读指南

| 特征距离 | 密度 | 最近邻 | 可能情况 | 建议操作 |
|---------|------|--------|---------|---------|
| 高 | 高 | 高 | 完全孤立异常 | 优先处理 |
| 高 | 低 | 低 | 远离中心但有邻居 | 检查分布 |
| 低 | 高 | 高 | 中心附近孤立点 | 检查局部 |
| 低 | 低 | 低 | 正常样本 | 无需处理 |

## 5. 特征嵌入分析说明

### 5.1 嵌入过程

#### 5.1.1 特征构建

```python
特征 = [coord_x, coord_y, radius, angle, value, value_centered]
```

- **coord_x, coord_y**：原始坐标
- **radius**：距离原点的欧氏距离
- **angle**：坐标的极角
- **value**：原始值
- **value_centered**：中心化后的值

#### 5.1.2 编码器

```python
隐藏特征 = tanh(特征 @ encoder_weight)
```

- encoder_weight: (6, 12)
- 输出维度: 12

#### 5.1.3 投影器

```python
投影特征 = (隐藏特征 @ projector_weight) / ||...||
```

- projector_weight: (12, 8)
- L2 归一化确保稳定性
- 输出维度: 8

### 5.2 特征库管理

#### 5.2.1 特征库更新

```python
def _update_feature_bank(self, projected: np.ndarray) -> None:
    bank = concatenate([feature_bank, projected])
    if len(bank) > queue_size:
        bank = bank[-queue_size:]
    feature_bank = bank
```

- 队列大小：默认 256
- FIFO 策略：保留最新的样本
- 在线更新：支持增量学习

#### 5.2.2 特征库状态

```python
# 查看特征库状态
```

## 6. 使用示例

### 6.1 基础使用

```python
import numpy as np
from deep_learning.models.anomaly_detection import ContrastiveAnomalyDetector

# 准备数据
coords = np.random.rand(100, 2)
values = np.random.rand(100)

# 训练模型
detector = ContrastiveAnomalyDetector()
training_result = detector.fit(coords, values, epochs=30)

print(f"训练完成:")
print(f"  最终损失: {training_result['final_loss']:.4f}")
print(f"  最佳损失: {training_result['best_loss']:.4f}")
print(f"  特征库大小: {training_result['feature_bank_size']}")

# 预测异常
result = detector.predict(coords, values, percentile=95.0)

print(f"\n检测结果:")
print(f"  异常数: {result['anomaly_count']}")
print(f"  阈值: {result['threshold']:.4f}")
print(f"  异常索引: {result['anomaly_indices']}")
```

## 7. 参数说明

| 参数 | 类型 | 默认值 | 说明 |
|-----|------|--------|------|
| `coords` | np.ndarray | 必需 | 坐标数组，形状 (N, 2) |
| `values` | np.ndarray | 必需 | 值数组，形状 (N,) |
| `threshold_method` | str | "percentile" | 阈值方法 |
| `percentile` | float | 95.0 | 百分位数 |
| `k` | float | 2.5 | 标准差倍数 |

### 7.5 参数调优建议

#### 7.5.1 特征维度 (feature_dim)

- **小维度 (8-10)**：
  - 计算快
  - 可能欠拟合

- **中等维度 (12-14)**：
  - 推荐
  - 平衡性能和复杂度

- **大维度 (15-16+)**：
  - 表达能力强
  - 可能过拟合

#### 7.5.2 投影维度 (projection_dim)

- **小维度 (4-6)**：
  - 紧凑表示
  - 可能信息损失

- **中等维度 (7-10)**：
  - 推荐
  - 保留足够信息

- **大维度 (11-12+)**：
  - 信息丰富
  - 可能不稳定

#### 7.5.3 温度参数 (temperature)

- **小温度 (0.1-0.15)**：
  - 更硬的分布
  - 更关注正样本

- **中等温度 (0.2-0.3)**：
  - 推荐
  - 平衡

- **大温度 (0.4-0.5)**：
  - 更软的分布
  - 更关注负样本

#### 7.5.4 特征库大小 (queue_size)

- **小库 (128-200)**：
  - 内存占用小
  - 可能信息不足

- **中等库 (256-400)**：
  - 推荐
  - 平衡

- **大库 (401-512+)**：
  - 信息丰富
  - 内存占用大

#### 7.5.5 训练轮数 (epochs)

- **快速训练 (20-25)**：
  - 快速验证
  - 可能未收敛

- **标准训练 (30-40)**：
  - 推荐
  - 通常能收敛

- **深度训练 (45-50+)**：
  - 更充分训练
  - 可能过拟合

## 8. 常见问题解答

### 8.1 训练相关

**Q: 训练损失不下降怎么办？**

A: 检查以下几点：
1. 调整 `temperature` 参数
2. 增加 `epochs`
3. 检查数据增强是否合理
4. 尝试不同的 `random_state`

**Q: 训练很慢怎么办？**

A: 优化建议：
1. 减少 `feature_dim` 和 `projection_dim`
2. 减少 `epochs`
3. 减少 `queue_size`
4. 使用数据采样

**Q: 特征库很快满了怎么办？**

A: 这是正常的：
1. 特征库是 FIFO 队列
2. 自动丢弃旧样本
3. 可以增加 `queue_size`
4. 或者接受在线更新特性

**Q: 如何判断训练是否收敛？**

A: 检查方法：
```python
# 查看训练历史
history = detector.history
losses = [item["total_loss"] for item in history]

# 检查最后几个epoch的损失变化
recent_losses = losses[-10:]
print(f"最近10轮损失变化: {np.std(recent_losses):.6f}")

# 可视化
plt.plot(losses)
plt.xlabel('Epoch')
plt.ylabel('Loss')
plt.title('训练损失曲线')
plt.show()
```

### 8.2 预测相关

**Q: 特征距离都很小？**

A: 可能原因：
1. 数据分布集中
2. 特征库太小
3. 特征维度不够
4. 解决：增加 `feature_dim` 或 `queue_size`

**Q: 密度分数都很高？**

A: 可能原因：
1. 数据稀疏
2. k 值设置不当
3. 特征表示不好
4. 解决：调整 k 值或特征维度

**Q: 最近邻分数不稳定？**

A: 可能原因：
1. 数据噪声大
2. 特征库更新频繁
3. 相似度计算不稳定
4. 解决：增加数据质量或减少更新频率

**Q: 综合分数与单个分数不一致？**

A: 这是正常的：
- 综合分数是加权组合
- 不同分数可能给出不同判断
- 需要综合分析

### 8.3 特征嵌入相关

**Q: 嵌入空间太密集怎么办？**

A: 解决方法：
1. 增加 `projection_dim`
2. 降低 `temperature`
3. 使用更好的数据增强
4. 增加训练轮数

**Q: 嵌入范数不接近1？**

A: 检查归一化：
```python
embeddings = detector.encode(coords, values)
norms = np.linalg.norm(embeddings, axis=1)
print(f"嵌入范数: 均值={norms.mean():.4f}, 标准差={norms.std():.4f}")
```
如果偏差大，检查代码中的归一化。

**Q: 如何评估嵌入质量？**

A: 评估方法：
1. 可视化嵌入空间（PCA/t-SNE）
2. 计算类内/类间距离
3. 检查相似度分布
4. 使用下游任务评估

### 8.4 在线更新相关

**Q: 在线更新后性能下降？**

A: 可能原因：
1. 新数据与旧数据分布不同
2. 特征库被新数据覆盖
3. 概念漂移
4. 解决：定期重新训练或使用更大的队列

**Q: 在线更新太频繁？**

A: 优化建议：
1. 批量更新而非逐个更新
2. 设置更新间隔
3. 使用更大的队列

**Q: 如何重置特征库？**

A: 重置方法：
```python
# 直接清空特征库
detector.feature_bank = np.zeros((0, detector.config.projection_dim), dtype=float)

# 重新训练
detector.fit(coords, values, epochs=30)
```

### 8.5 性能相关

**Q: 对比学习比其他模型慢？**

A: 原因：
1. 需要计算相似度矩阵
2. 特征库管理开销
3. 数据增强开销
4. 解决：减少特征维度或队列大小

**Q: 检测准确率不如其他模型？**

A: 可能原因：
1. 数据类型不适合
2. 参数不合适
3. 特征工程不足
4. 解决：调整参数或尝试其他模型

**Q: 误报率高？**

A: 降低误报：
1. 提高 `percentile`
2. 调整综合分数权重
3. 使用模型集成
4. 增加特征库大小

### 8.6 解释相关

**Q: 如何解释特征距离？**

A: 分析步骤：
1. 查看特征库分布
2. 分析样本在嵌入空间的位置
3. 比较正常样本和异常样本的距离
4. 使用可视化辅助理解

**Q: 如何解释密度？**

A: 分析步骤：
1. 查看局部邻域
2. 计算kNN距离
3. 分析密度分布
4. 识别低密度区域

**Q: 如何解释最近邻？**

A: 分析步骤：
1. 找到最近邻样本
2. 比较特征相似度
3. 分析空间距离
4. 判断是否真正相似

### 8.7 集成相关

**Q: 如何与其他模型集成？**

A: 参考代码：
```python
from deep_learning.models.anomaly_detection import (
    ContrastiveAnomalyDetector,
    VAEAnomalyDetector,
    AnomalyEnsembleIntegrator
)

# 训练多个模型

vae = VAEAnomalyDetector()
contrastive.fit(coords, values, epochs=30)
vae.fit(coords, values)

# 创建集成
detectors = {"contrastive": contrastive, "vae": vae}
ensemble = AnomalyEnsembleIntegrator(detectors)

# 集成预测
result = ensemble.detect(coords, values)
```

### 8.8 其他

**Q: 对比学习适合什么类型的数据？**

A: 适用数据：
- 有明确正常模式的场景
- 需要在线学习的场景
- 数据分布可能变化
- 需要检测孤立异常

**Q: 对比学习不适合什么数据？**

- 没有明确正常模式
- 数据完全随机
- 小数据集（< 30 样本）

**Q: 如何处理新类别？**
