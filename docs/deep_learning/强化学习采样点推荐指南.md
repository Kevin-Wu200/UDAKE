# 强化学习采样点推荐指南

## 概述

强化学习采样点推荐系统利用PPO、DQN、A2C等强化学习算法，智能推荐最优采样点位置，以最大化信息增益和降低不确定性。系统支持多种推荐策略，并提供详细的解释和评估指标。

## 核心功能

### 1. 多模型支持

支持三种主流强化学习算法：

- **PPO**：稳定性好，适合复杂环境
- **DQN**：基于价值，解释性强
- **A2C**：Actor-Critic架构，平衡性能和稳定性

### 2. 混合推荐策略

- **RL Only**：完全依赖强化学习策略
- **Rule Only**：基于不确定性峰值的规则方法
- **Hybrid**：结合RL和规则，优势互补

### 3. 实时推荐

支持实时在线推荐，适应动态变化的环境。

## 快速开始

### 基本使用

```python
import numpy as np
from deep_learning.models.sampling_rl import SamplingRLIntegrator

# 创建不确定性地图
uncertainty_map = np.random.rand(24, 24)

# 初始化集成器
integrator = SamplingRLIntegrator(model_name="ppo", seed=42)

# 训练模型
train_result = integrator.train(
    uncertainty_map=uncertainty_map,
    episodes=20,
    budget=16
)

# 生成推荐
recommendation = integrator.recommend(
    uncertainty_map=uncertainty_map,
    n_recommendations=8,
    fusion_strategy="hybrid",
    realtime=True
)

# 查看推荐结果
print(recommendation['recommendations'])
```

### 命令行使用

## API 参考

### SamplingRLIntegrator

#### 初始化

```python
SamplingRLIntegrator(model_name: str = "ppo", seed: int = 42)
```

**参数**：
- `model_name`：模型名称，可选 "ppo", "dqn", "a2c", "a3c"
- `seed`：随机种子

#### 训练方法

```python
train(
    uncertainty_map: np.ndarray,
    existing_points: np.ndarray | None = None,
    boundary: tuple[float, float, float, float] = (0.0, 1.0, 0.0, 1.0),
    episodes: int = 30,
    budget: int = 20
) -> dict[str, Any]
```

**参数**：
- `uncertainty_map`：不确定性地图 (H×W)
- `existing_points`：已有采样点坐标 (N×2)
- `boundary`：边界范围 (min_x, max_x, min_y, max_y)
- `episodes`：训练回合数
- `budget`：采样预算

**返回**：训练结果字典

#### 推荐方法

```python
recommend(
    uncertainty_map: np.ndarray,
    existing_points: np.ndarray | None = None,
    boundary: tuple[float, float, float, float] = (0.0, 1.0, 0.0, 1.0),
    n_recommendations: int = 10,
    fusion_strategy: str = "hybrid",
    realtime: bool = False
) -> dict[str, Any]
```

**参数**：
- `uncertainty_map`：不确定性地图
- `existing_points`：已有采样点
- `boundary`：边界范围
- `n_recommendations`：推荐点数
- `fusion_strategy`：融合策略 ("rl_only", "rule_only", "hybrid")
- `realtime`：是否实时模式

**返回**：推荐结果字典

## 推荐策略详解

### RL Only 策略

**特点**：
- 完全依赖训练好的强化学习策略
- 考虑长期累积奖励
- 策略稳定性高

**适用场景**：
- 有充足的训练数据
- 需要长期规划
- 环境相对稳定

**优势**：
- 充分利用历史经验
- 平衡探索和利用
- 适应复杂环境

### Rule Only 策略

**特点**：
- 基于不确定性峰值
- 简单直接，可解释性强
- 计算速度快

**适用场景**：
- 缺乏训练数据
- 需要快速响应
- 要求高度可解释性

**优势**：
- 无需训练
- 结果确定
- 易于理解和调试

### Hybrid 策略

**特点**：
- 结合RL和规则方法
- 自动选择最优候选
- 平衡性能和稳定性

**融合逻辑**：
1. 生成RL候选和规则候选
2. 评估两组候选的质量
3. 选择得分更高的候选集
4. 保持空间多样性

**适用场景**：
- 需要平衡性能和可解释性
- 环境动态变化
- 要求推荐质量

## 最佳实践

### 1. 模型选择

| 场景 | 推荐模型 | 理由 |
|------|----------|------|
| 复杂环境，需要稳定性 | PPO | 训练稳定，适合复杂任务 |
| 需要强解释性 | DQN | Q值直观，易于理解 |
| 平衡性能和资源 | A2C | 计算效率高，性能好 |

### 2. 参数调优

**训练参数**：
- `episodes`：30-50，根据复杂度调整
- `budget`：推荐点数的1.5-2倍
- `max_steps`：预算的1.5-2倍

**推荐参数**：
- `n_recommendations`：根据实际需求
- `fusion_strategy`：优先使用hybrid
- `realtime`：生产环境建议True

### 3. 评估指标

**质量评估**：
- `quality_score` > 0.7：优秀
- `quality_score` 0.5-0.7：良好
- `quality_score` < 0.5：需要改进

**效率评估**：
- `combined_efficiency_score` > 0.65：高效
- `combined_efficiency_score` 0.4-0.65：中等
- `combined_efficiency_score` < 0.4：低效

**鲁棒性评估**：
- `robustness_score` > 0.65：高鲁棒性
- `robustness_score` 0.4-0.65：中等鲁棒性
- `robustness_score` < 0.4：低鲁棒性

### 4. 数据准备

**不确定性地图**：
- 归一化到[0, 1]范围
- 避免极端值（<1e-6或>1）
- 考虑空间自相关性

**已有采样点**：
- 确保坐标在边界内
- 检查重复点
- 验证数据格式

## 故障排查

### 常见问题

**Q: 推荐结果不理想？**

A: 检查：
1. 不确定性地图质量
2. 训练回合数是否足够
3. 是否选择了合适的融合策略
4. 查看策略鲁棒性分析

**Q: 训练时间过长？**

A: 优化：
1. 减少训练回合数
2. 使用更小的地图尺寸
3. 调整最大步数
4. 考虑使用DQN替代PPO

**Q: 推荐点过于集中？**

A: 调整：
1. 增加采样点数量
2. 使用Hybrid策略
3. 检查空间多样性指标
4. 调整边界范围

**Q: 鲁棒性评分低？**

A: 改进：
1. 增加训练数据
2. 使用更稳定的模型（PPO）
3. 检查不确定性地图质量
4. 增加正则化

系统支持在线持续学习，适应环境变化：


使用多智能体系统提高推荐质量：

```python

# 创建多智能体系统

# 协作推荐

    uncertainty_map=uncertainty,

```

```python
    uncertainty_map=target_uncertainty,
    adaptation_mode=True
```


## 性能优化

### 批量处理

```python
# 批量推荐多个地图
results = []
for uncertainty_map in uncertainty_maps:
    result = integrator.recommend(
        uncertainty_map=uncertainty_map,
        realtime=True
    )
    results.append(result)
```

### 并行处理

```python
from concurrent.futures import ThreadPoolExecutor

def recommend_single(uncertainty_map):
    return integrator.recommend(
        uncertainty_map=uncertainty_map,
        realtime=True
    )

with ThreadPoolExecutor(max_workers=4) as executor:
    results = list(executor.map(recommend_single, uncertainty_maps))
```
