## 智能体参数

### PPOAgent 参数

PPO（Proximal Policy Optimization）是一种基于策略梯度的强化学习算法，通过裁剪策略更新来保证训练稳定性。

#### PPOConfig 配置

| 参数名 | 类型 | 默认值 | 范围 | 说明 |
|--------|------|--------|------|------|
| `learning_rate` | float | 0.02 | (0.001, 0.1) | 策略网络学习率 |
| `gamma` | float | 0.99 | (0.9, 0.999) | 折扣因子，控制未来奖励的重要性 |
| `gae_lambda` | float | 0.95 | (0.9, 0.99) | GAE（广义优势估计）的lambda参数 |
| `clip_ratio` | float | 0.2 | (0.1, 0.3) | PPO裁剪比率，限制策略更新幅度 |
| `entropy_coef` | float | 0.01 | (0.001, 0.05) | 熵奖励系数，鼓励探索 |
| `value_coef` | float | 0.5 | (0.1, 1.0) | 价值损失系数 |
| `grad_clip` | float | 1.0 | (0.1, 5.0) | 梯度裁剪阈值 |
| `update_epochs` | int | 4 | (1, 10) | 每次收集轨迹后的更新轮数 |

#### 参数调优建议

**学习率（learning_rate）**
- 较小值（0.001-0.01）：训练更稳定，但收敛慢
- 较大值（0.02-0.05）：收敛快，但可能不稳定
- 推荐：从0.02开始，根据训练曲线调整

**折扣因子（gamma）**
- 较小值（0.9-0.95）：关注短期奖励，适合快速变化的环境
- 较大值（0.98-0.999）：关注长期奖励，适合需要长期规划的任务
- 推荐：0.99适用于大多数采样优化场景

**裁剪比率（clip_ratio）**
- 较小值（0.1）：策略更新保守，稳定但可能收敛慢
- 较大值（0.3）：策略更新激进，可能不稳定
- 推荐：0.2是平衡稳定性和效率的常用值

**熵系数（entropy_coef）**
- 较小值（0.001）：探索不足，可能陷入局部最优
- 较大值（0.05）：探索充分，但学习效率低
- 推荐：0.01-0.02，根据探索需求调整

### DQNAgent 参数

DQN（Deep Q-Network）是一种基于价值迭代的强化学习算法，通过深度神经网络估计动作价值函数。

#### DQNConfig 配置

| 参数名 | 类型 | 默认值 | 范围 | 说明 |
|--------|------|--------|------|------|
| `learning_rate` | float | 0.02 | (0.001, 0.1) | Q网络学习率 |
| `gamma` | float | 0.98 | (0.9, 0.999) | 折扣因子 |
| `epsilon_start` | float | 1.0 | (0.5, 1.0) | 初始探索率 |
| `epsilon_end` | float | 0.05 | (0.01, 0.2) | 最终探索率 |
| `epsilon_decay` | float | 0.995 | (0.99, 0.999) | 探索率衰减因子 |
| `batch_size` | int | 32 | (16, 128) | 批量训练样本数 |
| `target_update_interval` | int | 30 | (10, 100) | 目标网络更新间隔 |
| `soft_tau` | float | 0.05 | (0.01, 0.1) | 软更新系数 |
| `exploration` | str | "epsilon_greedy" | - | 探索策略类型 |
| `network_type` | str | "dueling" | - | 网络架构类型 |

#### 探索策略类型

| 策略 | 说明 | 适用场景 |
|------|------|----------|
| `epsilon_greedy` | ε-贪婪策略，以ε概率随机选择动作 | 通用场景 |
| `boltzmann` | 基于Q值的Softmax概率选择 | 需要平滑探索时 |
| `ucb` | 上置信界策略，平衡利用和探索 | 需要理论保证时 |

#### 网络架构类型

| 类型 | 说明 | 优势 |
|------|------|------|
| `dqn` | 标准DQN架构 | 简单高效 |
| `graph` | 图卷积网络 | 考虑空间关系 |
| `dueling` | Dueling架构，分离状态价值和优势 | 更好的价值估计 |

#### 参数调优建议

**探索率衰减（epsilon_decay）**
- 较小值（0.99）：快速衰减，早期探索少
- 较大值（0.999）：缓慢衰减，持续探索
- 推荐：0.995-0.998，平衡探索和利用

**批量大小（batch_size）**
- 较小值（16-32）：更新频繁，但梯度噪声大
- 较大值（64-128）：梯度稳定，但更新慢
- 推荐：32-64，根据内存和速度权衡

**目标网络更新间隔（target_update_interval）**
- 较小值（10）：更新频繁，但不稳定
- 较大值（100）：更新稀疏，稳定但慢
- 推荐：30-50，平衡稳定性和效率

### ActorCriticAgent 参数

Actor-Critic算法结合了策略梯度和价值函数估计，通过Actor网络选择动作，Critic网络评估状态价值。

#### 配置参数

| 参数名 | 类型 | 默认值 | 范围 | 说明 |
|--------|------|--------|------|------|
| `learning_rate` | float | 0.02 | (0.001, 0.1) | 学习率 |
| `gamma` | float | 0.99 | (0.9, 0.999) | 折扣因子 |
| `entropy_coef` | float | 0.01 | (0.001, 0.05) | 熵系数 |
| `value_coef` | float | 0.5 | (0.1, 1.0) | 价值损失系数 |

#### 参数调优建议

Actor-Critic的参数调优与PPO类似，但需要注意：
- 熵系数通常比PPO稍大（0.01-0.03），因为Actor-Critic更容易陷入局部最优
- 学习率可以稍大（0.02-0.05），因为训练更不稳定

## 环境参数

### SamplingEnv 参数

采样环境是强化学习智能体与之交互的模拟环境。

#### 核心参数

| 参数名 | 类型 | 默认值 | 范围 | 说明 |
|--------|------|--------|------|------|
| `uncertainty_map` | np.ndarray | 必需 | - | 不确定性地图（二维数组） |
| `value_map` | np.ndarray | None | - | 真实值地图（可选，默认生成模拟数据） |
| `boundary` | tuple | (0.0, 1.0, 0.0, 1.0) | - | 边界范围（min_x, max_x, min_y, max_y） |
| `action_mode` | str | "discrete" | - | 动作空间类型 |
| `budget` | int | 20 | (5, 100) | 采样预算（最大采样点数） |
| `target_uncertainty` | float | 0.15 | (0.01, 0.5) | 目标不确定性阈值 |
| `max_steps` | int | 40 | (10, 200) | 最大步数 |
| `sampling_cost` | float | 0.03 | (0.01, 0.1) | 单次采样成本 |
| `min_distance` | float | 0.05 | (0.0, 0.2) | 最小采样点间距 |
| `seed` | int | 42 | - | 随机种子 |

#### 动作空间类型

| 类型 | 说明 | 动作格式 |
|------|------|----------|
| `discrete` | 离散动作空间 | 整数索引（0到grid_size-1） |
| `continuous` | 连续动作空间 | [x, y] 坐标 |
| `hybrid` | 混合动作空间 | {"position": [x, y], "sample_count": n} |

#### 参数调优建议

**预算（budget）**
- 较小值（5-10）：资源受限，快速训练
- 较大值（20-50）：充分采样，更精确
- 推荐：根据实际应用场景设置，通常15-30

**目标不确定性（target_uncertainty）**
- 较小值（0.01-0.1）：高精度要求，训练难度大
- 较大值（0.2-0.5）：低精度要求，容易达到
- 推荐：0.1-0.2，平衡精度和效率

**采样成本（sampling_cost）**
- 较小值（0.01）：采样成本低，鼓励密集采样
- 较大值（0.1）：采样成本高，鼓励稀疏采样
- 推荐：0.02-0.05，根据实际成本设置

**最小距离（min_distance）**
- 较小值（0.0-0.02）：允许密集采样
- 较大值（0.1-0.2）：强制分散采样
- 推荐：0.05-0.1，避免过度聚集

## 奖励参数

### RewardWeights 配置

奖励权重定义了不同奖励组件的重要性。

| 参数名 | 类型 | 默认值 | 范围 | 说明 |
|--------|------|--------|------|------|
| `uncertainty_reduction` | float | 0.40 | (0.1, 0.6) | 不确定性降低权重 |
| `accuracy_improvement` | float | 0.25 | (0.1, 0.4) | 精度提升权重 |
| `spatial_coverage` | float | 0.20 | (0.1, 0.3) | 空间覆盖权重 |
| `sampling_cost` | float | 0.08 | (0.01, 0.2) | 采样成本惩罚权重 |
| `boundary_constraint` | float | 0.04 | (0.01, 0.1) | 边界约束惩罚权重 |
| `distance_constraint` | float | 0.03 | (0.01, 0.1) | 距离约束惩罚权重 |

#### 奖励组件说明

**不确定性降低（uncertainty_reduction）**
- 测量采样后不确定性的减少量
- 范围：[0, 1]，1表示完全消除不确定性

**精度提升（accuracy_improvement）**
- 测量预测精度的提升量
- 范围：[0, 1]，1表示完全准确

**空间覆盖（spatial_coverage）**
- 测量采样区域的空间覆盖率
- 范围：[0, 1]，1表示完全覆盖

**采样成本惩罚（sampling_cost_penalty）**
- 惩罚采样成本，限制采样数量
- 范围：[0, ∞]，越大惩罚越强

**边界约束惩罚（boundary_penalty）**
- 惩罚超出边界的采样
- 范围：{0, 1}，1表示超出边界

**距离约束惩罚（distance_penalty）**
- 惩罚过于密集的采样
- 范围：[0, 1]，1表示违反最小距离

#### 参数调优建议

**优先级调整**
- 重视精度：增加 `accuracy_improvement` 权重（0.3-0.4）
- 重视覆盖：增加 `spatial_coverage` 权重（0.25-0.35）
- 控制成本：增加 `sampling_cost` 权重（0.1-0.15）

**权重归一化**
- 确保正向权重之和接近1
- 惩罚权重应该较小，避免过度惩罚

## 训练参数

### SamplingRLTrainingConfig 配置

训练配置控制整个训练过程的参数。

| 参数名 | 类型 | 默认值 | 范围 | 说明 |
|--------|------|--------|------|------|
| `model_name` | str | "ppo" | - | 模型名称（ppo/dqn/actor_critic） |
| `episodes` | int | 50 | (10, 500) | 训练回合数 |
| `max_steps_per_episode` | int | 40 | (10, 200) | 每回合最大步数 |
| `learning_rate` | float | 0.02 | (0.001, 0.1) | 学习率 |
| `optimizer` | str | "adam" | - | 优化器类型 |
| `reward_target` | float | 1.0 | (0.5, 2.0) | 目标奖励值 |
| `early_stopping_patience` | int | 8 | (3, 20) | 早停耐心值 |
| `distributed_workers` | int | 2 | (1, 8) | 分布式工作进程数 |
| `search_space` | dict | 见下方 | - | 超参数搜索空间 |

#### 超参数搜索空间

默认搜索空间：

```python
{
    "learning_rate": [0.01, 0.02, 0.03],
    "gamma": [0.95, 0.98, 0.99],
    "entropy_coef": [0.005, 0.01, 0.02]
}
```

#### 参数调优建议

**训练回合数（episodes）**
- 快速原型：10-20回合
- 正常训练：50-100回合
- 充分训练：200-500回合

**早停耐心值（early_stopping_patience）**
- 较小值（3-5）：快速停止，可能过早
- 较大值（10-20）：充分训练，可能浪费时间
- 推荐：8-12，平衡效率和充分性

**分布式工作进程数（distributed_workers）**
- 较小值（1-2）：低资源占用
- 较大值（4-8）：加速训练，高资源占用
- 推荐：2-4，根据可用CPU核心数

## 特征工程参数

### SamplingFeatureEngineer 参数

特征工程器负责从状态、动作、奖励中提取特征。

#### 特征类别

**状态特征（STATE_FEATURE_NAMES）**
- `state.mean_uncertainty`：平均不确定性
- `state.std_uncertainty`：不确定性标准差
- `state.max_uncertainty`：最大不确定性
- `state.p90_uncertainty`：90分位不确定性
- `state.gradient_mean`：不确定性梯度均值
- `state.sampled_ratio`：已采样覆盖率
- `state.sampled_value_mean`：采样值均值
- `state.budget_remaining_ratio`：预算剩余比例
- `state.step_progress_ratio`：步数进度比例
- `state.boundary_area`：边界面积

**动作特征（ACTION_FEATURE_NAMES）**
- `action.mode_id`：动作类型编码
- `action.position_x_norm`：X归一化位置
- `action.position_y_norm`：Y归一化位置
- `action.position_index_norm`：索引归一化
- `action.sample_count_norm`：采样数量归一化
- `action.new_point_ratio`：新点比例
- `action.coverage_gain_potential`：覆盖增益潜力

**奖励特征（REWARD_FEATURE_NAMES）**
- `reward.uncertainty_reduction`：不确定性降低
- `reward.accuracy_improvement`：精度提升
- `reward.spatial_coverage`：空间覆盖
- `reward.sampling_cost_penalty`：采样成本惩罚
- `reward.boundary_penalty`：边界惩罚
- `reward.distance_penalty`：距离惩罚
- `reward.weighted_positive`：加权正向收益
- `reward.weighted_penalty`：加权惩罚成本
- `reward.raw_total`：原始总奖励

**策略特征（POLICY_FEATURE_NAMES）**
- `policy.mode_id`：动作空间编码
- `policy.max_action_prob`：最大动作概率
- `policy.selected_action_prob`：选中动作概率
- `policy.policy_entropy`：策略熵
- `policy.top2_gap`：前二动作概率差
- `policy.prob_mean`：概率均值
- `policy.prob_std`：概率标准差

**价值特征（VALUE_FEATURE_NAMES）**
- `value.state_value`：当前状态价值
- `value.next_state_value`：下一状态价值
- `value.td_target`：TD目标
- `value.td_error`：TD误差
- `value.advantage_like`：优势近似项

**交互特征（STATE_ACTION_INTERACTION_NAMES）**
- `interaction.uncertainty_x_sampling`：不确定性与采样强度
- `interaction.coverage_gap_x_action_novelty`：覆盖缺口与动作新颖度
- `interaction.step_progress_x_action_density`：进度与动作密度

#### 特征参数

| 参数名 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `k` | int | 5 | 密度特征和拓扑特征的邻居数 |

## 超参数优化参数

### HyperparameterOptimizer 参数

超参数优化器支持多种搜索策略。

#### 搜索方法

**网格搜索（grid_search）**
- 遍历所有参数组合
- 优点：保证找到最优
- 缺点：计算成本高

**随机搜索（random_search）**
- 随机采样参数组合
- 优点：效率高，适合高维空间
- 缺点：不保证最优

**贝叶斯搜索（bayesian_search）**
- 基于历史结果指导搜索
- 优点：效率高，智能探索
- 缺点：实现复杂

#### 参数

| 参数名 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `seed` | int | 42 | 随机种子 |
| `n_trials` | int | 10-12 | 随机/贝叶斯搜索的试验次数 |

## 最佳实践

### 1. 参数选择流程

1. **从默认值开始**：使用推荐的默认参数
2. **逐步调整**：每次只调整一个参数
3. **监控训练曲线**：观察奖励、损失、熵的变化
4. **验证性能**：在验证集上评估最终性能
5. **迭代优化**：根据结果调整参数

### 2. 常见问题

**Q: 训练不稳定怎么办？**

A: 尝试以下调整：
- 降低学习率（0.01-0.02）
- 增加梯度裁剪（1.0-2.0）
- 减小clip_ratio（0.1-0.15）
- 增加target_update_interval（DQN）

**Q: 探索不足怎么办？**

A: 尝试以下调整：
- 增加熵系数（0.02-0.05）
- 减慢epsilon衰减（0.998-0.999）
- 增加初始epsilon（1.0）
- 使用更复杂的探索策略（UCB）

**Q: 收敛慢怎么办？**

A: 尝试以下调整：
- 增加学习率（0.03-0.05）
- 减少训练回合数（先用少量回合快速验证）
- 增加批量大小（64-128）
- 使用分布式训练

### 3. 参数敏感性分析

**高敏感参数**
- 学习率：影响收敛速度和稳定性
- 折扣因子：影响长期规划能力
- 熵系数：影响探索程度

**中等敏感参数**
- clip_ratio：影响策略更新幅度
- epsilon_decay：影响探索-利用平衡
- 奖励权重：影响行为偏好

**低敏感参数**
- 梯度裁剪：主要影响极端情况
- 软更新系数：影响目标网络更新速度
- 批量大小：主要影响训练速度
