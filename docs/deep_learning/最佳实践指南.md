# 深度学习最佳实践

1. 先跑 `tests/deep_learning`，确保基础设施稳定。
2. 新模型必须先注册到 `ModelRegistry`，再进入训练流程。
3. 所有训练参数必须来自 YAML 配置，避免硬编码。
4. 线上推理必须启用缓存和后处理，减少抖动与重复计算。
5. 将关键指标（loss、延迟、资源）统一写入监控仪表板。
6. 使用 LIME 解释时优先走异步任务接口，并显式设置 `top_k` 和 `max_retries`。
7. 同一批输入重复解释时优先复用缓存，避免频繁重新采样。
8. 前端展示建议同时使用 `feature_importance_list` 与 `local_explanations`，避免只看单一指标。
