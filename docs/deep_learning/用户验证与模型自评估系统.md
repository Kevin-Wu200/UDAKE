# 用户验证与模型自评估系统文档

## 1. 概述

阶段11新增“用户验证与模型自评估系统”，用于在用户反馈闭环中持续监控模型效果、检测漂移、执行模型切换与优化触发，并输出结构化报告。

核心能力：
- 实时评估：准确率、MAE、RMSE、滑动窗口、分区评估
- 误差分析：分布、热力图、时间趋势、异常误差
- 不确定性评估：可靠性图、ECE、Brier、LogLoss、PICP、PINAW
- 模型治理：选择、切换、回滚、A/B测试
- 优化闭环：触发、状态、取消、报告

## 2. API 文档

基础路径：`/api`

### 2.1 实时评估 API
- `POST /evaluation/realtime`：提交实时验证记录并返回实时评估结果
- `GET /evaluation/performance`：获取性能指标
- `GET /evaluation/errors`：获取误差分析
- `GET /evaluation/uncertainty`：获取不确定性评估

### 2.2 模型选择 API
- `POST /model-selection/select`：选择最佳模型
- `GET /model-selection/status`：获取模型状态
- `POST /model-selection/switch`：手动切换模型
- `POST /model-selection/rollback`：回滚模型

### 2.3 优化触发 API
- `POST /optimization/trigger`：触发优化任务
- `GET /optimization/status`：查询优化状态
- `POST /optimization/cancel`：取消优化任务

### 2.4 报告 API
- `GET /reports/performance`：性能报告
- `GET /reports/evaluation`：评估报告
- `GET /reports/optimization`：优化报告
- `POST /reports/generate`：生成综合报告

## 3. 评估指标说明

### 3.1 准确率指标
- `overall_accuracy`：综合准确率（用户接受率与回归精度融合）
- `local_accuracy`：区域维度准确率
- `time_accuracy`：时间维度准确率
- `classification_accuracy`：类别准确率
- `regression_accuracy`：MAE、RMSE、容差内比例

### 3.2 误差指标
- 统计指标：均值、方差、分位数、标准差
- 分布指标：误差直方分布
- 相关性：空间-误差相关、时间-误差相关
- 异常检测：IQR异常值
- 趋势指标：按小时误差趋势

### 3.3 不确定性指标
- 校准：可靠性图、ECE
- 概率评分：Brier Score、Log Loss
- 区间指标：PICP、PINAW、平均区间宽度
- 尖锐性：置信度标准差

### 3.4 用户满意度指标
- 接受率、拒绝率、修正率
- 平均置信度、置信度分布
- 平均响应时间、平均验证时间

### 3.5 数据覆盖率指标
- 空间覆盖率
- 时间覆盖率
- 数据密度
- 分布均衡性
- 数据缺口识别

## 4. 用户手册

### 4.1 快速开始
1. 使用 `POST /evaluation/realtime` 上传用户验证数据。
2. 使用 `/evaluation/performance|errors|uncertainty` 查看评估结果。
3. 使用 `/model-selection/select` 完成模型排名与自动切换。
4. 使用 `/optimization/trigger` 发起优化流程。
5. 使用 `/reports/generate` 产出评估报告。

### 4.2 权限与认证
- 所有接口需要 `X-API-Key`。
- 可选 `X-User-Id` 用于审计和归属。

## 5. 集成指南

### 5.1 数据反馈模块集成
将用户验证数据整理为统一记录后调用 `POST /evaluation/realtime`。

### 5.2 realtime_interpolation 集成
提交插值预测值与用户观测值，使用 `module: realtime_interpolation` 标记来源。

### 5.3 adaptive_sampling 集成
提交采样建议验证结果，使用 `module: adaptive_sampling`。

### 5.4 ai_extension 集成
提交异常检测验证结果，使用 `module: ai_extension`。

## 8. 最佳实践

- 使用滑动窗口（建议 60~180 分钟）平衡实时性与稳定性。
- 设置分层阈值（信息/警告/严重）防止告警噪声。
- 仅在收益超过 `switch_min_gain` 时自动切换模型。
- 对优化任务启用取消机制，避免资源拥塞。
- 周期性生成 `all` 报告并沉淀到运维看板。
