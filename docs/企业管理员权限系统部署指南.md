# 企业管理员权限系统部署指南

## 1. 部署前检查
- 确认已备份认证数据库（`auth.db` 或生产库）。
- 确认 SMTP 账号、授权码、白名单域名已准备。
- 确认定时任务参数（执行时间、重试次数）已配置。

## 2. 数据库迁移步骤
1. 备份数据库：
```bash
bash deployment/migration/backup_and_migrate.sh backup
```
2. 执行迁移：
```bash
bash deployment/migration/backup_and_migrate.sh migrate
```
3. 迁移后校验：
```bash
bash deployment/migration/backup_and_migrate.sh validate
```

## 3. 环境变量配置
- 参考 `configs/.env.example` 配置 SMTP 与调度器变量。
- 关键变量：
  - `SMTP_HOST/SMTP_PORT/SMTP_USER/SMTP_PASSWORD`
  - `KEY_EXPIRY_SCHEDULER_ENABLED`
  - `KEY_EXPIRY_SCHEDULER_EXPIRE_TIME`
  - `KEY_EXPIRY_SCHEDULER_REMINDER_TIME`

## 4. SMTP 配置说明
- 推荐生产环境使用 TLS（`SMTP_USE_TLS=true`）。
- 使用专用邮箱账号与最小权限授权码。
- 首次部署后通过前端 SMTP 配置页执行“连接测试”。

## 5. 定时任务配置说明
- 后端启动后自动注册两个任务：
  - `expiry_check`：检查并标记过期密钥。
  - `expiry_reminder`：发送到期提醒邮件（含失败重试）。
- 可通过 `/api/scheduler/*` 管理接口查看历史、手动触发、暂停/恢复与改时。

## 6. 回滚方案
1. 停止服务。
2. 还原数据库备份：
```bash
bash deployment/migration/backup_and_migrate.sh rollback /path/to/backup_file
```
3. 回退应用版本到上一稳定 Tag。
4. 启动服务并执行健康检查：`/health`。
