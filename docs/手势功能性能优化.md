# 手势功能性能优化

## 概述

本文档描述了手势增强功能的性能优化措施和最佳实践。

## 已实现的优化措施

### 1. 事件监听器优化

#### 被动事件监听器
使用被动事件监听器优化滚动和触摸事件性能：

```typescript
element.addEventListener('touchstart', this.handleTouchStart, { passive: true });
element.addEventListener('touchmove', this.handleTouchMove, { passive: false });
element.addEventListener('touchend', this.handleTouchEnd, { passive: true });
element.addEventListener('touchcancel', this.handleTouchCancel, { passive: true });
```

**优点**：
- 减少事件处理延迟
- 提高触摸响应速度
- 优化滚动性能

### 2. 手势识别优化

#### touchmove 实时预测触发
在 `touchmove` 阶段实时计算 `deltaX`、`deltaY`、`duration`、`velocity`，并用置信度模型提前触发高置信度手势（`confidence >= 0.9`）：

```typescript
const prediction = this.predictSingleFingerGesture(touch);
if (prediction && prediction.confidence >= 0.9) {
    this.triggerGesture(prediction.type, prediction.event);
}
```

**收益**：
- 不再等待 `touchend` 才触发滑动类手势
- 显著降低感知延迟（目标 < 20ms）
- 保持与原有 `touchend` 兼容，避免重复触发

#### 减少不必要的计算
- 只在需要时计算手势参数
- 缓存常用计算结果
- 使用惰性求值

```typescript
// 只在需要时计算三指距离
private getTripleFingerDistance(touches: TouchList): number {
    if (touches.length < 3) return 0; // 快速返回
    // 计算逻辑...
}
```

#### 优先级检测优化
使用快速优先级比较避免复杂的手势冲突检测：

```typescript
private checkGestureConflict(newGesture: GestureType): boolean {
    if (!this.options.enableGestureConflictDetection) return true;
    if (!this.activeGesture || !this.isGestureInProgress) return true;

    const newPriority = this.gesturePriority.get(newGesture) ?? 0;
    const activePriority = this.gesturePriority.get(this.activeGesture) ?? 0;

return newPriority > activePriority;
}
```

#### 冲突锁优化
新增短时冲突锁（`conflictLockMs`）避免同一触摸链路中低优先级手势抖动触发，提升识别稳定性。

### 3. 内存管理优化

#### 历史记录限制
限制撤销/重做历史记录的最大长度：

```typescript
private maxUndoSteps: number = 10;

private addToHistory(event: GestureEvent): void {
    this.gestureHistory.push(event);

    // 限制历史记录长度
    if (this.gestureHistory.length > this.maxUndoSteps) {
        this.gestureHistory.shift();
    }

    this.redoStack = []; // 清空重做栈释放内存
}
```

#### 资源清理
在销毁时正确清理所有资源：

```typescript
public destroy(): void {
    this.unbind();
    this.clearLongPress();
    this.callbacks.clear();
    this.touches.clear();
    this.gestureHistory = [];
    this.undoStack = [];
    this.redoStack = [];
    this.gesturePriority.clear();
    this.activeGesture = null;

    // 移除DOM元素
    if (this.feedbackElement && this.feedbackElement.parentNode) {
        this.feedbackElement.parentNode.removeChild(this.feedbackElement);
    }

    // 关闭音频上下文
    if (this.audioContext) {
        this.audioContext.close();
        this.audioContext = null;
    }
}
```

### 4. 触觉反馈优化

#### 异步触觉反馈
使用异步API避免阻塞主线程：

```typescript
private async hapticFeedback(style: ImpactStyle = ImpactStyle.Light): Promise<void> {
    if (!this.options.enableHaptic) return;

    try {
        await Haptics.impact({ style });
    } catch (e) {
        // 回退到标准vibrate API
        if ('vibrate' in navigator) {
            const pattern = this.getVibrationPattern(style);
            navigator.vibrate(pattern);
        }
    }
}
```

#### 批量触觉反馈
避免频繁的触觉反馈调用：

```typescript
// 在手势完成时统一提供反馈，而不是在每次移动时
private handleTouchEnd = (e: TouchEvent): void => {
    // ... 手势检测逻辑
    this.touchFeedback('gestureSuccess');
}
```

### 5. 可视化反馈优化

#### 使用requestAnimationFrame
使用requestAnimationFrame优化动画性能：

```typescript
private showVisualFeedback(type: string): void {
    if (!this.feedbackElement) return;

    // 播放动画
    requestAnimationFrame(() => {
        if (this.feedbackElement) {
            this.feedbackElement.style.transform = 'scale(1)';
            this.feedbackElement.style.opacity = '1';

            setTimeout(() => {
                if (this.feedbackElement) {
                    this.feedbackElement.style.transform = 'scale(1.5)';
                    this.feedbackElement.style.opacity = '0';
                }
            }, 200);
        }
    });
}
```

#### CSS动画优化
使用CSS transform和opacity进行动画，避免触发重排：

```css
.feedback-element {
    transform: scale(0);
    transition: transform 0.2s ease-out, opacity 0.2s ease-out;
    will-change: transform, opacity;
}
```

### 6. 手势检测优化

#### 阈值优化
使用合理的阈值避免误触发：

```typescript
private options: Required<TouchGestureManagerOptions> = {
    swipeThreshold: 50, // 滑动阈值
    quickSwipeThreshold: 100, // 快速滑动阈值
    longPressDelay: 500, // 长按延迟
    doubleTapDelay: 300, // 双击延迟

```

#### 防抖处理
对高频手势事件进行防抖处理：

```typescript
// 使用setTimeout延迟触发手势
this.longPressTimer = window.setTimeout(() => {
    if (this.checkGestureConflict('longPress')) {
        this.triggerGesture('longPress', event);
    }
}, this.options.longPressDelay);
```

## 性能监控

### 关键指标

1. **手势响应延迟**：从用户触摸到手势触发的延迟
   - 目标：< 20ms

2. **帧率**：手势动画的流畅度
   - 目标：≥ 60fps

3. **内存使用**：手势管理器的内存占用
   - 目标：< 5MB

4. **CPU使用率**：手势处理时的CPU占用
   - 目标：< 10%

### 监控方法

```typescript
// 使用Performance API监控手势性能
const startTime = performance.now();
// 执行手势处理
const endTime = performance.now();
const duration = endTime - startTime;
console.log(`Gesture processing time: ${duration}ms`);
```

## 最佳实践

### 1. 合理使用手势

- 只在需要时启用手势识别
- 避免同时启用太多手势类型
- 根据场景调整手势优先级

### 2. 优化事件处理

- 使用被动事件监听器
- 避免在事件处理中执行复杂计算
- 使用requestAnimationFrame进行动画

### 3. 内存管理

- 限制历史记录长度
- 及时清理不再使用的资源
- 避免内存泄漏

### 4. 用户体验

- 提供清晰的视觉反馈
- 优化触觉反馈的强度和时机
- 确保手势识别的准确性
