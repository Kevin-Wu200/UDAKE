# 模型融合系统文档

## 概述

模型融合系统是一个强大的工具，用于集成多个克里金模型的预测结果，通过加权平均、堆叠等方法提高预测稳定性和准确性。

## 核心功能

### 1. 多种融合策略

系统支持7种融合策略：

- **简单平均 (Simple Average)**: 所有模型预测的简单算术平均
- **加权平均 (Weighted Average)**: 基于模型性能的加权平均
- **中位数融合 (Median)**: 使用中位数作为融合结果，对异常值鲁棒
- **堆叠融合 (Stacking)**: 使用元学习器进行融合
- **贝叶斯模型平均 (BMA)**: 基于贝叶斯理论的模型平均
- **方差加权 (Variance Weighted)**: 基于预测方差的加权
- **最大最小融合 (Max-Min)**: 鲁棒融合方法

### 2. 智能权重计算

提供8种权重计算方法：

- **等权重**: 所有模型权重相等
- **基于RMSE**: RMSE越小，权重越大
- **基于MAE**: MAE越小，权重越大
- **基于R²**: R²越大，权重越大
- **交叉验证**: 基于交叉验证结果计算权重
- **贝叶斯模型平均**: 基于BIC的权重
- **基于不确定性**: 不确定性越低，权重越大
- **自适应**: 综合稳定性和准确性的自适应权重

### 3. 模型评估

支持多种评估指标：

- **RMSE**: 均方根误差
- **MAE**: 平均绝对误差
- **R²**: 决定系数
- **MAPE**: 平均绝对百分比误差
- **稳定性指标**: 预测稳定性评估

### 4. 高级功能

- **策略对比**: 自动比较不同融合策略的性能
- **权重优化**: 自动选择最佳权重计算方法
- **交叉验证**: K折交叉验证评估模型性能
- **Bootstrap稳定性评估**: 评估模型预测稳定性
- **不确定性传播**: 计算融合预测的不确定性

### 核心类

#### WeightCalculator
负责计算模型权重，支持多种权重计算方法。

```python
calculator = WeightCalculator()
weights = calculator.calculate_weights(
    method=WeightMethod.RMSE_BASED,
    model_metrics=model_metrics,
    min_weight=0.0,
    max_weight=1.0,
    normalize=True
)
```

#### FusionEngine
核心融合引擎，执行模型融合。

```python
engine = FusionEngine()
result = engine.fuse(
    config=fusion_config,
    models=model_predictions,
    true_values=true_values
)
```

#### FusionStrategies
实现各种融合策略。

```python
strategies = FusionStrategies()
fused_pred, fused_var = strategies.fuse(
    strategy=FusionStrategy.WEIGHTED_AVERAGE,
    models=models,
    weights=weights
)
```

#### ModelEvaluator
评估模型性能。

```python
evaluator = ModelEvaluator()
metrics = evaluator.evaluate_models(models, true_values)
```

## 使用示例

### Python代码示例

```python
from app.model_fusion.services.fusion_service import FusionService
from app.model_fusion.core.fusion_models import FusionStrategy, WeightMethod

# 创建服务实例
service = FusionService()

# 准备模型数据
models = [
    {
        'model_id': 'model1',
        'model_name': '普通克里金',
        'predictions': [10.5, 11.2, 10.8, 11.5, 10.9]
    },
    {
        'model_id': 'model2',
        'model_name': '泛克里金',
        'predictions': [10.8, 11.0, 10.9, 11.3, 11.1]
    }
]

# 配置融合参数
config = {
    'strategy': 'weighted_average',
    'weight_method': 'rmse_based',
    'min_weight': 0.0,
    'max_weight': 1.0,
    'normalize': True
}

# 真实值（可选，用于评估）
true_values = [10.6, 11.0, 10.9, 11.2, 10.8]

# 创建融合任务
task_id = service.create_fusion_task(models, config, true_values)

# 等待任务完成
import time
while True:
    status = service.get_task_status(task_id)
    if status['status'] in ['completed', 'failed']:
        break
    time.sleep(1)

# 获取结果
result = service.get_task_result(task_id)
print(f"融合预测: {result['fused_predictions']}")
print(f"模型权重: {result['weights']}")
print(f"评估指标: {result['metrics']}")
```

## 最佳实践

### 1. 权重方法选择

- 有真实值用于验证：使用 `rmse_based` 或 `cross_validation`
- 强调模型稳定性：使用 `adaptive`
- 不确定性已知：使用 `variance_weighted`
- 快速测试：使用 `equal`

### 2. 融合策略选择

- 快速融合：`simple_average` 或 `weighted_average`
- 鲁棒性要求：`median` 或 `max_min`
- 最高精度：`stacking` 或 `bma`
- 不确定性重要：`variance_weighted`

### 3. 参数调优

- `min_weight` 和 `max_weight`: 控制权重范围，避免某个模型权重过大
- `smoothing`: 启用权重平滑可以提高稳定性
- `enable_cross_validation`: 启用交叉验证可以更准确地评估模型性能

## 常见问题

### Q: 融合结果比最好的单模型还差？

A: 这可能是因为：
1. 模型权重计算不准确，尝试不同的权重方法
2. 融合策略不适合当前数据，尝试其他策略
3. 模型之间相关性太高，融合效果有限

### Q: 如何选择最佳融合策略？

A: 使用策略对比功能自动比较不同策略的性能：

```python
results = engine.compare_strategies(
    config=config,
    models=models,
    true_values=true_values
)
```

### Q: 如何处理缺失值？

A: 系统会自动处理缺失值：
- 如果某个模型的预测有缺失值，该位置会被排除在融合计算之外
- 使用中位数或最大最小融合策略对缺失值更鲁棒
