## 功能模块

### 路径规划系统

#### 概述

采样路径规划系统是一个智能化的野外采样路径优化工具，旨在帮助用户规划最优的采样路径，减少移动成本和时间，提高采样效率。

#### 功能特性

##### 核心功能

1. **多种路径规划算法**
   - Dijkstra算法：经典最短路径算法
   - A*算法：启发式搜索算法
   - TSP算法：旅行商问题求解（包含最近邻、2-opt、模拟退火）
   - 蚁群算法（ACO）：基于群体智能的优化算法

2. **多目标优化**
   - 最短距离优化
   - 最短时间优化
   - 最低成本优化
   - 综合优化

3. **灵活的约束条件**
   - 距离约束
   - 时间约束
   - 成本约束
   - 时间窗约束
   - 优先级约束
   - 车辆类型选择

4. **路径优化**
   - 自动优化现有路径
   - 添加/删除采样点
   - 重新排序采样点

5. **路径模板管理**
   - 保存常用路径配置
   - 快速加载模板
   - 模板版本管理

#### 算法说明

##### Dijkstra算法

- **适用场景：** 小规模问题（< 10个采样点）
- **优点：** 保证找到最优解
- **缺点：** 计算复杂度高，不适合大规模问题

##### A*算法

- **适用场景：** 有明确目标的路径搜索
- **优点：** 比Dijkstra更快，使用启发式信息
- **缺点：** 需要设计合适的启发函数

##### TSP算法

- **适用场景：** 中等规模问题（10-30个采样点）
- **包含算法：**
  - 最近邻算法：快速但可能不是最优
  - 2-opt算法：改进初始解
  - 模拟退火算法：跳出局部最优
- **优点：** 专为TSP问题设计
- **缺点：** 随着规模增大，计算时间急剧增加

##### 蚁群算法（ACO）

- **适用场景：** 大规模问题（> 30个采样点）
- **优点：** 适合复杂问题，能找到较好的解
- **缺点：** 需要调整参数，计算时间较长

#### 性能指标

| 规模 | 推荐算法 | 计算时间 | 内存占用 |
|------|---------|---------|---------|
| < 10个点 | Dijkstra/TSP | < 1秒 | 低 |
| 10-30个点 | TSP/ACO | 1-10秒 | 中 |
| 30-100个点 | ACO | 10-30秒 | 中高 |
| > 100个点 | ACO | > 30秒 | 高 |

#### 常见问题

##### Q: 路径规划时间过长怎么办？

A: 可以尝试以下方法：
1. 减少采样点数量
2. 使用更简单的算法（如Dijkstra）
3. 添加更多约束条件缩小搜索空间
4. 使用路径模板复用历史数据

##### Q: 如何选择合适的算法？

A: 根据采样点数量选择：
- < 10个点：使用Dijkstra或TSP
- 10-30个点：使用TSP
- > 30个点：使用ACO

##### Q: 如何处理不可达的采样点？

A: 系统会自动标记不可达点并在结果中显示警告信息。建议：
1. 在添加采样点时验证可达性
2. 使用路径模板保存已验证的采样点

### 模型融合系统

#### 概述

模型融合系统是一个强大的工具，用于集成多个克里金模型的预测结果，通过加权平均、堆叠等方法提高预测稳定性和准确性。

#### 核心功能

##### 1. 多种融合策略

系统支持7种融合策略：

- **简单平均**: 所有模型预测的简单算术平均
- **加权平均**: 基于模型性能的加权平均
- **中位数融合**: 使用中位数作为融合结果，对异常值鲁棒
- **堆叠融合**: 使用元学习器进行融合
- **贝叶斯模型平均 (BMA)**: 基于贝叶斯理论的模型平均
- **方差加权**: 基于预测方差的加权
- **最大最小融合**: 鲁棒融合方法

##### 2. 智能权重计算

提供8种权重计算方法：

- **等权重**: 所有模型权重相等
- **基于RMSE**: RMSE越小，权重越大
- **基于MAE**: MAE越小，权重越大
- **基于R²**: R²越大，权重越大
- **交叉验证**: 基于交叉验证结果计算权重
- **贝叶斯模型平均**: 基于BIC的权重
- **基于不确定性**: 不确定性越低，权重越大
- **自适应**: 综合稳定性和准确性的自适应权重

##### 3. 模型评估

支持多种评估指标：

- **RMSE**: 均方根误差
- **MAE**: 平均绝对误差
- **R²**: 决定系数
- **MAPE**: 平均绝对百分比误差
- **稳定性指标**: 预测稳定性评估

##### 4. 高级功能

- **策略对比**: 自动比较不同融合策略的性能
- **权重优化**: 自动选择最佳权重计算方法
- **交叉验证**: K折交叉验证评估模型性能
- **Bootstrap稳定性评估**: 评估模型预测稳定性
- **不确定性传播**: 计算融合预测的不确定性

#### 使用示例

##### Python代码示例

```python
from app.model_fusion.services.fusion_service import FusionService
from app.model_fusion.core.fusion_models import FusionStrategy, WeightMethod

# 创建服务实例
service = FusionService()

# 准备模型数据
models = [
    {
        'model_id': 'model1',
        'model_name': '普通克里金',
        'predictions': [10.5, 11.2, 10.8, 11.5, 10.9]
    },
    {
        'model_id': 'model2',
        'model_name': '泛克里金',
        'predictions': [10.8, 11.0, 10.9, 11.3, 11.1]
    }
]

# 配置融合参数
config = {
    'strategy': 'weighted_average',
    'weight_method': 'rmse_based',
    'min_weight': 0.0,
    'max_weight': 1.0,
    'normalize': True
}

# 真实值（可选，用于评估）
true_values = [10.6, 11.0, 10.9, 11.2, 10.8]

# 创建融合任务
task_id = service.create_fusion_task(models, config, true_values)

# 等待任务完成
import time
while True:
    status = service.get_task_status(task_id)
    if status['status'] in ['completed', 'failed']:
        break
    time.sleep(1)

# 获取结果
result = service.get_task_result(task_id)
print(f"融合预测: {result['fused_predictions']}")
print(f"模型权重: {result['weights']}")
print(f"评估指标: {result['metrics']}")
```

- **支持的模型数量**: 5+个
- **融合计算时间**: <单模型3倍
- **预测精度提升**: >10%
- **稳定性提升**: >20%

#### 最佳实践

##### 1. 权重方法选择

- 有真实值用于验证：使用 `rmse_based` 或 `cross_validation`
- 强调模型稳定性：使用 `adaptive`
- 不确定性已知：使用 `variance_weighted`
- 快速测试：使用 `equal`

##### 2. 融合策略选择

- 快速融合：`simple_average` 或 `weighted_average`
- 鲁棒性要求：`median` 或 `max_min`
- 最高精度：`stacking` 或 `bma`
- 不确定性重要：`variance_weighted`

##### 3. 参数调优

- `min_weight` 和 `max_weight`: 控制权重范围，避免某个模型权重过大
- `smoothing`: 启用权重平滑可以提高稳定性
- `enable_cross_validation`: 启用交叉验证可以更准确地评估模型性能

#### 常见问题

##### Q: 融合结果比最好的单模型还差？

A: 这可能是因为：
1. 模型权重计算不准确，尝试不同的权重方法
2. 融合策略不适合当前数据，尝试其他策略
3. 模型之间相关性太高，融合效果有限

##### Q: 如何选择最佳融合策略？

A: 使用策略对比功能自动比较不同策略的性能：

```python
results = engine.compare_strategies(
    config=config,
    models=models,
    true_values=true_values
)
```

##### Q: 如何处理缺失值？

A: 系统会自动处理缺失值：
- 如果某个模型的预测有缺失值，该位置会被排除在融合计算之外
- 使用中位数或最大最小融合策略对缺失值更鲁棒

### 已实现的优化措施

#### 1. 事件监听器优化

##### 被动事件监听器
使用被动事件监听器优化滚动和触摸事件性能：


#### 2. 手势识别优化

##### touchmove 实时预测触发
在 `touchmove` 阶段实时计算 `deltaX`、`deltaY`、`duration`、`velocity`，并用置信度模型提前触发高置信度手势（`confidence >= 0.9`）：

```typescript
const prediction = this.predictSingleFingerGesture(touch);
if (prediction && prediction.confidence >= 0.9) {
    this.triggerGesture(prediction.type, prediction.event);
}
```

**收益**：
- 不再等待 `touchend` 才触发滑动类手势
- 显著降低感知延迟（目标 < 20ms）
- 保持与原有 `touchend` 兼容，避免重复触发

##### 减少不必要的计算
- 只在需要时计算手势参数
- 缓存常用计算结果
- 使用惰性求值


```typescript
private checkGestureConflict(newGesture: GestureType): boolean {
    if (!this.options.enableGestureConflictDetection) return true;
    if (!this.activeGesture || !this.isGestureInProgress) return true;

    const newPriority = this.gesturePriority.get(newGesture) ?? 0;
    const activePriority = this.gesturePriority.get(this.activeGesture) ?? 0;

    return newPriority > activePriority;
}
```

##### 冲突锁优化
