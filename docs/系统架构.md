# 系统架构

## 概述

UDAKE 采用前后端分离架构，前端基于 Electron 桌面应用，后端基于 FastAPI 提供 RESTful API 服务，核心算法使用 PyKrige + GSTools + scikit-learn 实现。

## 整体架构

### 后端模块

- API 层：提供 RESTful 接口
- 服务层：业务逻辑处理
- 核心层：克里金算法实现
- 任务层：异步任务管理
- 工具层：通用工具函数

### 前端模块

- 地图展示：支持多引擎的地图组件
- 数据上传：文件上传与解析
- 参数配置：插值参数设置
- 结果展示：栅格图层可视化
- 任务监控：实时状态轮询

### 扩展模块

- AI 扩展：趋势识别、异常检测
- 自适应采样：采样优化算法
- 不确定性仪表板：风险评估与报告

## 数据流

1. 用户上传 GeoJSON 数据
2. 后端解析并存储数据
3. 用户配置插值参数
4. 后端执行克里金插值
5. 生成预测栅格和方差栅格
6. 前端加载并展示结果
7. 生成分析报告

## 地图引擎架构

系统采用适配器模式实现地图引擎的可切换性，支持高德地图、ArcGIS 和天地图三种引擎。

### 引擎抽象层

```
业务层（图层管理、主程序）
        ↓
    MapAdapter（统一接口）
        ↓
    ┌────────┬────────┬────────┐
    ↓        ↓        ↓        ↓
ArcGIS   Tianditu   AMap    (可扩展)
```

### BaseMapEngine 接口

位置：`frontend/js/map/core/BaseMapEngine.js`

定义统一的地图操作接口：

- `init(container, options)` - 初始化地图
- `setCenter(center)` / `getCenter()` - 中心点操作
- `setZoom(zoom)` / `getZoom()` - 缩放操作
- `fitToBounds(bounds)` - 适配到指定边界
- `onZoom(callback)` / `onMove(callback)` - 事件监听
- `destroy()` - 销毁实例

能力标记：
- `supportsCustomReset` - 是否支持自定义 reset 按钮

### 引擎实现

**AMapEngine（高德地图引擎）**

位置：`frontend/js/map/core/AMapEngine.js`

- 使用高德地图 JS API v2.0
- 动态加载脚本，按需初始化
- 支持用户设备定位
- 标准矢量图层渲染

**TiandituEngine（天地图引擎）**

位置：`frontend/js/map/core/TiandituEngine.js`

- `supportsCustomReset = true`
- 自研 WMTS 瓦片渲染
- 左键拖动、鼠标滚轮缩放
- 瓦片缓存和错误重试

**ArcGISEngine（ArcGIS 引擎）**

位置：`frontend/js/map/core/ArcGISEngine.js`

- `supportsCustomReset = false`（使用内置 Home 控件）
- 自动深色模式切换
- 原生 ArcGIS API 支持

### 管理器与组件

**MapManager（地图管理器）**

位置：`frontend/js/managers/MapManager.js`

- 根据配置创建对应的地图引擎
- 管理地图状态（initialCenter、initialZoom、geojson、mode）
- 条件渲染 reset 按钮（仅天地图模式）
- 处理模式切换（normal / areaSampling）

**ZoomControl（缩放控件）**

位置：`frontend/js/components/ZoomControl.js`

- 右下角动态显示
- 监听缩放变化更新滑块位置
- 显示后 1 秒自动淡出

**GeoUtils（地理工具）**

位置：`frontend/js/utils/GeoUtils.js`

- `calculateBoundsFromGeoJSON(geojson)` - 计算 GeoJSON 边界
- `extractCoordinates(geometry)` - 提取几何坐标
- `calculateDistance(point1, point2)` - 计算两点距离
- `expandBounds(bounds, padding)` - 扩展边界

## 性能优化

- 异步任务处理
- 分块克里金算法
- 栅格数据压缩
- 前端图层懒加载
- 地图引擎按需加载

## 扩展性

要添加新的地图引擎：

1. 继承 `BaseMapEngine`，实现所有抽象方法
2. 设置 `supportsCustomReset` 标记
3. 在 `MapManager` 中添加创建逻辑
4. 在 `map.config.js` 中添加引擎选项
