# 新建项目功能测试报告

## 测试环境
- 浏览器：需要支持 ES6 模块和 Geolocation API
- 测试文件：frontend/test-new-project.html
- 测试边界文件：frontend/test-boundary.geojson

## 测试场景

### 1. 自由采样 + 手动输入
**测试步骤：**
1. 点击"自由采样 + 手动输入"按钮
2. 在经度输入框输入：116.4074
3. 在纬度输入框输入：39.9042
4. 在采样值输入框输入：25.5
5. 点击"添加采样点"按钮

**预期结果：**
- 采样点成功添加
- 地图上显示采样点标记
- 项目信息面板更新，显示新增的采样点

### 2. 自由采样 + 自动定位
**测试步骤：**
1. 点击"自由采样 + 自动定位"按钮
2. 点击"获取当前位置"按钮
3. 允许浏览器获取位置权限
4. 在采样值输入框输入：30.2
5. 点击"添加采样点"按钮

**预期结果：**
- 成功获取设备位置
- 显示当前经纬度坐标
- 采样点成功添加

### 3. 区域采样 + 手动输入
**测试步骤：**
1. 点击"区域采样 + 手动输入"按钮
2. 点击"点击选择 GeoJSON 文件"
3. 选择 test-boundary.geojson 文件
4. 点击"上传边界"按钮
5. 在经度输入框输入：116.4（在边界内）
6. 在纬度输入框输入：39.9（在边界内）
7. 在采样值输入框输入：28.0
8. 点击"添加采样点"按钮

**预期结果：**
- 边界成功上传并在地图上显示
- 采样点成功添加（因为在边界内）
- 项目信息面板更新

### 4. 区域采样 + 自动定位
**测试步骤：**
1. 点击"区域采样 + 自动定位"按钮
2. 上传边界文件
3. 点击"获取当前位置"按钮
4. 输入采样值
5. 点击"添加采样点"按钮

**预期结果：**
- 如果设备位置在边界内，采样点成功添加
- 如果设备位置在边界外，显示错误提示

### 5. 点在区域外的情况
**测试步骤：**
1. 在区域采样模式下
2. 输入边界外的坐标（例如：经度 120.0，纬度 30.0）
3. 点击"添加采样点"按钮

**预期结果：**
- 显示错误提示："采样点超出区域边界，无法添加"
- 采样点不会被添加到项目中

### 6. 非法 GeoJSON
**测试步骤：**
1. 在区域采样模式下
2. 尝试上传一个非 Polygon/MultiPolygon 类型的 GeoJSON
3. 或上传一个格式错误的文件

**预期结果：**
- 显示错误提示："无效的多边形，仅支持 Polygon 或 MultiPolygon"
- 边界不会被加载

### 7. 定位权限拒绝
**测试步骤：**
1. 在自动定位模式下
2. 点击"获取当前位置"按钮
3. 在浏览器弹窗中拒绝定位权限

**预期结果：**
- 显示错误提示："定位权限被拒绝，请在浏览器设置中允许定位"
- 位置获取失败

### 8. 错误处理系统
**测试步骤：**
1. 点击"测试错误处理"按钮

**预期结果：**
- 依次显示各种类型的错误提示
- 每个提示都有正确的样式和动画
- 提示会在 3 秒后自动消失
