# 项目代码归档说明

## 1. 目标与范围

本说明用于项目收尾阶段的代码归档，覆盖以下事项：

- 清理无用代码与运行产物
- 整理代码结构与目录职责
- 补充关键脚本注释
- 更新项目文档导航
- 创建归档标签
- 建立归档分支
- 备份代码仓库
- 输出可复用归档流程

## 2. 已落地内容

### 2.1 清理无用代码

- 统一通过 `.gitignore` 约束临时文件、测试产物和本地缓存。
- 归档脚本默认要求干净工作区，避免把临时产物误归档。

### 2.2 整理代码结构

- 新增目录：`scripts/archive/`，集中放置归档自动化脚本。
- 新增文档：`docs/archive/`，集中放置归档类文档。

### 2.3 添加代码注释

- 在 `scripts/archive/project_archive.sh` 增加脚本头部注释，明确用途和行为。

### 2.4 更新代码文档

- 更新 `docs/README.md`，增加“项目收尾与归档”文档导航。

### 2.5 创建代码标签

建议标签命名规范：`archive-vYYYY.MM.DD`

### 2.6 建立代码分支

建议分支命名规范：`archive/project-closeout-YYYYMMDD`

### 2.7 备份代码仓库

推荐使用 `git bundle` 生成全量备份：

```bash
git bundle create backups/repo-archive-<timestamp>.bundle --all
```

### 2.8 编写归档说明

本文件即为归档说明，作为项目移交和审计留档依据。

## 3. 一键归档脚本

执行：

```bash
scripts/archive/project_archive.sh
```

可选参数：

```bash
scripts/archive/project_archive.sh \
  --branch archive/project-closeout-20260414 \
  --tag archive-v2026.04.14 \
  --backup-dir backups
```

## 4. 验证建议

- 执行 `git status --short`，确认归档提交后工作区干净。
- 执行 `git tag -l 'archive-v*'`，确认标签已创建。
- 执行 `git branch --list 'archive/*'`，确认归档分支已创建。
- 校验备份文件：`git bundle verify <bundle_path>`。
