# 代码目录结构总览

## 顶层目录职责

- `apps/`：前端应用与 Electron 启动层。
- `services/backend/`：后端服务、业务 API、数据访问与测试。
- `deep_learning/`：深度学习能力模块与适配层。
- `realtime_interpolation/`：实时插值引擎与 API。
- `multi_objective_optimization/`：多目标优化采样能力。
- `scripts/`：工程脚本（测试、质量检查、维护）。
- `docs/`：项目文档与运维/测试资料。
- `tests/`：前端与系统级测试。

## 归档相关目录

- `scripts/archive/`：归档自动化脚本。
- `docs/archive/`：归档流程与说明文档。
- `backups/`：归档备份产物（`git bundle`）。

## 结构约束

- 运行时缓存、日志、临时文件不进入 Git。
- 归档文档统一放在 `docs/archive/`，避免分散。
- 项目收尾动作尽量脚本化，保证可复现、可审计。
