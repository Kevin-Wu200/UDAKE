# 时空克里金分析模块集成实现说明（对应 04_集成要求）

## 1. 实时插值模块集成
- 新增 `realtime_interpolation/core/incremental_st_kriging.py`。
- `IncrementalSTKriging` 继承 `IncrementalKriging`，复用原有增量框架。
- 扩展了时空协方差计算与 Sherman-Morrison 时空增量更新。
- 提供 `incremental_update_st()` 用于时空批量增量更新。

## 2. 深度学习模块集成
在 `deep_learning/models/spatiotemporal/` 新增：
- `st_gnn.py`：`STGNNPredictor`，封装时空 GNN 推理。
- `st_uncertainty.py`：`STUncertaintyQuantifier`，封装时空不确定性量化。
- `st_fusion.py`：`STModelFusion`，实现克里金与深度学习结果融合。

## 3. GPU 加速模块集成
- 在 `services/backend/app/services/spatiotemporal_kriging_service.py` 中接入 `gpu_service`。
- 训练与预测阶段增加 GPU 探测/执行信息（自动 CPU/GPU 回退）。
- 监控项通过 `performance_metrics()` 返回 GPU 指标。

## 4. 多目标优化模块集成
在 `multi_objective_optimization/` 新增：
- `st_objectives.py`：时空目标函数。
- `st_constraints.py`：时空约束条件。
- `st_sampling_optimizer.py`：NSGA-II 时空采样优化器。

并在 `SpatiotemporalKrigingService.auto_select_model()` 中返回 `sampling_plan`。

## 5. 移动端 GPS 模块集成
- `services/backend/app/api/移动端GPS接口.py` 新增：
  - `POST /api/mobile-gps/spatiotemporal/predict`
- 通过 `predict_from_mobile_samples()` 将移动端样本转换为时空训练数据，完成训练与预测闭环。

## 6. 测试与验证
新增测试：
- `realtime_interpolation/tests/test_incremental_st_kriging.py`
- `services/backend/tests/test_spatiotemporal_module_integration.py`
- `multi_objective_optimization/tests/test_spatiotemporal_optimizer.py`

覆盖模块间关键链路：
- 实时插值 + 时空增量更新
- 时空克里金 + GPU信息输出
- 自动选型 + 时空采样优化
- 移动端GPS样本 -> 时空预测

## 7. 兼容性说明
- 新增集成使用“可选导入 + 回退策略”，在依赖不可用时不影响核心服务可用性。
- 未改变既有 API 路由语义，仅新增能力与返回字段。
