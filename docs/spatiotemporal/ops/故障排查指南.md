# 时空克里金模块故障排查指南

更新时间：2026-04-05

## 1. 常见故障
- 502 Bad Gateway
- 后端健康检查失败
- 数据库超时
- Redis 连接中断
- 预测接口响应慢

## 2. 排查步骤
1. `docker compose -f deployment/spatiotemporal_kriging/docker-compose.yml ps`
2. `docker compose -f deployment/spatiotemporal_kriging/docker-compose.yml logs --tail=300 backend nginx postgres redis`
3. `curl -v http://127.0.0.1/health`
4. `docker compose -f deployment/spatiotemporal_kriging/docker-compose.yml exec postgres pg_isready -U udake -d udake_kriging`
5. `docker compose -f deployment/spatiotemporal_kriging/docker-compose.yml exec redis redis-cli ping`

## 3. 解决方案
- 502：通常是 backend 未就绪，先看 backend 日志
- DB 超时：扩容连接池，检查慢查询
- Redis 异常：重启 redis 并确认 `REDIS_URL` 正确
- 接口慢：观察 CPU/内存和请求并发，调整资源配额

## 4. 预防措施
- 设置自动备份与校验
- 设置 Prometheus 告警
- 更新前先执行 `backup.sh`
