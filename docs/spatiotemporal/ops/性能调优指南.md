# 时空克里金模块性能调优指南

更新时间：2026-04-05

## 1. 性能分析
- API 层：关注 `/api/spatiotemporal/*` 的 P95/P99
- 计算层：关注时空协方差矩阵构建耗时
- 存储层：关注 PostgreSQL 慢查询与 Redis 命中率

## 2. 调优方法
- 应用：提高并发 worker（按 CPU 核数评估）
- 缓存：启用 `REDIS_ENABLED=True` 并优化键过期策略
- 数据库：为高频筛选字段建立索引，定期 VACUUM
- 计算：大样本任务优先离线批处理

## 3. 参数建议
- `WORKFLOW_REDIS_POOL_SIZE=20`
- `AUTH_DB_POOL_SIZE=10`
- `AUTH_DB_MAX_OVERFLOW=20`
- `LOG_LEVEL=INFO`（线上避免 DEBUG）

## 4. 最佳实践
- 压测后再扩容，避免盲目加资源
- 高峰期前预热热点缓存
- 通过 `update.sh` 分批更新
