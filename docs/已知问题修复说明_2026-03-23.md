# 已知问题修复说明（2026-03-23）

## 本次改动目标
本次修复聚焦于问题清单中的未完成/部分完成项，采用“统一基础设施 + 关键模块接入 + 测试补齐”的方式推进。

## 已落地能力

### 1. 日志治理
- 新增 `apps/frontend/js/utils/Logger.ts`。
- 支持日志级别控制（debug/info/warn/error/silent）。
- 对 `token/password/secret/authorization/cookie` 等敏感字段自动脱敏。
- 支持作用域日志，便于定位问题来源。

### 2. 定时器与事件监听统一管理
- 新增 `apps/frontend/js/utils/RuntimeLifecycle.ts`。
- 支持全局跟踪 `setTimeout/setInterval/addEventListener`。
- 支持按模块作用域创建与清理资源，减少泄漏风险。

### 3. API 稳定性与版本管理
- `APIService` 接入统一缓存策略 `CachePolicy`。
- 修复离线缓存读取缺失 `await` 的问题。
- 增加 API 版本头（默认 `X-API-Version: 2026-03`），支持后端版本演进。

### 4. 配置管理收敛
- 新增 `apps/frontend/js/config/AppConfig.ts` 统一运行时配置与默认值。
- `ConfigService` 改为复用统一默认配置，减少重复硬编码。

### 5. 地图引擎切换健壮性
- 地图初始化/切换超时改为可配置。
- 主流程在切换失败时增加回滚尝试，降低地图不可用风险。

### 6. 状态管理职责梳理
- 新增 `apps/frontend/js/store/StateBridge.ts`。
- 建立 `StateManager` 与 `Store` 的桥接同步，降低双轨状态不一致概率。

### 7. 性能监控补强
- `PerformanceMonitor` 增加用户行为记录（点击/输入/可见性变化）。
- 增加基线对比能力，支持性能回归检测。

### 8. 国际化可运维能力
- `I18n` 增加翻译覆盖率统计、缺失键扫描、缺失键使用统计。

### 9. 移动端适配改进
- `MobileNavigation` 增加安全区适配。
- 触控阈值根据视口动态调整。
- 事件监听纳入生命周期管理，销毁时统一清理。

### 10. 依赖与构建流程
- `package.json` 新增：
  - `build:incremental`（增量类型检查+构建）
  - `deps:check`（依赖树检查）
  - `deps:audit`（生产依赖安全审计）
- `tsconfig` 启用 `incremental` 与 `tsBuildInfoFile`。

## 测试补充
- 新增 `tests/logger-runtime.test.ts`。
- 新增 `tests/cache-policy.test.ts`。
- 补充 `i18n` 和 `performance-monitor` 的新能力测试。
