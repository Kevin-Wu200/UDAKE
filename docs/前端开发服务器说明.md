## 环境配置

### .env配置

项目使用`.env`文件配置环境变量：

```bash
# 应用配置
APP_NAME=智能不确定性驱动空间决策平台
VERSION=1.0.0
DEBUG=True

# 服务器配置
HOST=0.0.0.0
PORT=8000

# CORS配置
BACKEND_CORS_ORIGINS=["http://172.20.10.2:5173","http://172.20.10.2:3000","http://172.20.10.2:5173","http://172.20.10.2:5173"]

# 前端API配置
VITE_API_BASE_URL=http://172.20.10.2:8000
VITE_WS_URL=ws://172.20.10.2:8000
```

### Vite配置

前端开发服务器配置在`vite.config.ts`中：

```typescript
server: {
  port: 5173,
  strictPort: false,
  host: true,
  open: true,
  cors: true,
  proxy: {
    '/api': {
      target: env.VITE_API_BASE_URL || 'http://172.20.10.2:8000',
      changeOrigin: true,
    },
    '/ws': {
      target: env.VITE_WS_URL || 'ws://172.20.10.2:8000',
      ws: true,
      changeOrigin: true,
    },
  },
}
```

## 常见问题

### 1. 端口被占用

**错误信息**: `Port 5173 is in use, trying another one...`

**解决方案**:
- Vite会自动尝试下一个可用端口（如5174）
- 或手动终止占用端口的进程：
  ```bash
  lsof -ti :5173 | xargs kill -9
  ```

### 2. 前端无法访问后端

**可能原因**:
- 后端服务未启动
- CORS配置问题
- 网络地址配置错误

**解决方案**:
```bash
# 检查后端是否运行
curl http://172.20.10.2:8000/health

# 检查网络访问
curl http://172.20.10.2:8000/health

# 查看后端日志
# 后端日志会显示在终端输出中
```

### 3. API代理不工作

**检查点**:
- 确认`vite.config.ts`中的proxy配置正确
- 确认`.env`中的`VITE_API_BASE_URL`配置正确
- 尝试直接访问后端API进行验证

### 4. WebSocket连接失败

**检查点**:
- 确认后端支持WebSocket
- 检查`VITE_WS_URL`配置
- 查看浏览器控制台的WebSocket错误信息

## 停止服务

### 停止前端服务

```bash
# 查找前端进程
ps aux | grep vite

# 终止进程
kill <PID>
# 或
pkill -f vite
```

### 停止后端服务

```bash
# 查找后端进程
ps aux | grep "python.*run.py"

# 终止进程
kill <PID>
# 或
pkill -f "python.*run.py"
```

### 停止所有服务

```bash
# 使用dev.sh脚本时，按Ctrl+C会自动清理所有服务
# 或手动清理
pkill -f vite
pkill -f "python.*run.py"
```

## 开发工作流

### 典型开发流程

1. **启动开发环境**
   ```bash
   cd /Users/wuchenkai/UDAKE
   ./dev.sh
   ```

2. **访问应用**
   - 浏览器打开: http://172.20.10.2:5174

3. **修改代码**
   - 前端代码: 修改`frontend/`目录下的文件

5. **查看日志**
