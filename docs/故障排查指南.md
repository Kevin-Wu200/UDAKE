## 概述

本指南旨在帮助用户快速诊断和解决 UDAKE 平台使用过程中遇到的常见问题。本文档涵盖了安装、配置、运行、API 调用等各个方面的故障排查方法。

### 文档适用范围

- **目标用户**：UDAKE 平台用户、开发人员、运维人员
- **适用版本**：UDAKE 1.0.0+
- **适用平台**：Windows、macOS、Linux

### 常见问题分类

1. **安装配置问题**：环境依赖、GDAL 安装、数据库配置
2. **运行时问题**：启动失败、服务崩溃、内存不足
3. **API 调用问题**：认证失败、参数错误、响应超时
4. **性能问题**：响应慢、计算超时、并发问题
5. **数据问题**：文件上传失败、数据格式错误、数据损坏
6. **模型问题**：训练失败、推理错误、模型加载失败

## 常见错误及解决方案

### 1. 安装配置问题

#### 1.1 GDAL 导入错误

**错误信息**：
```
ImportError: libgdal.so.30: cannot open shared object file: No such file or directory
```

**错误原因**：
- GDAL 库未正确安装
- GDAL 环境变量未正确配置
- GDAL 版本不兼容

**解决方案**：

**macOS**：
```bash
# 安装 GDAL
brew install gdal

# 设置环境变量
export GDAL_LIBRARY_PATH=$(brew --prefix gdal)/lib/libgdal.dylib
export GEOS_LIBRARY_PATH=$(brew --prefix geos)/lib/libgeos_c.dylib

# 重新安装 GDAL Python 包
pip install --force-reinstall GDAL
```

**Ubuntu/Debian**：
```bash
# 安装 GDAL
sudo apt-get install gdal-bin libgdal-dev

# 设置环境变量
export GDAL_LIBRARY_PATH=/usr/lib/libgdal.so
export GEOS_LIBRARY_PATH=/usr/lib/libgeos_c.so

# 重新安装 GDAL Python 包
pip install --force-reinstall GDAL
```

**Windows**：
```bash
# 使用 conda 安装（推荐）
conda install -c conda-forge gdal

# 或下载预编译包
# 访问: https://www.lfd.uci.edu/~gohlke/pythonlibs/#gdal
pip install GDAL‑3.8.0‑cp39‑cp39‑win_amd64.whl
```

**验证安装**：
```python
from osgeo import gdal
print(gdal.__version__)
```

#### 1.2 Python 依赖安装失败

**错误信息**：
```
ERROR: Could not find a version that satisfies the requirement xxx
```

**错误原因**：
- Python 版本不兼容
- pip 版本过旧
- 网络连接问题
- 包名拼写错误

**解决方案**：

```bash
# 升级 pip
pip install --upgrade pip

# 升级 setuptools
pip install --upgrade setuptools

# 清除 pip 缓存
pip cache purge

# 使用国内镜像源
pip install -i https://pypi.tuna.tsinghua.edu.cn/simple -r requirements.txt

# 检查 Python 版本
python --version  # 需要 Python 3.9+
```

#### 1.3 数据库连接失败

**错误信息**：
```
psycopg2.OperationalError: could not connect to server: Connection refused
```

**错误原因**：
- PostgreSQL 服务未启动
- 连接参数配置错误
- 防火墙阻止连接
- 数据库不存在

**解决方案**：

```bash
# 检查 PostgreSQL 服务状态
# macOS
brew services list
brew services start postgresql@14

# Linux
sudo systemctl status postgresql
sudo systemctl start postgresql

# 检查数据库连接
psql -U postgres -d udake

# 检查环境变量
echo $DATABASE_URL

# 测试连接
python -c "import psycopg2; conn = psycopg2.connect('postgresql://user:password@localhost/udake'); print('Connected')"
```

#### 1.4 Redis 连接失败

**错误信息**：


**错误原因**：
- Redis 服务未启动
- 连接参数配置错误
- 端口被占用

**解决方案**：

```bash
# 检查 Redis 服务状态
```

### 2. 运行时问题

#### 2.1 后端启动失败

**错误信息**：
```
uvicorn.error: [Errno 98] Address already in use
```

**错误原因**：
- 端口被占用
- 其他服务正在使用相同端口

**解决方案**：

```bash
# 查找占用端口的进程
lsof -i :8000  # macOS/Linux
netstat -ano | findstr :8000  # Windows

# 杀死进程
kill -9 <PID>  # macOS/Linux
taskkill /PID <PID> /F  # Windows

# 或修改端口配置
# 编辑 configs/.env
PORT=8001
```

#### 2.2 前端构建失败

**错误信息**：
```
Error: Cannot find module 'xxx'
```

**错误原因**：
- node_modules 损坏
- 依赖未完全安装
- Node.js 版本不兼容

**解决方案**：

```bash
# 清除 node_modules
rm -rf node_modules
rm package-lock.json

# 重新安装依赖
npm install

# 升级 Node.js
nvm install 18
nvm use 18

# 清除 npm 缓存
npm cache clean --force
```

#### 2.3 内存不足错误

**错误信息**：
```
MemoryError: Unable to allocate array
```

**错误原因**：
- 数据集过大
- 系统内存不足
- 内存泄漏

**解决方案**：

```bash
# 检查系统内存
free -h  # Linux
vm_stat  # macOS

# 使用分块处理
# 修改插值参数
block_size = 1000  # 减小块大小

# 增加虚拟内存
# Linux
sudo swapon /swapfile

# 使用 GPU 加速
pip install cupy-cuda11x
```

#### 2.4 任务执行超时

**错误信息**：
```
TimeoutError: Task execution timeout
```

**错误原因**：
- 任务配置的超时时间过短
- 计算资源不足
- 数据集过大

**解决方案**：

```bash
# 增加任务超时时间
# 编辑 configs/.env
TASK_TIMEOUT=7200  # 2小时

# 使用 GPU 加速
export CUDA_VISIBLE_DEVICES=0

# 使用分布式计算
# 提交分布式任务

```

### 3. API 调用问题

#### 3.1 认证失败

**错误信息**：
```
401 Unauthorized
```

**错误原因**：
- 令牌过期
- 令牌无效
- 用户名或密码错误

**解决方案**：

```bash
# 重新登录
curl -X POST "http://localhost:8000/api/v2/auth/login" \
  -H "Content-Type: application/json" \
  -d '{
    "username": "admin",
    "password": "password"
  }'

# 检查令牌是否过期
# 解码 JWT 令牌
echo "YOUR_TOKEN" | cut -d'.' -f2 | base64 -d

# 刷新令牌
curl -X POST "http://localhost:8000/api/v2/auth/refresh" \
  -H "Content-Type: application/json" \
  -d '{
    "refresh_token": "YOUR_REFRESH_TOKEN"
  }'
```

## 日志分析

### 1. 日志级别

| 级别 | 说明 | 使用场景 |
|------|------|---------|
| DEBUG | 详细调试信息 | 开发调试 |
| INFO | 一般信息 | 正常运行 |
| WARNING | 警告信息 | 潜在问题 |
| ERROR | 错误信息 | 需要处理 |
| CRITICAL | 严重错误 | 系统崩溃 |

### 2. 日志位置

| 日志类型 | 路径 | 说明 |
|---------|------|------|
| 后端日志 | `logs/udake_backend.log` | 后端服务日志 |
| 前端日志 | `logs/udake_frontend.log` | 前端应用日志 |
| 错误日志 | `logs/error.log` | 错误信息日志 |
| 访问日志 | `logs/access.log` | API 访问日志 |
| 训练日志 | `logs/training.log` | 模型训练日志 |

### 3. 日志格式

```
[2026-04-13 02:20:00] [INFO] [request_id: abc123] API request received: GET /api/v2/health
[2026-04-13 02:20:01] [DEBUG] [request_id: abc123] Processing request...
[2026-04-13 02:20:02] [INFO] [request_id: abc123] Request completed: 200 OK
```

### 4. 日志分析工具

#### 使用 grep 搜索日志

```bash
# 搜索错误日志
grep "ERROR" logs/error.log

# 搜索特定请求 ID
grep "abc123" logs/udake_backend.log

# 统计错误数量
grep -c "ERROR" logs/error.log

# 查看最近 100 行错误
tail -n 100 logs/error.log
```

#### 使用 awk 分析日志

```bash
# 统计每个 API 的调用次数
awk '{print $7}' logs/access.log | sort | uniq -c | sort -rn

# 计算平均响应时间
awk '{sum+=$NF; count++} END {print sum/count}' logs/access.log

# 查找慢请求
awk '$NF > 1000 {print}' logs/access.log
```

#### 使用 logrotate 管理日志

```bash
# 编辑 logrotate 配置
sudo vim /etc/logrotate.d/udake

# 配置内容
/path/to/logs/*.log {
    daily
    rotate 7
    compress
    delaycompress
    missingok
    notifempty
    create 644 udake udake
}

# 测试配置
sudo logrotate -d /etc/logrotate.d/udake
```
