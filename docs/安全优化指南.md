# 密钥验证安全优化指南

## 已落地项
- IP信誉评分系统（成功加分、失败扣分、触发限流重扣分）
- 黑白名单自动写入
- 审计日志结构化记录（IP、UA、脱敏密钥、耗时、结果）
- 密钥和IP脱敏输出
- AES-GCM 数据库字段加密（product_key_ciphertext）
- 配置文件热重载（watchdog + SIGHUP）
- 审计日志 HMAC 签名链防篡改
- 增强 ABAC 访问控制策略（8 条安全策略）
- KMS 密钥轮换机制

## 风险控制
- 暴力验证通过三级限流阻断（IP/密钥/用户）
- 低信誉IP短时封禁
- 审计日志用于追溯攻击行为
- TAMM 签名链保证日志完整性
- 配置热重载确保运行时策略即时更新

## 数据库字段加密
- **加密算法**: AES-256-GCM
- **密钥管理**: KMS 多密钥版本管理（格式: `kmsf:v1:{key_id}:{ciphertext}`）
- **已加密字段**: product_keys.product_key_ciphertext
- **待评估字段**: users.email（需 Email 服务适配层）
- **迁移工具**: `auth_db/field_encryption_migration.py`
  - `--dry-run` 试运行
  - `--rollback` 回滚
  - `--verify` 验证一致性

## 配置热重载
- **触发方式**: watchdog 文件监听 / SIGHUP 信号 / 定时轮询（降级）
- **版本管理**: 配置版本号 + diff 对比 + 一键回滚
- **并发安全**: RLock + 原子版本切换
- **回调机制**: 各运行时模块注册重载回调

## 建议
- 生产环境接入集中日志系统（ELK/ClickHouse）
- 配置告警通知到值班渠道
- 密钥轮换周期建议 6 个月
- 定期导出审计签名根到外部可信存储
