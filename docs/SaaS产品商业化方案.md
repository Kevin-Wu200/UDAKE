## 一、产品定位与价值主张

### 1.1 产品定位

UDAKE 是一个智能不确定性驱动的自适应克里金引擎平台，为环境监测、地质勘探、城市规划、农业科学等领域提供高效、精准、可视化的空间数据分析决策支持。

### 1.2 核心价值

- **降低门槛**：自动参数推荐 + AI 助手，非 GIS 专家也能使用
- **提升效率**：GPU 加速 + 分布式计算，分析速度提升 10-100 倍
- **决策支撑**：不确定性量化 + 风险分析，从"插值"到"决策"
- **全栈覆盖**：数据上传 → 分析 → 可视化 → 报告，一站式工作流

### 1.3 目标用户

| 用户群体 | 典型角色 | 核心需求 |
|---------|---------|---------|
| 科研机构 | 环境科学研究者 | 快速试验不同模型、发表论文 |
| 政府环保部门 | 环境监测工程师 | 土壤/水质/空气污染空间分析 |
| 地质勘探企业 | 地质工程师 | 矿产资源分布预测 |
| 农业科技公司 | 精准农业工程师 | 土壤养分空间分布、施肥建议 |
| 城市规划院 | GIS 分析师 | 城市热岛、噪声污染空间分析 |

## 三、多租户架构设计

### 3.1 数据隔离方案

```
┌────────────────────────────────────────────┐
│            共享应用层                        │
│    (同一套 backend + frontend)               │
└───────────────┬────────────────────────────┘
                │
    ┌───────────┼───────────┐
    ▼           ▼           ▼
┌────────┐ ┌────────┐ ┌────────┐
│Schema A│ │Schema B│ │Schema C│
│(Tenant1)│ │(Tenant2)│ │(Tenant3)│
└────────┘ └────────┘ └────────┘
         PostgreSQL (Schema-per-Tenant)
```

### 3.2 隔离策略

| 维度 | 策略 | 说明 |
|------|------|------|
| 数据库 | Schema-per-Tenant | 每个租户独立 PostgreSQL Schema |
| 文件存储 | 目录隔离 | `data/{tenant_id}/` |
| 缓存 | 前缀隔离 | `{tenant_id}:key` |
| 认证 | JWT + Tenant Claim | JWT 中包含 tenant_id |

### 3.3 多租户中间件

```python
# FastAPI 中间件示例
async def tenant_middleware(request: Request, call_next):
    tenant_id = request.headers.get("X-Tenant-ID")
    if not tenant_id:
        raise HTTPException(status_code=400, detail="Missing tenant")
    # 设置当前请求的租户上下文
    request.state.tenant_id = tenant_id
    # 切换到租户 Schema
    await set_tenant_schema(tenant_id)
    return await call_next(request)
```
