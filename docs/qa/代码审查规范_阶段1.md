# 代码审查规范（阶段1）

## 1. 目标与范围

本规范用于落实《09_质量保证_1》中的代码审查任务，覆盖以下代码域：

- 核心代码：`deep_learning/`、`realtime_interpolation/`、`multi_objective_optimization/`、`adaptive_sampling/`
- API代码：`services/`
- 前端代码：`apps/frontend/`、`apps/admin-frontend/`
- 测试代码：`tests/`
- 配置代码：`configs/`、`deployment/`

## 2. 审查准入标准

合入前必须满足以下条件：

1. 通过 `npm run quality:check`。
2. 新增/修改代码具备对应测试，且测试可稳定复现。
3. 无 `critical/high` 级安全问题。
4. 无未解释的 `TODO/FIXME/HACK`。

## 3. 审查维度

1. 正确性：边界条件、异常路径、并发一致性。
2. 安全性：输入校验、权限校验、敏感信息保护、日志脱敏。
3. 性能：复杂度、内存占用、I/O与网络开销。
4. 可维护性：命名、模块职责、重复逻辑、注释质量。
5. 可测试性：依赖隔离、可观测性、回归用例完整度。

## 4. 审查分级规则

- `critical`：可导致敏感数据泄露、远程代码执行、严重业务中断。
- `high`：明显安全风险或高概率故障路径。
- `medium`：功能/性能/可维护性缺陷，短期需修复。
- `low`：风格、日志、文档或可读性改进项。

## 5. 执行流程

1. 运行自动审查：`npm run qa:phase1:audit`。
2. 在 `docs/qa/代码审查问题跟踪_阶段1.md` 记录问题。
3. 责任人提交修复并关联问题编号。
4. 审查人复核，确认“修复结果 + 回归证据”。
5. 在测试报告中更新结论与剩余风险。

## 6. 交付物

- 审查问题清单与状态跟踪。
- 安全审计报告（阶段1）。
- 性能调优报告（阶段1）。
- 用户体验与可访问性测试报告（阶段1）。
