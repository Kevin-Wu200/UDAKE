# 部署指南（管理员后台访问）

本文档补充管理员后台初始化与访问步骤，适用于开发/测试环境。

## 1. 环境配置检查

确认 `configs/env/.env` 中关键项：

- `HOST=127.0.0.1`
- `PORT=8000`
- `VITE_API_BASE_URL`（前端实际以 `apps/admin-frontend/.env.development` 为准）

前端开发环境默认接口地址：

- `apps/admin-frontend/.env.development` -> `VITE_API_BASE_URL=http://127.0.0.1:8000/api`

## 2. 启动服务

后端启动后执行健康检查：

```bash
curl http://127.0.0.1:8000/health
```

前端启动后检查：

```bash
curl http://127.0.0.1:5173/
```

## 3. 初始化管理员账号

推荐使用脚本：

```bash
python3 scripts/init_admin.py \
  --base-url http://127.0.0.1:8000 \
  --email 1447954419@qq.com \
  --password 'Zafu@104021' \
  --product-key UDAKE-DEFAULT-PRODUCT-KEY
```

脚本会自动执行：

1. 健康检查
2. 注册账号
3. 登录验证并确认管理员角色

## 4. 前端登录验证

访问：

- `http://127.0.0.1:5173/#/login`

验证项：

- 登录成功后跳转到 `/dashboard`
- `localStorage` 写入 `udake_access_token` 与 `udake_user_info`
- 非管理员角色登录会被阻止并提示无管理员权限

## 5. 注意事项

- `AUTH_BOOTSTRAP_PRODUCT_KEY` 默认值为 `UDAKE-DEFAULT-PRODUCT-KEY`。
- 默认引导密钥仅可首次注册使用（使用后状态变为 `active`）。
- 当前认证服务数据为进程内存态，服务重启后账号与会话会丢失。

## 6. 企业管理员权限系统增量部署（2026-04-15）

### 6.1 新增环境变量
请同步 `configs/.env.example` 中以下变量：
- SMTP：`SMTP_HOST`、`SMTP_PORT`、`SMTP_USER`、`SMTP_PASSWORD`
- 调度：`KEY_EXPIRY_SCHEDULER_ENABLED`、`KEY_EXPIRY_SCHEDULER_EXPIRE_TIME`、`KEY_EXPIRY_SCHEDULER_REMINDER_TIME`
- 重试：`KEY_EXPIRY_REMINDER_RETRY_TIMES`、`KEY_EXPIRY_REMINDER_RETRY_INTERVAL_SECONDS`

### 6.2 定时任务
后端启动后自动启动调度器，默认任务：
- `expiry_check`：每日执行过期检查
- `expiry_reminder`：每日发送到期提醒

可通过 `/api/scheduler/jobs` 与 `/api/scheduler/history` 观察任务状态。

### 6.3 SMTP 页面与邮件日志
管理员前端新增：
- `/#/smtp-settings`
- `/#/email-logs`

上线前请完成连接测试并验证失败邮件重发。
