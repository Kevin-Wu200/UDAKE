## 技术架构

### 设计模式

采用 **适配器模式（Adapter Pattern）**：

```
业务层（图层管理、主程序）
        ↓
    MapAdapter（统一接口）
        ↓
    ┌───────┴───────┐
    ↓               ↓
ArcGISAdapter   TiandituAdapter
    ↓               ↓
ArcGIS API      Leaflet + 天地图
```

### 核心组件

#### 1. 配置层
- `map.config.js` - 地图引擎选择配置
- `arcgis.config.js` - ArcGIS 参数配置
- `tianditu.config.js` - 天地图参数配置

#### 2. 适配器层
- `MapAdapter.js` - 抽象基类，定义统一接口
- `ArcGISAdapter.js` - ArcGIS 实现
- `TiandituAdapter.js` - 天地图实现

#### 3. 业务层
- `地图初始化.js` - 根据配置动态选择适配器
- `图层管理.js` - 使用适配器接口操作地图
- `主程序.js` - 应用入口，初始化流程

## 实施细节

### 1. 配置文件创建

**map.config.js**
```javascript
export const MapConfig = {
    MAP_PROVIDER: 'tianditu',  // 'arcgis' 或 'tianditu'
    getProvider() { return this.MAP_PROVIDER; },
    isArcGIS() { return this.MAP_PROVIDER === 'arcgis'; },
    isTianditu() { return this.MAP_PROVIDER === 'tianditu'; }
};
```

**tianditu.config.js**
- 配置天地图服务 URL
- 定义默认中心点和缩放级别
- 支持 Token 配置
- 提供 Mock 模式检测

### 2. 适配器接口定义

定义了 13 个统一方法：
- `initMap()` - 初始化地图
- `getView()` - 获取地图视图
- `addPointsLayer()` - 添加点图层
- `addRasterLayer()` - 添加栅格图层
- `addMarker()` - 添加标记
- `addPolygon()` - 添加多边形
- `toggleLayer()` - 切换图层可见性
- `setLayerOpacity()` - 设置透明度
- `removeLayer()` - 移除图层
- `clearAllLayers()` - 清除所有图层
- `zoomToLayer()` - 缩放到图层
- `setClickHandler()` - 设置点击事件
- `getSamplingPoints()` - 获取采样点

### 3. ArcGIS 适配器实现

- 封装了原有的 ArcGIS 地图初始化逻辑
- 保留了所有 ArcGIS 特性（深色模式、Graphics 图层等）
- 实现了完整的图层管理功能
- 支持 GeoJSON 和栅格图层

### 4. 天地图适配器实现

- 使用 Leaflet 作为地图引擎
- 加载天地图矢量底图和注记图层
- 实现了点图层、标记、多边形等功能
- 提供了栅格图层接口（需要插件支持）

### 5. 核心模块重构

**地图初始化.js**
- 根据配置动态选择适配器
- 返回适配器实例而非地图视图
- 提供 `getMapProvider()` 查询当前引擎

**图层管理.js**
- 构造函数接收适配器而非视图
- 所有地图操作通过适配器进行
- 移除了 ArcGIS 特定的代码

**主程序.js**
- 初始化流程适配新架构
- 从适配器获取地图视图
- 传递适配器给图层管理器

### 6. 前端依赖更新

- 安装 Leaflet 1.9.4
- 在 index.html 中引入 Leaflet CSS 和 JS
- 保留 ArcGIS API 引用（按需加载）

### 7. 坐标转换增强

- 添加独立的 WGS-84 ↔ Web Mercator 转换函数
- 保留 ArcGIS 坐标转换接口
- 支持批量坐标转换

## 优势分析

### 1. 架构优势
- ✅ **完全解耦** - 业务代码不依赖具体地图引擎
- ✅ **易于扩展** - 可轻松添加其他地图引擎
- ✅ **向后兼容** - 保留所有 ArcGIS 代码
- ✅ **统一接口** - 降低维护成本

### 2. 开发优势
- ✅ **配置驱动** - 无需修改代码即可切换
- ✅ **Mock 模式** - 无需 Token 也能开发
- ✅ **清晰文档** - 完整的使用和测试文档

### 3. 性能优势
- ✅ **轻量级** - 天地图模式资源占用少
- ✅ **快速加载** - Leaflet 启动速度快
- ✅ **灵活选择** - 根据需求选择引擎

## 已知限制

### 1. 栅格图层支持
- 天地图模式下加载 GeoTIFF 需要额外插件
- 推荐使用 `georaster-layer-for-leaflet`
- 当前实现提供了接口，但未集成插件

### 2. 深色模式
- ArcGIS 支持自动切换深色底图
- 天地图暂不支持深色模式
- 可以通过 CSS 调整界面样式

### 3. Token 配置
- 天地图需要申请 Token
- 未配置 Token 时进入 Mock 模式
- Mock 模式下地图可能无法正常显示
