# 开发指南

## 开发环境配置

### 依赖安装

```bash
cd dist
npm install
```

### 启动开发服务器

```bash
bash dist/scripts/dev.sh
```

应用基于 Electron，启动后会自动打开 DevTools（开发模式）。

## 代码结构

### 后端

后端基于 FastAPI，提供 RESTful API 服务。基础 URL：`http://localhost:8000/api`，数据格式为 JSON，字符编码 UTF-8。

### 前端

前端使用原生 JavaScript（ES6 模块），详细文件结构参见 [系统架构.md](系统架构.md)。

## API 接口说明

### 数据上传

`POST /upload-data`

上传空间采样点数据（GeoJSON 文件），Content-Type 为 multipart/form-data。

响应示例：
```json
{
  "data_id": "uuid",
  "point_count": 100,
  "bounds": { "min_x": 116.0, "min_y": 39.0, "max_x": 117.0, "max_y": 40.0 },
  "message": "数据上传成功"
}
```

### 启动克里金任务

`POST /start-kriging`

请求示例：
```json
{
  "data_id": "uuid",
  "method": "ordinary",
  "variogram_model": "spherical",
  "grid_resolution": 100,
  "nlags": 6,
  "enable_cross_validation": true,
  "n_folds": 5
}
```

响应示例：
```json
{
  "task_id": "uuid",
  "status": "pending",
  "message": "克里金任务已启动"
}
```

### 查询任务状态

`GET /task-status/{task_id}`

响应示例：
```json
{
  "task_id": "uuid",
  "status": "running",
  "progress": 45.5,
  "message": "正在执行插值",
  "created_at": "2024-01-01T00:00:00",
  "updated_at": "2024-01-01T00:01:00",
  "error": null
}
```

### 获取预测结果

`GET /result/prediction/{task_id}`

响应示例：
```json
{
  "task_id": "uuid",
  "geotiff_url": "/results/uuid_prediction.tif",
  "statistics": { "mean": 25.5, "std": 5.2, "min": 10.0, "max": 40.0 }
}
```

### 获取方差结果

`GET /result/variance/{task_id}`

响应示例：
```json
{
  "task_id": "uuid",
  "geotiff_url": "/results/uuid_variance.tif",
  "statistics": { "mean": 2.5, "std": 1.2, "min": 0.5, "max": 8.0 }
}
```

### 生成分析报告

`GET /result/report/{task_id}`

响应示例：
```json
{
  "task_id": "uuid",
  "method": "ordinary",
  "variogram_model": "spherical",
  "point_count": 100,
  "grid_resolution": 100,
  "cross_validation": { "rmse": 2.5, "mae": 1.8, "r2": 0.85, "mse": 6.25 },
  "execution_time": 15.5,
  "generated_at": "2024-01-01T00:02:00"
}
```

### 错误码

- 400：请求参数错误
- 404：资源不存在
- 500：服务器内部错误

### 任务状态码

- pending：任务待执行
- running：任务执行中
- completed：任务完成
- failed：任务失败

## 地图引擎开发

### 切换地图引擎

编辑 `frontend/js/config/map.config.js`：

```javascript
MAP_PROVIDER: 'amap'  // 可选值：'amap'、'arcgis'、'tianditu'
```

或使用切换脚本：

```bash
./scripts/switch-map-engine.sh amap
./scripts/switch-map-engine.sh arcgis
./scripts/switch-map-engine.sh tianditu
./scripts/switch-map-engine.sh status  # 查看当前配置
```

### 添加新地图引擎

1. 在 `frontend/js/map/core/` 下创建新引擎类，继承 `BaseMapEngine`
2. 实现所有抽象方法，设置 `supportsCustomReset` 标记
3. 在 `MapManager` 中添加创建逻辑
4. 在 `map.config.js` 中添加引擎选项

### 适配器接口

适配器定义了 13 个统一方法：

- `initMap()` - 初始化地图
- `getView()` - 获取地图视图
- `addPointsLayer()` - 添加点图层
- `addRasterLayer()` - 添加栅格图层
- `addMarker()` - 添加标记
- `addPolygon()` - 添加多边形
- `toggleLayer()` - 切换图层可见性
- `setLayerOpacity()` - 设置透明度
- `removeLayer()` - 移除图层
- `clearAllLayers()` - 清除所有图层
- `zoomToLayer()` - 缩放到图层
- `setClickHandler()` - 设置点击事件
- `getSamplingPoints()` - 获取采样点
