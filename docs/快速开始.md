# UDAKE 地图引擎切换 - 快速开始指南

## 🚀 5 分钟快速上手

### 步骤 1: 选择地图引擎

使用快速切换脚本：

```bash
# 切换到天地图模式（推荐用于开发）
./switch-map-engine.sh tianditu

# 或切换到 ArcGIS 模式
./switch-map-engine.sh arcgis

# 查看当前配置
./switch-map-engine.sh status
```

### 步骤 2: 配置 Token（可选）

#### 天地图模式
如果使用天地图，建议配置 Token（可跳过，使用 Mock 模式）：

1. 访问 https://console.tianditu.gov.cn/ 申请 Token
2. 编辑 `frontend/js/config/tianditu.config.js`
3. 替换 `TOKEN: 'YOUR_TIANDITU_TOKEN_PLACEHOLDER'`

#### ArcGIS 模式
如果使用 ArcGIS，建议配置 API Key（可跳过，使用 Mock 模式）：

1. 访问 https://developers.arcgis.com/ 申请 API Key
2. 编辑 `frontend/js/config/arcgis.config.js`
3. 替换 `API_KEY: "YOUR_ARCGIS_API_KEY_PLACEHOLDER"`

### 步骤 3: 启动应用

```bash
cd dist
npm start
```

### 步骤 4: 验证功能

打开应用后，检查：
- ✅ 地图是否正常显示
- ✅ 控制台是否有错误
- ✅ 能否添加采样点

## 🔧 常见问题

### Q1: 地图无法加载？

**检查项：**
1. 确认已选择正确的地图引擎
2. 检查浏览器控制台错误信息
3. 确认网络连接正常
4. 尝试切换到另一个地图引擎

**解决方法：**
```bash
# 查看当前配置
./switch-map-engine.sh status

# 尝试切换引擎
./switch-map-engine.sh tianditu
```

### Q2: Token 错误？

**解决方法：**
- 未配置 Token 时会进入 Mock 模式，地图仍可使用
- 如需正式使用，请申请并配置真实 Token
- 检查 Token 是否正确复制（无多余空格）

### Q3: 栅格图层无法显示（天地图模式）？

**说明：**
- 这是预期行为
- 天地图模式下加载 GeoTIFF 需要额外插件
- 建议使用 ArcGIS 模式测试栅格功能

**解决方法：**
```bash
# 切换到 ArcGIS 模式
./switch-map-engine.sh arcgis
cd dist && npm start
```

### Q4: 如何切换回原来的引擎？

**解决方法：**
```bash
# 切换到 ArcGIS
./switch-map-engine.sh arcgis

# 切换到天地图
./switch-map-engine.sh tianditu
```

### Q5: 测试工具如何使用？

**方法 1: 自动化测试**
```
在浏览器中打开 frontend/test-map-engine.html
点击"运行所有测试"
```

**方法 2: 手动测试**
```
参考 测试检查清单.md 逐项测试
```

## 📊 功能对比

| 功能 | 天地图模式 | ArcGIS 模式 |
|------|-----------|------------|
| 地图加载 | ✅ 快速 | ✅ 中等 |
| 采样点 | ✅ | ✅ |
| GeoJSON | ✅ | ✅ |
| 栅格图层 | ⚠️ 需插件 | ✅ |
| 深色模式 | ❌ | ✅ |
| 资源占用 | ✅ 低 | ⚠️ 较高 |

**推荐：**
- 开发阶段：使用天地图模式（快速、轻量）
- 生产环境：根据需求选择
- 栅格功能：使用 ArcGIS 模式
