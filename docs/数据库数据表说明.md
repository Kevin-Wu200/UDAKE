### 3.2 `tickets` — 密钥申请工单

产品密钥申请和续期工单管理，包含严格的状态机和业务规则验证。

| 列名 | 类型 | 可空 | 默认值 | 说明 |
|------|------|------|--------|------|
| `id` | BIGINT | NO | 自增 | 主键 |
| `ticket_id` | VARCHAR(32) | NO | — | 工单编号（唯一） |
| `ticket_type` | VARCHAR(50) | NO | — | 工单类型：`key_request` / `key_extension` |
| `status` | VARCHAR(20) | NO | `'pending'` | 状态：`pending` → `approved`/`rejected` → `completed` |
| `email` | VARCHAR(255) | NO | — | 申请人邮箱 |
| `phone` | VARCHAR(20) | NO | — | 申请人电话 |
| `industry` | VARCHAR(100) | NO | — | 所属行业 |
| `organization` | VARCHAR(128) | NO | — | 组织名称 |
| `usage_purpose` | TEXT | NO | — | 使用目的 |
| `key_type` | VARCHAR(50) | NO | — | 申请密钥类型 |
| `existing_key` | VARCHAR(100) | YES | — | 已有密钥（续期时必填） |
| `company_id` | BIGINT | YES | — | 公司 FK → `companies.id` (SET NULL) |
| `processed_by` | BIGINT | YES | — | 处理人 FK → `users.id` (SET NULL) |
| `processed_at` | TIMESTAMPTZ | YES | — | 处理时间 |
| `approval_notes` | TEXT | YES | — | 审批备注 |
| `assigned_key` | VARCHAR(100) | YES | — | 分配的密钥 |
| `response_message` | TEXT | YES | — | 回复消息 |
| `created_at` | TIMESTAMPTZ | NO | `now()` | 创建时间 |
| `updated_at` | TIMESTAMPTZ | NO | `now()` | 更新时间（自动更新） |

**状态流转**: `pending` → `approved` / `rejected` → `completed`  
**业务规则**:
- `key_request` 不允许填 `existing_key`，`key_extension` 必须填 `existing_key`
- `pending` 状态时 `processed_by` 和 `processed_at` 必须为空
- `approved`/`rejected`/`completed` 状态时 `processed_by` 和 `processed_at` 不可为空
- `completed` 状态必须填写 `assigned_key`
- `processed_by` 和 `processed_at` 一旦写入不可修改

**约束**: `PK(id)`, `UNIQUE(ticket_id)`, `CHECK(ticket_type)`, `CHECK(status)`, `CHECK(key_type)`  
**索引**: `ticket_id`, `email`, `status`, `created_at`, `ticket_type`, `processed_by`, `company_id`  
**关联**: ← `users` (处理人), ← `companies`
