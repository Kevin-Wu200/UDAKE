## 本地开发

### 启动应用

#### 开发模式

```bash
bash dist/scripts/dev.sh
```

开发模式下会自动打开 DevTools。

#### 生产模式

```bash
cd dist
npm start
```

### 地图引擎配置

#### 选择地图引擎

编辑 `frontend/js/config/map.config.js`：

```javascript
MAP_PROVIDER: 'amap'  // 可选值：'amap'、'arcgis'、'tianditu'
```

或使用切换脚本：

```bash
./scripts/switch-map-engine.sh amap      # 高德地图
./scripts/switch-map-engine.sh arcgis    # ArcGIS
./scripts/switch-map-engine.sh tianditu  # 天地图
./scripts/switch-map-engine.sh status    # 查看当前配置
```

#### 配置高德地图

编辑 `frontend/js/config/amap.config.js`，配置 API Key 和安全密钥。

申请地址：[高德开放平台](https://console.amap.com/)

注意事项：
- 安全密钥必须在加载 JS API 之前设置
- 需要在高德开放平台配置域名白名单（添加 `localhost`、`127.0.0.1`）
- Electron 环境需要设置 `webSecurity: false`

#### 配置 ArcGIS

编辑 `frontend/js/config/arcgis.config.js`，配置 API Key。

申请地址：[ArcGIS Developers](https://developers.arcgis.com/)

#### 配置天地图

编辑 `frontend/js/config/tianditu.config.js`，配置 Token。

申请地址：[天地图开发者平台](https://console.tianditu.gov.cn/)

### 部署检查

#### 文件部署

确保以下文件已正确部署到 `dist/build/frontend/js/`：

- `map/core/` - 地图引擎核心文件
- `managers/` - 管理器
- `components/` - UI 组件
- `utils/` - 工具类
- `adapters/` - 适配器
- `config/` - 配置文件

#### 验证步骤

1. 启动应用，确认地图正常显示
2. 检查浏览器控制台无错误
3. 验证采样点添加功能正常
4. 验证插值计算功能正常

## 本地服务器部署

### 前提条件

- macOS系统（其他系统参考对应命令）
- Python 3.9+ 已安装
- Android设备已连接USB调试
- 笔记本和Android设备在同一WiFi网络

### 步骤1：获取笔记本IP地址

#### macOS
```bash
ifconfig | grep "inet " | grep -v 127.0.0.1
```

输出示例：`inet 192.168.1.100`

#### Windows
```bash
ipconfig
```

找到"无线局域网适配器 WLAN"下的IPv4地址

### 步骤2：启动后端服务

```bash
cd /Users/wuchenkai/UDAKE/backend
python run.py
```

后端将启动在 `0.0.0.0:8000`，监听所有网络接口。

**预期输出**：
```
🚀 启动 智能不确定性驱动空间决策平台
📍 地址: http://0.0.0.0:8000
📖 API文档: http://0.0.0.0:8000/docs
```

### 步骤3：配置防火墙

#### macOS 防火墙设置

**方案A：图形界面**
1. 系统偏好设置 → 安全性与隐私 → 防火墙
2. 防火墙选项 → 添加 Python 应用
3. 允许传入连接

**方案B：命令行（临时关闭，仅测试用）**
```bash
sudo /usr/libexec/ApplicationFirewall/socketfilterfw --setglobalstate off
```

**方案C：添加Python到允许列表**
```bash
sudo /usr/libexec/ApplicationFirewall/socketfilterfw --add /usr/local/bin/python3
sudo /usr/libexec/ApplicationFirewall/socketfilterfw --unblock /usr/local/bin/python3
```

### 步骤4：创建环境配置文件

```bash
cd /Users/wuchenkai/UDAKE
cp .env.example .env
```

编辑 `.env` 文件，配置以下关键参数：

```env
# 后端API地址（替换为你的实际IP）
VITE_API_BASE_URL=http://192.168.1.100:8000
VITE_WS_URL=ws://192.168.1.100:8000

# 地图API密钥（必须配置）
VITE_AMAP_API_KEY=你的高德API密钥
VITE_AMAP_SECURITY_CODE=你的高德安全码

# 其他配置
APP_NAME=智能不确定性驱动空间决策平台
DEBUG=True
HOST=0.0.0.0
PORT=8000
```

### 步骤5：测试后端连接

#### 在笔记本上测试

```bash

curl http://localhost:8000/health

# 测试局域网访问（使用实际IP）
curl http://192.168.1.100:8000/health
```

**预期输出**：`{"status": "healthy"}`

#### 从Android设备测试

1. 打开手机浏览器
2. 访问：`http://192.168.1.100:8000/docs`
3. 应该能看到 FastAPI 的交互式文档页面

### 步骤6：构建前端和APK

```bash
cd /Users/wuchenkai/UDAKE

# 构建生产版本前端
npm run build:prod

# 构建Android APK（包含以下步骤）
# 1. 更新 Capacitor 配置
# 2. 同步到 Android 项目
```

APK生成位置：`android/app/build/outputs/apk/release/app-release.apk`

### 优缺点对比

#### 优点

- ✅ **免费**：无需购买云服务器
- ✅ **低延迟**：本地网络传输速度快
- ✅ **数据安全**：数据在本地，不上传云端
- ✅ **完全控制**：可以随时修改配置和代码
- ✅ **适合开发**：便于调试和测试

#### 缺点

- ⚠️ **依赖笔记本**：需要保持笔记本运行
- ⚠️ **网络限制**：只能在同一WiFi网络使用
- ⚠️ **休眠影响**：笔记本休眠时服务不可用
- ⚠️ **防火墙配置**：需要正确配置防火墙
- ⚠️ **IP变化**：IP地址可能改变

## 云服务器部署

### 概述

UDAKE（智能不确定性驱动空间决策平台）是一个基于克里金插值的空间分析系统，包含以下组件：

- **后端服务**：FastAPI + PyKrige + NumPy/SciPy
- **前端应用**：Vite + 原生JavaScript
- **地图引擎**：支持高德地图、ArcGIS、天地图
- **移动应用**：Android（Capacitor）
- **桌面应用**：Electron

### 服务器要求

#### 最低配置（测试环境）

| 资源 | 要求 |
|------|------|
| CPU | 2核心 |
| 内存 | 4GB |
| 存储 | 40GB SSD |
| 带宽 | 5Mbps |
| 操作系统 | Ubuntu 20.04+ / CentOS 8+ |

#### 推荐配置（生产环境）

| 资源 | 要求 |
|------|------|
| CPU | 4核心+ |
| 内存 | 8GB+ |
| 存储 | 100GB SSD |
| 带宽 | 10Mbps+ |
| 操作系统 | Ubuntu 22.04 LTS / CentOS 9 Stream |

#### 软件依赖

- Docker 20.10+（Docker部署）
- Docker Compose 2.0+（Docker部署）
- Python 3.9+（传统部署）
- Node.js 18+（传统部署）
- Nginx 1.20+
- Git

### 部署方案选择

#### 方案对比

| 方案 | 优点 | 缺点 | 适用场景 |
|------|------|------|----------|
| **Docker部署** | 环境隔离、部署快速、易于维护 | 需要Docker知识 | 推荐使用 |
| **传统部署** | 完全控制、资源占用少 | 配置复杂、依赖管理麻烦 | 无Docker环境 |
| **混合部署** | 灵活性高 | 运维复杂 | 特殊需求 |

#### 推荐选择

**生产环境推荐：Docker部署**
- 环境一致性保证
- 快速部署和回滚
- 便于扩展和迁移

### 方案一：Docker部署（推荐）

#### 1. 准备服务器

```bash
# 更新系统
sudo apt update && sudo apt upgrade -y

# 安装基础工具
sudo apt install -y git curl wget vim

# 克隆项目
git clone <your-repo-url> /opt/udake
cd /opt/udake
```

#### 2. 安装Docker

```bash
# 安装Docker
curl -fsSL https://get.docker.com -o get-docker.sh
sudo sh get-docker.sh

# 安装Docker Compose
sudo curl -L "https://github.com/docker/compose/releases/latest/download/docker-compose-$(uname -s)-$(uname -m)" -o /usr/local/bin/docker-compose
sudo chmod +x /usr/local/bin/docker-compose

# 验证安装
docker --version
docker-compose --version

# 将当前用户添加到docker组
sudo usermod -aG docker $USER
newgrp docker
```

#### 3. 创建后端Dockerfile

在项目根目录创建 `backend/Dockerfile`：

```dockerfile
FROM python:3.9-slim

# 设置工作目录
WORKDIR /app

# 安装系统依赖
RUN apt-get update && apt-get install -y \
    gdal-bin \
    libgdal-dev \
    gcc \
    g++ \
    && rm -rf /var/lib/apt/lists/*

# 复制依赖文件
COPY requirements.txt .

# 安装Python依赖
RUN pip install --no-cache-dir -r requirements.txt

# 复制应用代码
COPY . .

# 暴露端口
EXPOSE 8000

# 启动命令
CMD ["python", "run.py"]
```

#### 4. 创建前端Dockerfile

在项目根目录创建 `frontend/Dockerfile`：

```dockerfile
FROM node:18-alpine AS builder

WORKDIR /app

# 复制package文件
COPY package*.json ./

# 安装依赖
RUN npm ci

# 复制源代码
COPY . .

# 构建生产版本
RUN npm run build:prod

# 生产镜像
FROM nginx:alpine

# 复制构建产物
COPY --from=builder /app/dist /usr/share/nginx/html

# 复制nginx配置
COPY deployment/nginx.conf /etc/nginx/nginx.conf

EXPOSE 80

CMD ["nginx", "-g", "daemon off;"]
```

#### 5. 配置环境变量

```bash
# 复制环境变量模板
cp deployment/生产环境变量.env .env

# 编辑环境变量
vim .env
```

关键配置项：

```env
# 应用配置

# 服务器配置

# CORS配置

# 任务配置
MAX_CONCURRENT_TASKS=10

```


```ini
Description=UDAKE Backend Service
After=network.target

[Service]
Type=simple
User=www-data
WorkingDirectory=/opt/udake/backend
Environment="PATH=/opt/udake/venv/bin"
ExecStart=/opt/udake/venv/bin/python run.py
Restart=always
RestartSec=10
```


### Nginx配置

#### 完整配置文件

```nginx
# /etc/nginx/sites-available/udake

# 后端API代理
upstream backend {
    server 127.0.0.1:8000;
    keepalive 32;
}

# HTTP服务器
server {
    listen 80;
    server_name your-domain.com www.your-domain.com;

    # 重定向到HTTPS
    return 301 https://$server_name$request_uri;
}

# HTTPS服务器
server {
    listen 443 ssl http2;
    server_name your-domain.com www.your-domain.com;

    # SSL证书配置
    ssl_certificate /etc/letsencrypt/live/your-domain.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/your-domain.com/privkey.pem;
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers HIGH:!aNULL:!MD5;
    ssl_prefer_server_ciphers on;

    # 安全头部
    add_header X-Frame-Options "SAMEORIGIN" always;
    add_header X-Content-Type-Options "nosniff" always;
    add_header X-XSS-Protection "1; mode=block" always;
    add_header Strict-Transport-Security "max-age=31536000; includeSubDomains" always;

    # 日志配置
    access_log /var/log/nginx/udake_access.log;
    error_log /var/log/nginx/udake_error.log;

    # 客户端上传大小限制
    client_max_body_size 100M;

    # 前端静态文件
    location / {
        root /opt/udake/frontend/dist;
        index index.html;
        try_files $uri $uri/ /index.html;

        # 静态资源缓存
        location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2|ttf|eot)$ {
            expires 1y;
            add_header Cache-Control "public, immutable";
        }
    }

    # API代理
    location /api/ {
        proxy_pass http://backend/api/;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_cache_bypass $http_upgrade;
        proxy_read_timeout 300s;
        proxy_connect_timeout 75s;
    }

    # WebSocket代理
    location /ws/ {
        proxy_pass http://backend/ws/;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_read_timeout 86400;
    }

    # 结果文件
    location /results/ {
        alias /opt/udake/backend/app/结果文件/;
        autoindex off;
        add_header Content-Type application/octet-stream;
        add_header Content-Disposition "attachment";
    }

    # Gzip压缩
    gzip on;
    gzip_vary on;
    gzip_min_length 1000;
    gzip_comp_level 6;
    gzip_types text/plain text/css application/json application/javascript text/xml application/xml application/xml+rss text/javascript;
}
```

```bash
# 测试局域网连接
curl http://192.168.1.100:8000/health
```

### 问题2：防火墙阻止连接

**症状**：本地可以访问，但局域网其他设备无法访问

**解决方案**：
```bash

sudo /usr/libexec/ApplicationFirewall/socketfilterfw --setglobalstate off

# 添加Python到允许列表
sudo /usr/libexec/ApplicationFirewall/socketfilterfw --add /usr/local/bin/python3
sudo /usr/libexec/ApplicationFirewall/socketfilterfw --unblock /usr/local/bin/python3
```

### 问题3：IP地址变化

**症状**：重启后无法连接，IP地址改变

**解决方案**：

**方案A：路由器静态IP分配**
1. 登录路由器管理界面
2. 找到DHCP设置
3. 为笔记本MAC地址分配固定IP（如 `192.168.1.100`）

**方案B：使用mDNS/Bonjour（macOS）**
```bash
# 在.env中使用主机名

VITE_WS_URL=ws://你的笔记本名.local:8000
```

注意：Android设备需要支持mDNS（部分设备可能不支持）

### 问题4：地图无法显示

**原因**：地图API密钥未配置或无效

**解决方案**：
1. 检查 `.env` 文件中的API密钥配置
2. 确认密钥已激活且未过期
3. 检查密钥是否有域名限制（本地测试应无限制）

## 错误分析和实施方案

### 错误概览

根据 Console.log 的分析，发现了以下几类错误：

#### 1. 核心错误（最严重）：`Cannot read properties of null (reading 'postMessage')`

**错误信息**：
```
地图引擎切换失败: TypeError: Cannot read properties of null (reading 'postMessage')
    at Object.setStatus (map-amap-zFu4M8h7.js:148:1162)
```

**发生场景**：
1. 用户从 arcgis 切换到 amap
2. amap 开始初始化，创建 iframe
3. 用户立即又从 amap 切换回 arcgis（或多次快速切换）
4. 切换逻辑清理了旧的 iframe
5. 但 amap 的初始化过程仍在继续执行（异步操作）
6. 当 amap 的 `setStatus` 方法被调用时，iframe 已经被销毁
7. 导致 `iframe.contentWindow` 为 null，调用 `postMessage` 时出错

**根本原因**：
- 地图引擎切换时的竞态条件（Race Condition）
- 异步初始化操作没有被正确取消
- iframe 的清理和地图初始化是并发的，没有等待机制

### 实施方案

#### 方案一：修复地图引擎切换的竞态条件（高优先级）

**问题**：快速切换地图引擎时，iframe 被销毁但异步操作仍在继续

**实施步骤**：

1. **添加取消令牌（AbortController）**
   - 在 `amap.config.js` 中使用 AbortController 控制异步操作
   - 在切换地图引擎时取消所有未完成的异步操作
