## 概述

UDAKE（Uncertainty-Driven Adaptive Kriging Engine）是一个智能不确定性驱动的自适应克里金引擎平台，支持Web、桌面和移动端多平台部署。

### 支持的平台

- **Web应用**: 浏览器访问
- **Electron桌面应用**: macOS、Windows、Linux
- **Android移动应用**: Android 5.0+

### 环境变量配置

本项目的前端与域名替换流程统一使用 `configs/env/` 目录下的环境文件。建议按环境维护以下文件：

- 开发：`configs/env/.env.development`（Vite 还会合并读取 `configs/env/.env`）
- 测试：`configs/env/.env.testing`
- 生产：`configs/env/.env.production`

关键域名变量：

| 变量名 | 必填 | 用途 |
|------|------|------|
| `OFFICIAL_WEB` | 是 | 官网域名，用于文档、视频、社区、更新链接拼接 |
| `ADMIN_WEB` | 是 | 管理后台域名，用于后台入口与配置注入 |

`OFFICIAL_WEB` 和 `ADMIN_WEB` 必须同时存在；`scripts/replace-domains.js` 会在构建后校验这两个变量，缺失会直接失败。

推荐同时配置以下 API 相关变量：

| 变量名 | 用途 |
|------|------|
| `VITE_API_BASE_URL` | 前端 API 基础地址（建议不带 `/api`） |
| `VITE_API_URL` | 兼容旧代码（可带 `/api`） |
| `VITE_WS_URL` | 前端 WebSocket 地址 |

开发、测试、生产示例：

```bash
# configs/env/.env.development
VITE_API_BASE_URL=http://127.0.0.1:8000
VITE_WS_URL=ws://127.0.0.1:8000
OFFICIAL_WEB=https://dev.udake.local
ADMIN_WEB=https://admin-dev.udake.local

# configs/env/.env.testing
VITE_API_BASE_URL=https://api-test.udake.xyz
VITE_WS_URL=wss://api-test.udake.xyz
OFFICIAL_WEB=https://test.udake.xyz
ADMIN_WEB=https://admin-test.udake.xyz

# configs/env/.env.production
VITE_API_BASE_URL=https://api.udake.xyz
VITE_WS_URL=wss://api.udake.xyz
OFFICIAL_WEB=https://udake.xyz
ADMIN_WEB=https://admin.udake.xyz
```

### 构建时域名替换机制

`npm run build:dev`、`npm run build:test`、`npm run build:prod` 在 Vite 构建后会自动执行：

```bash
node scripts/replace-domains.js --mode=<development|testing|production>
```

替换脚本会将以下固定域名替换为基于 `OFFICIAL_WEB` 的占位路径，并输出到 `dist/domain-replaced/`：

- `https://docs.udake.local/workflow/*` -> `${OFFICIAL_WEB}/docs/workflow/*`
- `https://video.udake.local/tutorials/*` -> `${OFFICIAL_WEB}/videos/tutorials/*`
- `https://community.udake.io` -> `${OFFICIAL_WEB}/community`
- `https://update.udake.com/releases` -> `${OFFICIAL_WEB}/updates/releases`

同时，`configs/vite.config.ts` 会把 `OFFICIAL_WEB` / `ADMIN_WEB` 注入为前端可读取的 `import.meta.env.VITE_OFFICIAL_WEB` 与 `import.meta.env.VITE_ADMIN_WEB`。

### 生产环境部署

#### 1. 使用Gunicorn + Uvicorn

安装Gunicorn：

```bash
pip install gunicorn
```

启动服务：

```bash
cd services/backend
gunicorn app.main:app \
  --workers 4 \
  --worker-class uvicorn.workers.UvicornWorker \
  --bind 0.0.0.0:8000 \
  --timeout 120 \
  --access-logfile - \
  --error-logfile - \
  --log-level info
```

#### 2. 使用Systemd管理服务

创建服务文件 `/etc/systemd/system/udake.service`：

```ini
[Unit]
Description=UDAKE Backend Service
After=network.target

[Service]
Type=notify
User=udake
Group=udake
WorkingDirectory=/opt/udake
Environment="PATH=/opt/udake/venv/bin"
ExecStart=/opt/udake/venv/bin/gunicorn app.main:app \
  --workers 4 \
  --worker-class uvicorn.workers.UvicornWorker \
  --bind 0.0.0.0:8000 \
  --timeout 120
Restart=always
RestartSec=10

[Install]
WantedBy=multi-user.target
```

启动服务：

```bash
sudo systemctl daemon-reload
sudo systemctl enable udake
sudo systemctl start udake
sudo systemctl status udake
```

#### 3. 使用Supervisor管理服务

安装Supervisor：

```bash
pip install supervisor
```

创建配置文件 `/etc/supervisor/conf.d/udake.conf`：

```ini
[program:udake]
command=/opt/udake/venv/bin/gunicorn app.main:app --workers 4 --worker-class uvicorn.workers.UvicornWorker --bind 0.0.0.0:8000
directory=/opt/udake
user=udake
autostart=true
autorestart=true
redirect_stderr=true
stdout_logfile=/var/log/udake.log
stdout_logfile_maxbytes=50MB
stdout_logfile_backups=10
```

启动服务：

```bash
sudo supervisorctl reread
sudo supervisorctl update
sudo supervisorctl start udake
```

## Docker部署

### 创建Dockerfile

#### Backend Dockerfile

创建 `services/backend/Dockerfile`：

```dockerfile
FROM python:3.11-slim

WORKDIR /app

# 安装系统依赖
RUN apt-get update && apt-get install -y \
    gcc \
    g++ \
    gdal-bin \
    libgdal-dev \
    && rm -rf /var/lib/apt/lists/*

# 复制依赖文件
COPY requirements.txt .

# 安装Python依赖
RUN pip install --no-cache-dir -r requirements.txt

# 复制应用代码
COPY app/ ./app/

# 创建日志目录
RUN mkdir -p logs

# 暴露端口
EXPOSE 8000

# 启动命令
CMD ["gunicorn", "app.main:app", "--workers", "4", "--worker-class", "uvicorn.workers.UvicornWorker", "--bind", "0.0.0.0:8000"]
```

#### Frontend Dockerfile

创建 `apps/frontend/Dockerfile`：

```dockerfile
FROM node:18-alpine as builder

WORKDIR /app

# 复制package文件
COPY package*.json ./
COPY configs/tsconfig.json ../configs/
COPY configs/vite.config.ts ../configs/

# 安装依赖
RUN npm ci

# 复制源代码
COPY apps/frontend/js ./apps/frontend/js/

# 构建生产版本
RUN npm run build:prod

# 生产环境
FROM nginx:alpine

# 复制构建产物
COPY --from=builder /app/apps/frontend/dist /usr/share/nginx/html

# 复制Nginx配置
COPY deployment/nginx反向代理配置.conf /etc/nginx/conf.d/default.conf

# 暴露端口
EXPOSE 80

# 启动Nginx
CMD ["nginx", "-g", "daemon off;"]
```

### Docker Compose配置

更新 `deployment/docker-compose.yml`：

```yaml
version: '3.8'

services:
  backend:
    build:
      context: ../services/backend
      dockerfile: Dockerfile
    ports:
      - "8000:8000"
    environment:
      - APP_ENV=production
      - HOST=0.0.0.0
      - PORT=8000
    volumes:
      - ../data_samples:/app/data_samples
      - ../services/backend/app/结果文件:/app/app/结果文件
      - ./logs:/app/logs
    restart: unless-stopped
    healthcheck:
      test: ["CMD", "curl", "-f", "http://172.20.10.2:8000/api/health"]
      interval: 30s
      timeout: 10s
      retries: 3

  frontend:
    build:
      context: ../apps/frontend
      dockerfile: Dockerfile
    ports:
      - "80:80"
    depends_on:
      - backend
    restart: unless-stopped

  redis:
    image: redis:7-alpine
    ports:
      - "6379:6379"
    volumes:
      - redis_data:/data
    restart: unless-stopped

volumes:
  redis_data:
```

### 构建步骤

#### 1. 安装Capacitor依赖

```bash
npm install @capacitor/cli @capacitor/core @capacitor/android
npx cap init UDAKE com.udake.app
```

#### 2. 同步Capacitor配置

```bash
npx cap sync android
```

#### 3. 配置签名密钥

```bash
# 生成密钥库
keytool -genkey -v -keystore release.keystore -alias udake -keyalg RSA -keysize 2048 -validity 10000

# 复制密钥库到android目录
cp release.keystore android/
```

配置 `android/gradle.properties`：

```properties
MYAPP_RELEASE_STORE_FILE=release.keystore
MYAPP_RELEASE_KEY_ALIAS=udake
MYAPP_RELEASE_STORE_PASSWORD=your_keystore_password
MYAPP_RELEASE_KEY_PASSWORD=your_key_password
```

#### 4. 构建Release APK

```bash
cd android
./gradlew assembleRelease

# 或使用npm脚本
npm run build:android
```

APK位置: `android/app/build/outputs/apk/release/app-release.apk`

#### 5. 构建AAB (Google Play)

```bash
cd android
./gradlew bundleRelease
```

AAB位置: `android/app/build/outputs/bundle/release/app-release.aab`

#### 完整CI/CD流程

创建 `.github/workflows/ci-cd.yml`：

```yaml
name: CI/CD Pipeline

on:
  push:
    branches: [ main, develop ]
  pull_request:
    branches: [ main ]

jobs:
  test:
    runs-on: ubuntu-latest
    strategy:
      matrix:
        node-version: [18.x]
        python-version: [3.11]

    steps:
      - name: Checkout code
        uses: actions/checkout@v3

      - name: Set up Node.js ${{ matrix.node-version }}
        uses: actions/setup-node@v3
        with:
          node-version: ${{ matrix.node-version }}

      - name: Set up Python ${{ matrix.python-version }}
        uses: actions/setup-python@v4
        with:
          python-version: ${{ matrix.python-version }}

      - name: Install Node dependencies
        run: npm ci

      - name: Install Python dependencies
        run: pip install -r requirements.txt

      - name: Type check
        run: npm run typecheck

      - name: Lint
        run: npm run lint

      - name: Frontend tests
        run: npm run test

      - name: Backend tests
        run: pytest tests/ -v

  build:
    needs: test
    runs-on: ubuntu-latest
    if: github.ref == 'refs/heads/main'

    steps:
      - name: Checkout code
        uses: actions/checkout@v3

      - name: Set up Node.js
        uses: actions/setup-node@v3
        with:
          node-version: 18.x

      - name: Install dependencies
        run: npm ci

      - name: Build frontend
        run: npm run build:prod

      - name: Build Electron
        run: npm run build:electron

      - name: Upload artifacts
        uses: actions/upload-artifact@v3
        with:
          name: electron-build
          path: release/

  deploy:
    needs: build
    runs-on: ubuntu-latest
    if: github.ref == 'refs/heads/main'

    steps:
      - name: Checkout code
        uses: actions/checkout@v3

      - name: Deploy to server
        uses: appleboy/ssh-action@master
        with:
          host: ${{ secrets.DEPLOY_HOST }}
          username: ${{ secrets.DEPLOY_USER }}
          key: ${{ secrets.DEPLOY_KEY }}
          script: |
            cd /opt/udake
            git pull origin main
            docker-compose down
            docker-compose up -d --build
```

## 故障排查

### 域名环境变量常见问题

#### 问题1: 构建时报错“环境变量缺失：OFFICIAL_WEB 与 ADMIN_WEB 必须同时存在”

**解决方案**:

1. 检查当前构建模式对应的环境文件是否存在。
2. 确认文件中同时包含 `OFFICIAL_WEB` 与 `ADMIN_WEB`。
3. 必要时显式指定环境文件：

```bash
node scripts/replace-domains.js --mode=production --env-file=configs/env/.env.production
```

#### 问题2: 构建成功但链接仍是旧域名

**解决方案**:

1. 查看 `dist/domain-replaced/` 下对应输出文件是否已更新。
2. 确认旧链接是否命中替换规则（见 `scripts/replace-domains.js` 中 `REPLACERS`）。
3. 若新增了域名模式，需要补充替换规则并执行 `npm run test:replace-domains`。

#### 问题3: 前端运行时读取不到 `VITE_OFFICIAL_WEB` 或 `VITE_ADMIN_WEB`

**解决方案**:

1. 变量名需为 `OFFICIAL_WEB` / `ADMIN_WEB`（Vite 注入时自动映射到 `VITE_*`）。
2. 修改环境文件后重启 `npm run dev`。
3. 在浏览器控制台执行 `import.meta.env.VITE_OFFICIAL_WEB` 验证值是否正确。
