# 采样优化模块

## 模块简介

采样优化模块是UDAKE平台的智能化功能，通过自适应采样策略和算法优化，帮助用户在有限的采样预算下获得最大的信息收益。该模块能够自动识别高不确定性区域，推荐最优的新增采样点位置，显著提高数据采集效率和预测精度。

## 核心功能

### 1. 自适应采样策略

- **不确定性驱动采样**
  - 基于克里金方差的采样点推荐
  - 高不确定性区域优先采样
  - 动态调整采样密度

- **多目标优化采样**
  - 平衡覆盖度与收益
  - 考虑成本效益比
  - 支持约束条件优化

- **时空联合采样**
  - 时间序列采样优化
  - 空间分布均衡考虑
  - 时序相关性分析

### 2. 采样点推荐

- **智能推荐算法**
  - 基于不确定性的推荐
  - 考虑空间分布均衡性
  - 支持多候选点排序

- **推荐结果可视化**
  - 推荐点位地图展示
  - 优先级标识
  - 预期收益估算

- **交互式选择**
  - 用户可调整推荐点
  - 实时更新优化结果
  - 支持批量操作

### 3. 采样方案评估

- **覆盖度分析**
  - 空间覆盖度计算
  - 采样密度分布
  - 覆盖盲区识别

- **收益预测**
  - 预期不确定性降低
  - 预测精度提升估算
  - 成本效益分析

- **方案对比**
  - 多方案并行评估
  - 关键指标对比
  - 最优方案推荐

### 4. 采样密度优化

- **密度分布优化**
  - 自适应密度调整
  - 区域差异化配置
  - 动态密度更新

- **采样预算管理**
  - 预算约束下的优化
  - 分阶段采样计划
  - 资源配置建议

   - 导出推荐采样点坐标

   - 支持多种格式导出

## 配置说明

### 采样策略配置

#### 不确定性驱动策略
```json
{
  "strategy": "uncertainty_driven",
  "target_points": 10,
  "uncertainty_threshold": 0.3,
  "min_distance": 100,
  "max_distance": 1000
}
```

#### 多目标优化策略
```json
{
  "strategy": "multi_objective",
  "objectives": ["coverage", "uncertainty", "cost"],
  "weights": [0.4, 0.4, 0.2],
  "budget": 5000,
  "target_points": 15
}
```

#### 时空联合策略
```json
{
  "strategy": "spatiotemporal",
  "time_window": 7,
  "spatial_balance": 0.7,
  "temporal_balance": 0.3,
  "max_new_points": 20
}
```

### 优化参数设置

#### 基础参数
- **目标采样点数**：根据需求和预算设置
- **最小间距**：避免采样点过于集中，建议50-200米
- **最大间距**：确保覆盖度，建议500-2000米

#### 高级参数
- **不确定性权重**：0.0-1.0，越大越重视不确定性
- **覆盖度权重**：0.0-1.0，越大越重视空间分布
- **成本权重**：0.0-1.0，越大越重视成本控制

### 约束条件

#### 空间约束
- **可采样区域**：定义可采样的地理范围
- **禁采区域**：标记无法采样的区域
- **边界缓冲**：避免边界效应

#### 成本约束
- **总预算限制**：设置最大采样成本
- **单点成本**：定义每个采样点的成本
- **交通成本**：考虑采样点可达性

#### 时间约束
- **采样时间窗口**：限制采样时间范围
- **采样频率**：控制采样频次
- **时序要求**：满足特定时间需求

## 技术实现

### 核心算法

- **不确定性计算**
  ```
  U(x) = σ²(x) / max(σ²)
  ```
  其中：U(x)为位置x的归一化不确定性，σ²(x)为克里金方差

- **覆盖度计算**
  ```
  C = 1 - exp(-λ × N / A)
  ```
  其中：C为覆盖度，N为采样点数，A为区域面积，λ为覆盖系数

- **多目标优化**
  ```
  max Σ wᵢ × fᵢ(x)
  s.t. gⱼ(x) ≤ 0
  ```
  其中：wᵢ为目标权重，fᵢ为目标函数，gⱼ为约束条件

### 后端实现

- **核心类**：`SamplingOptimizer`、`UncertaintySampler`、`MultiObjectiveOptimizer`
- **算法库**：scipy.optimize、pymoo、DEAP
- **性能优化**：并行计算、缓存机制、算法加速

### 前端实现

- **可视化组件**：基于Leaflet、ECharts、Plotly
- **交互设计**：拖拽调整、实时反馈、方案对比
- **响应式布局**：支持桌面端和移动端

## 常见问题

### Q1: 如何选择合适的采样策略？

A: 根据应用场景选择：
- **不确定性驱动**：适用于已有初步数据，需要提高精度的场景
- **多目标优化**：适用于需要综合考虑多个因素的场景
- **时空联合**：适用于需要考虑时间序列的场景

### Q2: 采样优化结果的可靠性如何保证？

A: 通过以下方式保证：
- 使用经过验证的优化算法
- 考虑多种影响因素
- 提供多个候选方案
- 支持人工调整和验证

### Q3: 如何平衡采样成本和精度需求？

A: 采用以下方法：
- 进行成本效益分析
- 设置合理的预算约束
- 分阶段实施采样
- 优先采样高价值区域

### Q4: 采样优化对现有采样计划的影响？

A: 影响程度取决于：
- 现有采样点的分布和质量
- 优化目标的设定
- 可用的采样预算
- 时间约束条件

### Q5: 如何验证采样优化方案的效果？

A: 通过以下方法验证：
- 实施后重新进行不确定性分析
- 对比优化前后的预测精度
- 统计采样效率提升情况
- 收集实际应用反馈
