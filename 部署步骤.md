# 本地服务器部署步骤

将PC笔记本电脑作为本地服务器，让Android设备通过WiFi访问后端服务。

## 前提条件

- macOS系统（其他系统参考对应命令）
- Python 3.9+ 已安装
- Android设备已连接USB调试
- 笔记本和Android设备在同一WiFi网络

## 步骤1：获取笔记本IP地址

### macOS
```bash
ifconfig | grep "inet " | grep -v 127.0.0.1
```

输出示例：`inet 192.168.1.100`

### Windows
```bash
ipconfig
```

找到"无线局域网适配器 WLAN"下的IPv4地址

## 步骤2：启动后端服务

```bash
cd /Users/wuchenkai/UDAKE/backend
python run.py
```

后端将启动在 `0.0.0.0:8000`，监听所有网络接口。

**预期输出**：
```
🚀 启动 智能不确定性驱动空间决策平台
📍 地址: http://0.0.0.0:8000
📖 API文档: http://0.0.0.0:8000/docs
```

## 步骤3：配置防火墙

### macOS 防火墙设置

**方案A：图形界面**
1. 系统偏好设置 → 安全性与隐私 → 防火墙
2. 防火墙选项 → 添加 Python 应用
3. 允许传入连接

**方案B：命令行（临时关闭，仅测试用）**
```bash
sudo /usr/libexec/ApplicationFirewall/socketfilterfw --setglobalstate off
```

**方案C：添加Python到允许列表**
```bash
sudo /usr/libexec/ApplicationFirewall/socketfilterfw --add /usr/local/bin/python3
sudo /usr/libexec/ApplicationFirewall/socketfilterfw --unblock /usr/local/bin/python3
```

## 步骤4：创建环境配置文件

```bash
cd /Users/wuchenkai/UDAKE
cp .env.example .env
```

编辑 `.env` 文件，配置以下关键参数：

```env
# 后端API地址（替换为你的实际IP）
VITE_API_BASE_URL=http://192.168.1.100:8000
VITE_WS_URL=ws://192.168.1.100:8000

# 地图API密钥（必须配置）
VITE_AMAP_API_KEY=你的高德API密钥
VITE_AMAP_SECURITY_CODE=你的高德安全码

# 其他配置
APP_NAME=智能不确定性驱动空间决策平台
DEBUG=True
HOST=0.0.0.0
PORT=8000
```

## 步骤5：测试后端连接

### 在笔记本上测试

```bash
# 测试本地访问
curl http://localhost:8000/health

# 测试局域网访问（使用实际IP）
curl http://192.168.1.100:8000/health
```

**预期输出**：`{"status": "healthy"}`

### 从Android设备测试

1. 打开手机浏览器
2. 访问：`http://192.168.1.100:8000/docs`
3. 应该能看到 FastAPI 的交互式文档页面

## 步骤6：构建前端和APK

```bash
cd /Users/wuchenkai/UDAKE

# 构建生产版本前端
npm run build:prod

# 构建Android APK（包含以下步骤）
# 1. 更新 Capacitor 配置
# 2. 同步到 Android 项目
# 3. 编译 Release APK
npm run build:android
```

APK生成位置：`android/app/build/outputs/apk/release/app-release.apk`

## 步骤7：安装APK到Android设备

```bash
# 检查设备连接
adb devices

# 安装APK
adb install android/app/build/outputs/apk/release/app-release.apk
```

## 常见问题解决

### 问题1：无法连接到后端

**检查清单**：
- [ ] 后端是否正在运行？
- [ ] 防火墙是否阻止了连接？
- [ ] IP地址是否正确？
- [ ] 手机和笔记本是否在同一WiFi？
- [ ] 后端是否监听在 `0.0.0.0`？

**诊断命令**：
```bash
# 检查后端进程
ps aux | grep python

# 检查端口监听
lsof -i :8000

# 测试本地连接
curl http://localhost:8000/health

# 测试局域网连接
curl http://192.168.1.100:8000/health
```

### 问题2：防火墙阻止连接

**症状**：本地可以访问，但局域网其他设备无法访问

**解决方案**：
```bash
# 查看防火墙状态
sudo /usr/libexec/ApplicationFirewall/socketfilterfw --getglobalstate

# 临时关闭防火墙（仅测试）
sudo /usr/libexec/ApplicationFirewall/socketfilterfw --setglobalstate off

# 添加Python到允许列表
sudo /usr/libexec/ApplicationFirewall/socketfilterfw --add /usr/local/bin/python3
sudo /usr/libexec/ApplicationFirewall/socketfilterfw --unblock /usr/local/bin/python3
```

### 问题3：IP地址变化

**症状**：重启后无法连接，IP地址改变

**解决方案**：

**方案A：路由器静态IP分配**
1. 登录路由器管理界面
2. 找到DHCP设置
3. 为笔记本MAC地址分配固定IP（如 `192.168.1.100`）

**方案B：使用mDNS/Bonjour（macOS）**
```bash
# 在.env中使用主机名
VITE_API_BASE_URL=http://你的笔记本名.local:8000
VITE_WS_URL=ws://你的笔记本名.local:8000
```

注意：Android设备需要支持mDNS（部分设备可能不支持）

### 问题4：地图无法显示

**原因**：地图API密钥未配置或无效

**解决方案**：
1. 检查 `.env` 文件中的API密钥配置
2. 确认密钥已激活且未过期
3. 检查密钥是否有域名限制（本地测试应无限制）
4. 重新构建APK：`npm run build:android`

### 问题5：构建APK失败

**常见原因**：
- 签名密钥未配置
- Java版本不兼容
- Gradle配置错误

**检查步骤**：
```bash
# 检查签名密钥
ls -la .android_keystore/UDAKE.jks

# 检查Java版本
java -version

# 检查Gradle
cd android
./gradlew --version
```

## 优缺点对比

### 优点

- ✅ **免费**：无需购买云服务器
- ✅ **低延迟**：本地网络传输速度快
- ✅ **数据安全**：数据在本地，不上传云端
- ✅ **完全控制**：可以随时修改配置和代码
- ✅ **适合开发**：便于调试和测试

### 缺点

- ⚠️ **依赖笔记本**：需要保持笔记本运行
- ⚠️ **网络限制**：只能在同一WiFi网络使用
- ⚠️ **休眠影响**：笔记本休眠时服务不可用
- ⚠️ **防火墙配置**：需要正确配置防火墙
- ⚠️ **IP变化**：IP地址可能改变

## 进阶配置

### 1. 后台运行后端

使用 `nohup` 或 `screen` 让后端在后台运行：

```bash
# 使用nohup
nohup python run.py > backend.log 2>&1 &

# 使用screen
screen -S backend
python run.py
# Ctrl+A+D 退出screen，但进程继续运行
```

### 2. 自动启动服务

创建启动脚本 `start-backend.sh`：

```bash
#!/bin/bash
cd /Users/wuchenkai/UDAKE/backend
python run.py
```

添加执行权限：
```bash
chmod +x start-backend.sh
```

### 3. 监控后端状态

```bash
# 查看后端日志
tail -f backend.log

# 检查进程
ps aux | grep "python run.py"

# 查看端口占用
lsof -i :8000
```

### 4. 性能优化

编辑 `backend/app/config.py`：

```python
# 增加并发任务数
MAX_CONCURRENT_TASKS: int = 10

# 调整超时时间
TASK_TIMEOUT: int = 7200

# 优化日志级别
LOG_LEVEL: str = "WARNING"  # 生产环境
```

## 端口使用说明

| 端口 | 服务 | 用途 |
|------|------|------|
| 8000 | 后端API | HTTP RESTful API |
| 8000 | WebSocket | 实时任务状态推送 |
| 5173 | 前端开发服务器 | 开发时使用（生产环境不需要） |

## 安全建议

1. **仅在本地网络使用**：不要将后端暴露到公网
2. **配置防火墙**：只允许特定IP访问
3. **使用HTTPS**：生产环境建议配置SSL证书
4. **定期更新**：及时更新依赖包和安全补丁
5. **数据备份**：定期备份任务数据和结果文件
