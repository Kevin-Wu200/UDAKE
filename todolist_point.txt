## 阶段2：后端API开发（优先级：高）

- [x] 5. 创建后端API文件 /backend/app/api/采样点影响评估接口.py
  - 基础框架搭建
  - 依赖注入配置
  - 错误处理中间件

- [x] 6. 实现 POST /api/sampling-impact/evaluate-candidates 接口
  - 功能：评估候选采样点的影响
  - 请求参数：task_id, candidate_points, strategy
  - 返回：评估结果列表（variance_reduction, local_improvement, comprehensive_score等）
  - 性能要求：处理50个候选点响应时间<5秒

- [x] 7. 实现 POST /api/sampling-impact/preview-effect 接口
  - 功能：实时预览添加新采样点后的效果
  - 请求参数：task_id, new_point
  - 返回：variance_reduction_map, total_variance_reduction, influence_radius等
  - 性能要求：响应时间<3秒

- [x] 8. 实现 POST /api/sampling-impact/recommend-optimal 接口
  - 功能：推荐最优采样点
  - 请求参数：task_id, n_recommendations, strategy, constraints
  - 返回：推荐点列表和收益摘要
  - 支持约束：min_distance, region_bounds, max_points_per_region
  - 性能要求：推荐10个点响应时间<10秒

- [x] 9. 实现 POST /api/sampling-impact/batch-simulate 接口
  - 功能：批量模拟多个采样方案的效果
  - 请求参数：task_id, sampling_plans
  - 返回：各方案的对比结果
  - 用途：对比不同采样策略的效果
  - 优先级：中

## 验收标准

1. 推荐的采样点能显著降低预测方差（20-30%）
2. 插值精度提升（RMSE降低15-25%）
3. 采样成本降低30-40%
4. 实时预览响应时间<3秒
5. 推荐生成响应时间<10秒
6. 所有核心功能通过端到端测试
7. 用户体验流畅，无明显卡顿

## 风险和注意事项

1. **性能风险**：LOO交叉验证计算量大，需要优化
   - 解决方案：使用缓存、并行计算、近似算法

2. **兼容性风险**：新API可能与现有功能冲突
   - 解决方案：保持向后兼容、充分测试

3. **用户体验风险**：功能复杂可能导致用户困惑
   - 解决方案：提供清晰引导、默认合理设置

4. **数据风险**：候选点生成可能覆盖区域不完整
   - 解决方案：多策略混合、自适应密度调整
